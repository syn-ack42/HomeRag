CADIS Knowledge Base : Infoscan

Created by Mohamed.Shebl, last modified by Julian Beyer on Sep 08, 2025

General information

To get shipment information of a package and with that to get an overview of the shipment and its packages, the function “infoscan” can be used.

The function can be selected in the app or in the web user interface – independent from other processes. Afterwards the package is scanned or the barcode, respectively the package number is entered manually.

Required authorizations:

  • Infoscan Standard: ScanItem, ViewItem, ViewItemQuickView

  • Infoscan Advanced: EditStoringPosition, EditDeviations

Following information is available:

  1. Shipment information

  2. Package information of the scanned package, respectively of the further packages of the shipment (package number, SHOULD storing location, IS storing location, unloading-, inventory-, loadingstatus)

  3. Execution date

  4. Positions

  5. Instructions

  6. Loading equipment

  7. Dangerous goods information

  8. Status of the package in the respective process

  9. Name and status of the unloading list, on which the package was last registered

  10. Unloading status

  11. Dispatching info of the shipment

  12. Shipment history

  13. Handling instructions – filter- and sorting function as well as free text search is included

Furthermore the user can update storing location information (IS location) or document damages / deviations. IS storing location can be selected, entered by free text or it can be set by scan of the function barcode.

If the user scans a barcode, for which there is no match, the user gets an error message.

In case of several matches the user can select based on compact shipment data or look at the shipment details of all potentially qualified shipments. After selecting a shipment the user reaches the detail dialog of the shipment. The handling instructions tab is also visible there.

The masterpallet in the infoscan

If a master pallet is recognized in the infoscan, it is displayed in an own mask

  • Header

  • Quick view (if corresponding rights are available)

    • Consignee – in case of a MP consignee

  • Information

    • Consignee – in case of a MP consignee

  • Contact data (in case of a MP consignee)

  • Refresh function

Functions

  • Storing location can be set

  • Deviations can be set

  • Packages of the masterpallet are displayed

  • History is displayed

  • Photos can be displayed

  • Masterpallet can be closed

  • Masterpallet can be dissolved

Setting a storing location

The user has the possibility to set a storing location for the masterpallet by infoscan. The storing location can be scanned directly (analogous to the storing location scan of a package) or it can be entered. It is transferred to all packages.

An entry in the history of the masterpallet and in the history of the package is done.

Setting deviations

The user has the possibility to set a deviation for the masterpallet by infoscan (analogous to set deviations for a package). It is not transferred to the packages. The deviation is displayed at the end of the information section.

Analogous to the packages all last status values of the events of the masterpallet are displayed.

Display of the packages

Packages that are integrated into the masterpallet get the masterpallet icon. Packages that are not integrated into the masterpallet do not have any icon.

Packages that are integrated into another masterpallet get the masterpallet icon and are represented greyed out.

Display of the history

The history of a masterpallet can be displayed to the user by infoscan (own button). A window is opened (analogous to the packages history).

Display of the masterpallet photos

The tab for the masterpallet photos is only visible if photos are available. Name, creation time, event and reason are displayed for every photo.

Closing the masterpallet

The masterpallet can be closed by infoscan. The function is only offered, when the masterpallet has not been yet closed, dissolved or if it is not empty.

Dissolving the masterpallet

The masterpallet can be dissolved by infoscan.

A security question is made, if the masterpallet really should be dissolved.

  • Masterpallet in no open event: “The assignment for all packages is cancelled. Dissolve masterpallet?”

  • Masterpallet in an open event: “The assignment for all packages is cancelled. The masterpallet is in a not completed event: EVENT XXXX. Would you dissolve the masterpallet anyway?”
