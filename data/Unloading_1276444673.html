Unloading Overview

Open the Unloading overview using the :Entladung: symbol in the cadis control panel.

The unloading overview shows the unloading list for a day in overview form with the ability to monitor the unloading progress.

Filter area

You can filter the unloading lists by date, type of transport, inbound relation and status

Table

Tooltips on the columns show column details. For columns with totals these are displayed in the footer.

Here are additional details for specific columns:

  • Status or possible status values are:

    • Created

    • Released (for unloading)

    • In progress (Unloading has started)

    • Finished (Terminal staff have reported the unloading is complete)

    • Closed (Dispatcher has finalized the unloading)

  • Shipments / Packages / Weight
    In these columns you can follow the unloading progress.

  • Damaged goods, damaged packaging, issues
    The :SpaltenBeschAbweich: columns show the damages and issues that are registered during this unloading process. The :SpaltenBeschAbweichVorgänge: columns show the damages and issues that are registered in other operations (loading / unloading, inventory, delivery).

  • Property column
    These are represented by symbols. Per tooltip you get the number of shipments with the corresponding property.

Context menu

  • Unloading details.
    Opens the Details screen, alternatively, you can also double-click on the unloading list.

  • Release unloading list.
    Terminal staff can register the unloading list after its release.

  • Close unloading.
    Finalizes the loading list after a terminal employee has reported end of the loading.

  • Unloading report
    The unloading report contains a summary of the unloading process that have been executed.

  • Print start label
    The staff employee can link to the unloading list using the start label.

Unloading details

In the Unloading details you can see the details of an unloading list particularly the shipments.

Display areas

Menu

  • Release unloading list
    Terminal staff can link to the unloading list

  • Close unloading
    This is used to finalize the unloading process after a terminal employee has reported end of unloading.

  • Third loading unit
    A section for recording a third loading unit is shown in the “Loading unit” area

  • Move loading units
    You can change the order of the loading units.

  • Print
    unloading report and start label

Toolbar

  • :refresh:: Refresh the screen

  • Cancelled values: Shows cancelled shipments

  • Previous / Next unloading list: If you have selected several unloading processes for opening in the unloading overview, you can use these buttons to scroll between the selected unloading lists.

Display block left

General information concerning the unloading process. Here you can for example enter the arrival time.

Header area

  • The numbers / weights which are available as a target position on the unloading list are evaluated as “Planned”.

  • “Dynamic” are the shipments that were not present on the unloading list as a target, but were actually scanned as unloading, and for which the shipment data is known. These shipments are then dynamically added to the unloading list.

  • Unknown packages are those for which no shipment data is known and which were not on the unloading list.

  • The bar color below the numbers means:
    Black: Normal state, no specifics
    Green: All shipments of the target position have been scanned
    Orange: Unknown packages

  • Loading units
    You can see the loading units that are being unloaded. The values were either specified in the unloading list by the host system or were recorded by cadisAPP. You also have the option to record the loading units yourself.

Shipments Tab

Displays the shipments with unloading state.

Fields with issues vs. normal state, e.g. no shipment number (Unknown package) or Unloading with issues (for ex. Damage) are framed in orange.

The following functions are made available using the context menu:

  • Item unloaded
    This sets the status “Unloaded” (without issues) for all packages, regardless of what the mobile system has previously reported. Since the status set by cadisCONTROL has priority over the status set by the mobile system, the entire shipment is considered unloaded.

  • Item not unloaded
    This sets the status “Expected” for all packages, regardless of what the mobile system has previously reported. Since the status set by cadisCONTROL has priority over the status set by the mobile system, the entire shipment is considered as not unloaded (status Expected).

  • Set MDE status again
    Deletes the status set using cadisCONTROL and resumes the status assigned by the mobile system.

  • Status details
    Switches to the status details, description see below.

  • Print dangerous goods attachment

  • Print dangerous goods international attachment

  • Shipment details
    Switches to the Shipment details screen.

  • Edit instructions
    Edits the instructions. You can add, modify or remove instructions.

  • Set processing status
    Sets a processing status for processing that takes place outside cadis.
    Note: The processing status only applies to this unloading list, it is not visible in other processes or in the shipment overview!

Vehicle / Loading unit Tab

Shows vehicles and loading units that were specified when importing the unloading list in grey.

The current values are shown in black. Initially the default values are used over as current values, but they can be changed using cadisCONTROL or using cadisAPP.

Unloading report Tab

Enter data concerning loading without a shipment reference. Data that was entered by cadisAPP is also available here.

Gates Tab

Shows imported values of imported unloading lists in grey and the manually entered gates (using cadisCONTROL or cadisAPP) are in black.

Seals Tab

Shows imported values of imported unloading lists in grey and the manually entered seals (using cadisCONTROL or cadisAPP) are in black.

Photos Tab

Master pallets Tab

Here you can see the master pallets to be unloaded. When a master pallet is unloaded, the status of the shipments that were unloaded with the master pallet are also set to Unloaded, as shown in the Shipments tab.

Status details

Shows the status details of the unloading list for a shipment.

Packages Tab

Status at package level
Among other things, you can see the status that was assigned using cadisAPP (MDE status), the status that was assigned using cadisCONTROL (GUI status) and the resulting status (Status column).

Context menu:

  • Package unloaded
    Sets the status “Unloading” (without issue) for the package, regardless of what the mobile system has previously reported. Since the status set by cadisCONTROL has priority over the status set by the mobile system, this package is considered unloaded.

  • Package not unloaded
    Sets the status “Planned” for the package, regardless of what the mobile system has previously reported. Since the status set by cadisCONTROL has priority over the status set by the mobile system, this package is considered not unloaded (Planned status).

  • Set MDE status again
    Delete the status assigned using cadisCONTROL and reverts to the status assigned by the mobile system.

  • Package details
    Shows further details relating to the package.

    Shows the issues recorded by cadisAPP. With the :PosAddSub: buttons you add an additional issue (discrepancy) or you delete the selected one.

  • Photo documentation
    Displays photos of the package that have been taken.

Loading equipment Tab

Shows the loading equipment recorded by cadisAPP.
If non-exchangeable items have been entered, you can see them in a separate window by double-clicking on the corresponding line. You can also enter loading equipment here.

Unloading report Tab

You can enter additional information about the shipment.

Photos Tab

Displays photos of the shipment that have been taken.
