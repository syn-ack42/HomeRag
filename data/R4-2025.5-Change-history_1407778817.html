cadis 2025.5.4

Released on November 11th 2025

New features

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-4388] [cadisAPP] During pickup/delivery, special “dm” loading equipment (pallets, cages) must be scanned and signed. Status updates for collection/delivery including loading equipment exchange are sent to abmSoft.

  • [RD-4492] [Business code] Added the ability to export GPS coordinates of vehicles transporting "dm" cargo.

Improvements

Technical topics

  • [RD-4463] [Business code] ESB SFTP transfer functionality has been introduced, allowing files to be securely transferred between systems using the SFTP protocol.

  • [RD-4573] [Business code] ESB/REST additional http headers may now contain ESB pattern variables and credential-store lookups

Bug fixes

General

  • [RD-3968] [cadisSTUDIO] All Boolean icons are now correctly included in the CSV export.

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-4131] [Business code] Avoid exception for items wihout dispatchable status in dispatch bulk operation.

  • [RD-4318] [Business code] Prevents the creation of error entries in the event log in cases where the workflow is already completed, but the order is not yet finalized because a new PUD status has been set.

  • [RD-4344] [cadisAPP] Resolved an issue that caused the app to restart when a stop was removed from a tour.

  • [RD-4371] [cadisSTUDIO] Resolved an issue where setting a date or creating a new Collection templates in the Template Data section and selecting a product triggered an error, preventing the changes from being saved.

  • [RD-4558] [Business code] If a shipment is reactivated on a departure that was previously scheduled and cancelled, the system will automatically reschedule the shipment.

  • [RD-4576] [cadisSTUDIO] In the mobile workflow configuration for order closure Order signature types are listed in the order of selection.

cadis 2025.5.0

Released on September 18th 2025

General

New features

  • [RD-3688] [Software update] A new update screen ensures mandatory cadisAPP updates. Users may postpone briefly, but must update after the transition period to keep using the app. Mandatory versions set in cadisSTUDIO are now enforced. Updates are required even if users stay logged in and skip the normal login check.

Improvements

  • [RD-3971] [cadisSTUDIO] User management: Display additional information in CADIS user overview (Created, Created by, Last change, Last change by, Last activity) to help administrators identify unused accounts and improve account lifecycle management.

  • [RD-3972] [cadisSTUDIO] User management:  Enhanced OIDC integration to obtain and display users' first name, last name, email, phone number, and department from OIDC provided tokens in the CADIS user overview, improving user profile completeness and administrative visibility.

  • [RD-1916] [Error messaging] Improved error message when an Azure login service issue occurs. Previous: "Server error: User directory is not available", New: "Login service not available. Please contact your IT administrator."

  • [RD-3047] [cadisSTUDIO] Geocoding: Added pin placement functionality to allow users to add pins directly on the map in both stop locations details and collection (template) creation dialogues, providing manual location selection capability alongside automated geocoding results.

  • [RD-3249] [cadisSTUDIO] Resolved issues that caused cadisSTUDIO to crash due to network problems. When the server is updated (i.e., the version changes), the client now automatically restores connectivity by updating itself accordingly. If the server is offline or unreachable, the application now displays a dedicated offline page instead of crashing.

  • [RD-3264] [cadisSTUDIO] The function menu accessed via the 3-dot button now has a consistent appearance and behaviour across all locations in the application.

  • [RD-3343] [cadisSTUDIO] Improved the display of the "Pin Column" submenu in tables.

  • [RD-3345] [cadisSTUDIO] The "Open" action in all masks has been renamed to "Details".

  • [RD-3667] [cadisSTUDIO] The application no longer shows a loading spinner just because the desktop notification or the overview page polls the server every 30 seconds.

    • Fixed table busy indicator behaviour to prevent loading spinner during background polling operations.

    • Implemented proper spinner display for newly added tables during filter changes and manual refresh operations.

    • Hidden "No rows to show" message when table busy indicator is active.

  • [RD-3988] [cadisSTUDIO] Removed the “Alpha” status from key features in cadisSTUDIO required for the Free Trial offering, such as Shipments and related master data components (PackageDeviations, PackageTypes, OrgTree). These features are now fully enabled to support essential workflows without limitations.

  • [RD-3989] [cadisSTUDIO] The filter reset functionality now works properly even when the search bar contains text, eliminating the need for users to manually clear the search field before resetting other filters across all tables.

  • [RD-4179] [cadisSTUDIO] In all tables: Quick filter text is now also stored in the filter view, ensuring consistent behaviour across the application and improving user experience by preserving search criteria when saving and restoring filter views.

  • [RD-4200] [cadisSTUDIO] Improved combo box visibility in small masks so that all values are fully visible and the combo box opens on top of the mask for better user experience and data accessibility.

Bug fixes

  • [RD-4078] [cadisAPP] Corrected barcode processing for various update scenarios, with a focus on package cancellations.

  • [RD-4186] [cadisCONTROL] The width of all boolean columns can be individually changed. This applies to all tables.
