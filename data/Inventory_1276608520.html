Inventory Overview

Open the Inventory Overview using the :Inventur: symbol in the symbol control panel.

The inventory overview shows the inventory lists for a day in overview form with the option to monitor progress.

Filter area

You can filter the inventory processes by date.

Table

In the column headers, you can see an explanation of the column as a tooltip. For columns for which totals are displayed, this is shown in the footer.

Here are additional explanations for some columns:

  • Status, possible status values are:

    • Created

    • Released (for inventory)

    • In progress (Inventory list has been started)

    • Finished (Terminal staff has reported the end of unloading)

    • Closed (Dispatcher has finalized the unloading)

  • Shipments / Packages / Weight
    In these columns you can follow the progress of the inventory.

  • Damaged goods, damaged packaging, issues
    The :SpaltenBeschAbweich: columns show the damages and issues that are registered in this inventory. The :SpaltenBeschAbweichVorgänge: columns show the damages and issues that are registered in other operations (loading / unloading, inventory, delivery).

  • Property column
    These are represented by symbols. Using the tooltip you get the number of shipments with the corresponding property.

Context menu

  • Inventory details
    Open the Inventory Details screen, alternatively you can also double-click on the inventory.

  • Release inventory list
    Terminal employees can register for the inventory after the release.

  • Close inventory list
    This finalizes the inventory after a terminal employee has reported the end of the inventory.

  • Generate planned inventory data
    All shipments matching the created inventory (Type of inventory, Type of area, Area), which are shown as in the terminal (status “At terminal” or “partially at terminal”) are assigned to the inventory list.

  • Print start label
    Prints a start label that can be used by the terminal employee to register the inventory list.

New

With the New button or alternatively Edit menu/ New you can create a new inventory list. Complete the yellow fields in the Inventory details screen and save your data.

For type of area “Area” select an area from the area master data in the area field.

For type of area “Position” select a location from the location master data in the area field.

For type of area “Special” enter the storing location in plain text in the special storing place field.

Inventory Details

Display Areas

Menu

  • Edit:
    You can set the status of the physical inventory list or generate target data

  • Print:
    You can print the start label which is used by the terminal employee to register the inventory in cadisAPP. The shipments can also be printed.

Toolbar

  • :refresh:: Refresh the screen

  • Cancelled values: Also displays cancelled shipments.

  • Previous / Next Inventory: If you have selected several inventory lists to open in the inventory overview, you can then use these buttons to scroll between the selected inventories.

Storing positions in this inventory list

All storing locations are listed here that either exist as target locations for shipments in this inventory list or are actual storing locations of shipments that were previously scanned.

Header area (Shipments/Packages/Weight)

Here you can track the progress of the inventory list.

The bar color below describes a different status:

  • Black: Normal case, no special features

  • Green: All shipments of the target position are scanned

  • Orange: Unknown packages

Filter

You filter the shipments by inventory status and issues. With the :AbweichFilter: button you can filter according to several issues.

Hide main shipments: If you scan a package of a subsequent delivery shipment, it is displayed in the subsequent delivery shipment and in the original main shipment. It therefore appears in the overview in 2 shipments. If you check this box, only the subsequent delivery item will appear in the overview.

Shipments Tab

The columns are explained as a tooltip on the column header.

Fields with issues from the normal case, such as no shipment number known (unknown package), registered with issues (e.g. damage) and location issues are framed in orange.

The following functions are available using the context menu

  • Register item
    Sets the “Registered (no issues)” status for all items, regardless of what the mobile device has previously reported. Since the status set by cadisCONTROL has priority over the status set by the mobile device, the entire shipment is considered as “Registered“.

  • Deregister item
    Sets the “Expected” status for all items, regardless of what the mobile system has previously reported. Since the status set by cadisCONTROL has priority over the status set by the mobile system, the entire shipment is considered as not registered in the inventory list (status Expected).

  • Set MDE status again
    Deletes the status assigned using cadisCONTROL and so the status assigned by the mobile system is restored.

  • Status details
    Changes the status details, description see below.

  • Shipment details
    Switches to the Shipment details screen.

  • Edit instructions
    Edits the instructions. You can add, change or remove them.

  • Set processing status
    Sets the processing status for processes that take place outside of cadis.
    Please note: The processing status only applies to this physical inventory, it is not visible in other transactions or in the shipment overview!

Packages Tab

Status at package level
Among other components, you see the status that was assigned using cadisAPP (PDC status), the status that was assigned using cadisCONTROL (GUI status) and the resulting status (Status column)

Context menu:

  • Register package
    Sets the “Registered (no issues)” status for the package regardless of what the mobile system has previously reported. Since the status set by cadisCONTROL has priority over the status set by the mobile system, this package is considered “registered”.

  • Deregister package
    Sets the “Expected” status for the package regardless of what the mobile system has previously reported. Since the status set by cadisCONTROL has priority over the status set by the mobile system, this package is considered as not registered in the inventory list (status Expected).

  • Set MDE Status again
    Deletes the status assigned using cadisCONTROL and so the status assigned by the mobile system is restored.

  • Package details
    In the Package details screen you can see further details about the package. 

    You see here the issues recorded in cadisAPP. Using the :PosAddSub: buttons you can add an additional issue (discrepancy) or you delete the selected one.

  • Edit expected storing position

    Defines a new target storing location for a package. You select the target storing location from the location master data. If you click on “Special storing location” you can enter free text.

  • Photo documentation
    Displays the photos of the package

Loading Equipment Tab

Here you can see the loading equipment recorded by cadisAPP.

If non-exchangeable items were entered, you can see them in a separate window by double-clicking on the corresponding line.

Inventory report Tab

You have the possibility to enter additional information about the shipment.

Photos Tab

Displays all photos taken for the shipment.
