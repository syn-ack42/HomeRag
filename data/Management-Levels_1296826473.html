We use the term “user” synonymously for “user account”, as the cadis GUI does.

Each user has exactly one Management level assigned.

There are the following Management levels available with increasing power:

  • None
    meaning no Management level.
    Most cadis users have this level. They have no access to the User Management nor to the User roles.
    If you have this level, you might want to stop reading here.

  • Users locally
    to manage users locally i.e., focused on the (usually few) depots.
    User Management is restricted to users without Management levels. Major use case: Manage the user changes the depots they are managing. User roles can be viewed, but not edited.

  • Users globally
    to manage users (and only users, not the whole system) globally (for all depots running on an installation) AND to maintain the cadis User roles.

  • System

to manage the whole cadis system: Do everything without any restrictions, without any roles assigned.

If you have a Management level other than None, you can perform your cadis user management tasks in “Users & Devices”

under “User management” and “User roles”:

The user Administrator is always available and thus visible. It is the one and only built-in user, used exclusively by the CADIS staff to support the cadis management responsibility:

In your cadis system, the Administrator probably created at least one user for a person in your organization. That user has at least Management Level Users globally and is thus allowed to manage users and user roles.

In addition to the user Administrator, cadis lists all users that were imported or created in cadis.

Moreover, cadis lists external users (users that log in via an external authentication system). They are marked as “External users” in the corresponding column.

User Administrator and external users cannot be updated in cadis.

User specifics per Management level

System

Users with Management level System assigned, in short System managers, have no roles assigned: They’re granted permission to do everything without having roles assigned.

Users globally

Users with Management level Users globally assigned, in short Users globally managers, manage users globally, which means installation wide.

They usually have further roles in their organization. Thus, they may have one or more roles assigned, either installation-wide or for a few depots.

The user details might look like this:

Users locally

Users with Management level Users locally assigned, in short Users locally managers, support Users globally managers by managing users locally i.e., restricted to one (or a few) depots: They have knowledge about the daily changes to staff, for example who’s starting to work as a driver or which drivers have quit their jobs.

Users locally managers usually have roles in cadis on top: They’re deeply involved in the daily transport business. A role may be assigned installation-wide or for one or more specific depots.

Use Cases per Management level

Now find out about your own Management level in the User management section and focus on the chapter for that Management level. There you will find your management options and the how-tos.

Management level System

As user with Management level System assigned, in short System manager, you manage the entire cadis system.
This includes the management of all non-external users and management of the users role assignments, which is the focus of this chapter.

You may as well delegate the user management and user roles management to Users globally managers.

Your initial use cases

  • You may import users, no matter what their Management level is, or delegate the user import to your Users globally managers.

  • You may add another System manager for your colleague if you want to share your responsibility with him.

  • Add Users globally managers

If your focus is system management, you might want to delegate the user management:
Add one or very few users with Management level Users globally that will manage users across all depots.

Fill in the personal user details and select the Management level Users globally.

Add a role for the user you’re adding: What’s his role? In which depots? Choose “all depots installation wide” or select one or several specific depots.

Add more roles if you want:

In the User management tab User roles, you may create user roles or update built-in user roles.

Your general use cases

  • Create/edit/delete users

  • Edit user roles

  • Import users

  • Import user roles

Management level Users globally

As user with Management level Users globally assigned, in short Users globally manager, you manage users globally i.e., installation-wide. You can create, edit or delete user roles. You may import users. However, there’s one restriction: You may not manage (create/edit/delete) System managers.

You may also delegate the depot-specific user management role.

Your initial use cases

  • You may import users if they don’t have Management level System

  • Add other Users globally “colleagues”

Your System manager may have delegated the task “Add a few more Users globally managers”. Please refer to the Management level “System” section and there to “Add Users globally managers”.

  • Add Users locally managers
    You will probably focus on managing users globally, and delegate the depot-specific user management. Therefore, add one or more users with Management level “Users locally”:

  • Users locally managers

    • are supposed to manage users focused on depots

    • know about the daily changes to staff, probably better than you do

    • Example: They know who starts working for a depot as a new driver or which driver has quit their job.

Thus, click “create” in the User management.
Fill in the personal user attributes.

Add a role for the user you’re adding: What’s his role? In which depots? Choose “all depots installation wide” or select one or several specific depots he works for.
Add more roles if you want.

Select the Management level “Users locally” and select the depots for which he shall manage users.

  • Edit or delete existing non-System users

  • Edit User roles

  • Under User management in User roles tab, you can create user roles or update built-in user roles.

Your general use cases

  • Create/edit/delete existing users except System managers.

  • Edit user roles

  • Import users (non-System users only)

  • import User roles

Management level Users locally

As user with Management level Users locally assigned, in short Users locally manager, you support Users globally managers by managing users locally: Only in the depots you’re managing, in “your” depots.

You know about the daily changes to staff, for example who starts driving at your depot or which drivers have quit their jobs. Furthermore, you know which user changes his role in “your” depots.
cadis ensures that you can update / maintain user data with only a local impact, affecting “your” depots only.

Your Management level does not include User roles management nor the import of users or user roles. You can open the user roles in order to get familiar with them.

Your use cases

  • Add users with Management level None

Click “create” in the User management.
Fill in the personal user attributes.

Add a role for the user you’re adding: What’s his role? In which depots? You may only assign/unassign “your” depots.

Add more roles if you want.

The Management level is None by default. You cannot change it.

  • Edit existing users
    In order to make sure your edits have no impact in depots other than “your” depots, your edit options are restricted:

    • Once another Users locally manager or Users globally manager has assigned a depot other than “yours”: You cannot edit the personal user data (User and Contact details) anymore.
      If you see the need for such an update, please ask your Users globally colleague to do so.

    • You can still manage roles for “your” depots

      • Assign/unassign “your” depots from a role

      • remove a role which has only “your” depots assigned

      • add roles, assign “your” depots

    • Restriction due to a role category
      Each permission is of a certain category, which is one of {BUSINESS, INTERNAL, ADMIN, GLOBALADMIN}.
      Each role automatically gets the highest category of all of its permissions (ascending from BUSINESS to GLOBALADMIN). The categories are intended to distinguish certain levels of business power a user gets through a role assignment.

      • You cannot edit users that have roles assigned of the category GLOBALADMIN.
        If you see the need for such an update, please ask your Users globally manager colleague to do so.

  • Delete existing users
    There are the same restrictions as for “Edit existing users”, see above.
