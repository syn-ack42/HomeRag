Please also check out the R4 2024.2 change history here: R4 2024.2 Change history

Logistics Process Platform (LPP) – General functionality

Many features listed here are available in all basic service cut levels (LPP-ENT or LPP- CLS) but for some an additional license may be required while for others accessibility or utility may vary with the limitations of the service cut and your individual usage of cadis. In some cases, availability of information and/or functions in a feature may depend on service cut and other licensed options.

Other Improvements and Bugfixes

  • Default for session timeout increased from previously 4 hours to 4 days for the session to last over the weekend

  • cadisSTUDIO: Search now ignores accented characters and umlauts. I.e. you can search for e.g. “leon” and find “León”, “munchen” and find “München”, “weissmuller” and find “Weißmüller”. In addition, client side searches now match any substring, not just words that start with the search string.

  • “Priority” and “TOP” product shipments are now displayed in a consistent manner:
    A “crown” Icon displays a shipment with a product “isTopProduct = true” according to the master data
    An “arrow up” displays the priority value. This holds for priority=0 as well

  • Fixed a bug when displaying the timeline of a shipment in the details view within cadisSTUDIO

  • online help

    • opens in your browser language or in English as fallback if your language is not supported

    • popular blockers won’t block the online help any more

  • User Management:

    • When you create a user, you can now add login name and password without your browser autocompleting the login name.

    • Removing or changing roles now has an impact on existing sessions:

      • Log off for users with no roles left or an update of their management level

      • Update for users with roles added or roles removed but some permissions left

  • cadisAPP:

    • The welcome screen shows “User” as registration type instead of “Scanner”, presuming you have already (re)registered in cadisAPP with this release.

    • The user details dialog shows the following information about your login if available: User, Vehicle, Driver, Depot and Client-ID.

    • Fixed an occasional error when clicking buttons quickly multiple times, including the BACK button, or scanning a barcode multiple times

Pick Up and Delivery (PUD) 

Many features listed here are available in all basic service cut levels (PUD-DRV, PUD-LDR, PUD-PLD) but for some an additional license may be required while for others accessibility or utility may vary with the limitations of the service cut and your individual usage of cadis. In some cases, availability of information and/or functions in a feature may depend on service cut and other licensed options.

Other Improvements and Bugfixes

  • In the dispatching check message  “Same delivery stop already exists”, you now find the name of the departure where the delivery stop already exists.

  • In order details, type Collection, tab “Packages” the column “Container ID” has been removed (it was always empty)

  • In cadisSTUDIO, F&L Mile, Planning, Stop list: Status “Arrived” is only displayed when actually reached, this was not working properly beforehand.

  • In a trip template, you can either define a vehicle or a vehicle type. A trip created from a trip template takes over the defined alternative (vehicle resp. vehicle type), now even if a vehicle and vehicle type with the same ID exist, this was not possible in the previous version. 

  • cadisSTUDIO: MyTrips and First&Last Mile: Several bugfixes: A message explains why a shipment cannot be dispatched. The unexpected function to register “unknown” packages is now disabled in dispatching.

  • When you select several vehicles resp. trips in the vehicle resp. trip overview, and choose the function “message” to send a message to all these vehicles,  you now see the list of available and selected vehicles in the “Send message” dialogue, resolving an issue which occured in earlier versions.

  • Unprocessed collections (neither collected nor collection hindrance) will now be un-dispatched when trip is finished, resulting in previous problems now being resolved.

  • Training mode for registration type DRIVER is now possible.

  • Dissolving a masterpallet on a trip causes an automatic un-dispatch immediately.

  • In the order details you can print a preview. The formatting of the preview’s history has been improved, irrelevant overlays have been removed.

  • When you try to print an empty report “Delivery with signatures”, you get a message that there are no such deliveries to be printed.

  • The “approach” event for the next stop will now be exported even if the next stop is within the identical geofence.

Notable Minor Changes and Bugfixes

  • Terminal Handling, Import general: Create missing vehicle added.

  • If configured, vehicle master data entries will now be created when importing unknown Vehicle IDs within unloading lists

  • You can configure this new option here:

  • Bug fixed: In Planning, the capacity check concerning weight and pallet storing area now works correctly even if a vehicle type is selected

  • Bug fixed: If an unknown package is unloaded as surplus, and cadis imports the corresponding shipment afterwards, the unloading process will now list the shipment. (The package remains listed as cancelled.)

  • Bug fixed: You won’t see unloading status for collections any longer. Instead, you see the unloading events for collections in the corresponding shipment.

  • Bug fixed: In the trip overview, column “capacity utilization” shows the proper values again. The capacity check (weight / pallet storing area) works not only for defined vehicles, but now for defined vehicle types as well.

Temperature Control (TCL)

ePOT (Electronic Proof Of Temperature)

  • The report now denotes the delivery process as “Delivery”, no longer as “DRV”.

  • There used to be gaps in the temperature curves whenever the temperature sensor value was unavailable for longer than 15 minutes. Now such gaps are only visible if temperature values are unavailable for longer than 30 minutes. For shorter periods the values are interpolated in the curve.

  • A package listed below a sensor curve is now listed by its default barcode only. The latter is no longer preceded by the GlobalTrackingNumber of the shipment it belongs to: This information is redundant because the report is shipment specific anyway.

  • The report now also includes processes (loading and unloading) in which the package was handled without a resulting compartment change resp. temperature sensor change.

  • The report lists all processes done for a package. Specifically, for packages that were not scanned during unloading, the preceding loading scan is no longer missing in the report.

Temperature curve for collections (retroactive assignment)

When unloading collections documented on stop level, cadis now offers a feature for monitoring the first mile transport of collections retroactively, during unloading.
In cadisAPP unloading, if configured, you are now asked to enter the vehicle compartment of the collection you are unloading. As a result, the temperature report visualizes the collection process from collection time until unloading time.

Terminal Handling, Unloading, LOCAL_TRAFFIC, section Temperature: “Loading unit thermo assignment required.” This configuration is for cadisAPP to have the user document the previously unknown vehicle compartment (i.e. scan the barcode on the vehicle compartment) for collected packages contained in the unloading list.

Please read the how-to page for more details: How to document temperature conditions during collection retrospectively, when unloading the packages.

Other Improvements and Bugfixes

  • Improved data retrieval from provider Masternaut to avoid retrieval-related gaps in the EPOT’s temperature curves.

  • cadisAPP: The colour of the temperature banner indicates the temperature range: Blue for 2-8°C, Orange for 15-25°C.

  • If an unloading event for a temperature controlled package is cancelled thereafter by the terminal worker, the scanned package is assumed to remain in the vehicle. Temperature monitoring will continue as if the unloading event didn’t occur.

  • It is now possible to change a terminal compartment’s name
