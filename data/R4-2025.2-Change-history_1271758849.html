cadis 2025.2.11

Released on August 20th 2025

Bug fixes

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-4182] [PTV2] Added configuration option for PTV2 calculation in cadis.ptv2.

    • WORKING_TIME (default, per EU Directive 2002/15/EC)

    • DRIVING_TIME (per EU Regulation EC 561/2006)

    • NONE

cadis 2025.2.10

Released on July 9th 2025

Bug fixes

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-3849] [PTV] Implemented caching of PTV2 Dima data to reduce redundant listDima service calls and improve performance.

  • [RD-3861] [Business code] Host statistics calculation : The calculation of host statistics is now triggered during shipment import, even if the import process makes the shipment dispatchable again.

  • [RD-3924] [Business code] Rejected collections from mobile devices will now be ignored if the corresponding collection shipment has been cancelled in the meantime.

cadis 2025.2.9

Released on June 23rd 2025

Improvements

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-3816] [cadisSTUDIO] Mandatory Volume Entry in GTP:

    • Validation on Save/Route: Action is blocked if both calculated and manual volume are missing. Error: “Volume entry required: Please enter here or in the shipment items.”

    • Calculated OR Manual Volume > 0: Save/Route is allowed.

    • Both present: No restriction.

cadis 2025.2.7

Released on June 3rd 2025

Bug fixes

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-3716] [Reports] Delivery note, Pickup note: The package table in reports previously truncated long serial numbers. Now, long values wrap correctly. Horizontal and vertical lines have been added to improve the layout.

cadis 2025.2.4

Released on April 29th 2025

Improvements

General

  • [RD-3431] [CADIS-SOTI] The timestamp format used in the CADIS-SOTI integration has been updated for improved readability and consistency. The previous format (e.g. 20250403104011Z) has been changed to a more human-readable format: YYYY-MM-DD HH:MM:SS (e.g., 2025-04-03 10:40:11). The new format uses a 24-hour clock for the time component.

Temperature Control (TCL)

  • [RD-841] [Improvement] Masternaut: swap data logger ID and Name.

Bug fixes

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-3459] [Business code] Shipments (mainly collections) with only cancelled packages will now be treated as if they had no packages initially. The main difference is that they remain planned/dispatched at their current trip.

  • [RD-3482] [cadisAPP] Resolved an issue in manual data synchronisation if more than two tour topics are subscribed.

  • [RD-3495] [Business code] Added database changeset to restore lost freight carrier data (tour/departure table) to prior migration.

  • [RD-3537] [Reports] Delivery list: “Prio” is displayed; the amount is empty if there is actually nothing there.

  • [RD-3545] [Business code] When creating unloading lists from local traffic trips within cadis, duplicated license plates will now be avoided (truck is preferred over trailer).

  • [RD-3549] [Public API] Do not filter shipments with state UNDISPO_BY_CLOSED on departures API/Export/Report services. Because they were planned and the state is set by cadis automatically if shipment not loaded.

  • [RD-3603] [Business code] Resolved an issue where shipments that had no packages before and are updated to have qualified package identifications now, were not updated on mobile devices.

Bug fixes

General

  • [RD-3412] [Business code] Resolved an issue where OpenSearch transfers are being attempted even if configuration says otherwise.

  • [RD-3364] [cadisAPP] After a device reboot, DataWedge starts after receiving the LOCKED_BOOT_COMPLETED intent. Calling DataWedge intent APIs before the LOCKED_BOOT_COMPLETED intent may cause unexpected behaviour as DataWedge is still not started yet. Calling DataWedge intent APIs soon after receiving the LOCKED_BOOT_COMPLETED intent may also cause unexpected behaviour as DataWedge may still be in the initialisation process. It is recommended to wait for a few seconds after receiving the LOCKED_BOOT_COMPLETED intent before calling any DataWedge intent APIs.

  • [RD-3367] [cadisAPP] Resolved an issue that caused cadisAPP to crash when no navigation app was installed on the device. The app now detects the absence of a navigation app and provides appropriate feedback to the user, preventing unexpected crashes.

  • [RD-3400] [cadisAPP] Resolved an issue that caused the cadisAPP to crash when rendering the scanner camera preview. This update enhances camera stability, preventing unexpected crashes and ensuring a smoother user experience. Additionally, performance optimisations have been implemented to improve responsiveness and reliability during scanning operations.

  • [RD-3423] [cadisAPP] Resolved an issue where telemetry data and MDM configurations were not being properly loaded or applied during login with cadisAPP.

  • [RD-3447] [cadisAPP] Resolved an issue on devices with Android > 8 and SOTI data integration enabled where users are prompted to grant file system permissions.
