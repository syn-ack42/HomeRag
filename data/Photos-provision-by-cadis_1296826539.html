With cadisAPP, users can document by photos damaged packages at any stage (Loading, delivery, unloading…). The photos taken by users as visible in real time in cadis back office & are also provided to your host system. This is especially important because depending on your wishes, data can be stored in cadis for 3 to 6 months only.

At (Un)Loading stage

After users have closed their (Un)loading list in cadis back office, cadisSERVER issues an xml file, the “(Un)LoadingStatus” which contains the photo ID. Example:

At Delivery Stage

After the driver has closed a delivery stop in cadisAPP, cadisSERVER issues an xml file, the “OrderState“. The photo can be taken at shipment level or package level, generating slightly different tags.

Pdf Export

A pdf file containing all photos taken at package or shipment level can also be exported from cadis towards an endpoint. To do so, you must communicate to cadis a related URL/Token.

How to retrieve the photos

  • Manually via a URL (login to cadisSTUDIO required) : https://YOURSERVER/theblob/details/PhotoID

  • Via an application: a Java Web Token (JWT) must be inserted in the HTTP header

You could also, instead of retrieving & storing the photos, implement a link from your host, opening a web page displaying the photo. In that case, you would be submitted to the cadis data retention period mentioned above (90-180 days)
