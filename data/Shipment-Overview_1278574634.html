Open the List of shipments using the :sendungsuebersicht: symbol in the cadis control panel:

The List of shipments shows all shipments that match the selected filter criteria. The overview is updated automatically every 30 seconds, but can also be updated manually using the :refresh: symbol.

Filter area

To display as many shipments as possible, you can hide the filter area and reveal it using the :FilterDown: symbol.

You can filter the shipments according to different criteria. Different filter sets are implemented for different applications, which you select using the Filter list box.

Each filter set contains different fields that can be used for filtering. One or more fields are used as the main filter criteria. These must always be specified; the fields are marked with a yellow background as mandatory fields.

Most filter options are self-explanatory. Here are some specific cases:

  • “Today” in the filter set means that the execution date is preset to today’s date. But you can change this as you like.

  • Failed shipment import: If shipments are imported from a leading system, updates can be rejected by cadis if the shipment has already been processed in cadis, for example. This is shown on the shipment as an event requiring confirmation and can be acknowledged using the context menu (see below). This filter is used to filter corresponding shipments.

  • Hide main shipments: If a shipment is not delivered completely, the remaining shipment will be delivered using a subsequent delivery shipment. This filter ensures that in such cases, only the subsequent delivery shipment is filtered, but not the main shipment.

  • Filter according to shipment characteristics: it is possible to filter for many shipment characteristics: 
    :reglerMitteGrau: The characteristic is not considered during the filter process 
    :reglerObenGruen: Shipments that have this characteristic are filtered
    :reglerUntenRot: Shipments that do not have this characteristic are filtered

  • Multiple selection
    For some filters whose filter values are selected from selection lists, multiple selections are also possible:
    E.g. Transport type can be selected from the selection list by a single selection

    or several values can also be selected using the :AbweichFilter: button:

  • Cancelled values: This is an additional filter. It can be set at the top of the symbol line. If this checkbox is ticked, then cancelled shipments are also displayed, which are marked with a black background in the column “Cancelled”.

Table

The columns are self-explanatory. In unclear cases, you can display the description of the column as a tooltip by hovering the mouse over the column header. In many cases, if the field content is unclear, you can use the tooltip to obtain further details.

Context menu

Using the context menu (press the right mouse button on the table) you can perform different actions on the selected shipments. Actions that are shown in grey cannot be selected for the selected shipments. You will see the reason for this as a tooltip if you let the mouse hover over the entry.

Actions

Dispatch (secondary distribution) / (primary distribution)

You will be directed to the Dispatching screen, where you can select a secondary distribution route / primary distribution departure for which the selected shipments will be dispatched.

Shipment details

Displays the shipment details. For multiple selected shipments, you can scroll through the selected shipments in the Shipment Details screen.

Arrival Notification

Only available for last mile shipments / collections, which have the corresponding attribute. The notification is triggerd (if configured), at least documented.

Freight Forwarder

Offers the possibility to assign a freight forwarder / Master dispatcher to the shipment or to remove the assignment.

Auto dispatching

This triggers the Auto dispatching function for the selected shipments, whereby the dispatching destination has already been defined using postal code routing and the automatic dispatching rules engine.

Bordereau supplement

Herewith the shipment is manually set to outgoing, the loading list is set to outgoing, the terminal status is not on terminal and the dispatching status is not available.

Start free route optimization with selection

Starts the route optimization for the selected shipments. You will be directed to the Route Optimization screen, where you define the options for the optimization.

Loading list, Unloading list, Inventory status depot

Opens the corresponding list in the case that the shipment is on this type of list.

Related order

Shows the collection or delivery order if it was provided for the mobile system after the route release of the secondary distribution route.

Corresponding tour

If the shipment is on a secondary distribution transport tour, the List of tours is opened and the corresponding tour and stop are selected.

Shipment dispatchable

This sets the shipment to the “dispatchable” status. If you want to deliver a shipment again after a delivery issue, for example, you must set it to “dispatchable” so that it is shown in the list of “dispatchable” (available) shipments and can be scheduled manually or using the automatic dispatching function.

Create partial shipment

A partial shipment is created for the shipment.

Set PUD Status

Sets the collection or delivery status of a secondary distribution shipment if it was not set by the driver of the secondary distribution vehicle or if you want to correct the status assigned by the driver.

Set processing status

You can set a processing status to record processing steps that take place outside cadis, for example, clarification processes in the case of delivery issues.

You can enter a status and explanatory text in this screen:

The current processing status is displayed in the list of shipments in the Processing status shipment column, as a tooltip completed with text and name of the user who entered the status.

Change execution date

If you want to deliver or collect a shipment on a later day, you can change the execution date. The shipment will then no longer appear in the “dispatchable” shipments for the current day and will therefore not be taken into account by the automatic dispatching function for the current day.

Edit instructions

The Shipment details screen is opened with the option of changing or deleting information text or entering new information text

Change shipment group

You can use this function to change the group assignment of the shipment that was made using the Shipment group rules engine and assign the shipment to another group.
To do this, you select one from the list of existing groups:

Dispatching / Loading lock, Remove dispatching / loading lock

You use this function to manually set a loading lock or remove a loading lock that was set in cadis. You cannot remove loading locks that were set by the host system in cadis. Loading locks are marked with a lock symbol :schloss: in the :verladesperre: column. The tooltip indicates the reason for the loading lock.

Cancel collection

Collections that are created in cadis can be cancelled here.

Dissolve master pallets

For master pallets only: Master pallet is dissolved. The previously assigned packages are now no longer assigned to a master pallet.

Close master pallet

For master pallets only: Master pallet is closed, further processing (adding, removing packages) is not possible.

Acknowledge failed shipment import

Acknowledges one of shipment imports that has been rejected by cadis (see above Failed shipment import filter)

Refuse

Refuse collections from other depots.

Change priority

Assign a new priority to the shipment by entering a number here.

Print

Print dangerous goods attachment, delivery note, collection order, temperature report, details with/without notes.

Opens the corresponding document in the PDF reader. From here you can print or save it.
