cadis 2025.1.10

Not yet released

Improvements

General

  • [RD-2991] [cadisAPP] When the scanner is activated, the application first checks if the scanning service is available. If the service is not available, the application will wait for a predefined timeout period. If the scanning service remains unavailable after this period, the application will proceed to initiate scanning using the device’s camera as a fallback method.The application will not be blocked at startup until DataWedge is ready.

  • [RD-3462] [Improvement] Improved error handling in workflow-timer: do not stop on the first exception.

  • [RD-3496] [cadisAPP] CadisAPP now supports Honeywell CT37 and CT47.

  • [RD-3527] [Improvement] Set missing application/x-www-form-urlencoded Content-Type header for oidc token call.

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-3473] [Business code] Don’t create monitoring event when cadisSTUDIO accidentally requests tour template names with out weekdays and fix cadisSTUDIO so this does not happen anymore.

Bug fixes

General

  • [RD-3478] [Improvement] Previously, some table columns were renamed internally without updating the corresponding references in stored user preferences. As a result, user preferences related to those columns could not be restored correctly. This issue has now been resolved.

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-3453] [Business code] Resolved an issue when changing trip details.

  • [RD-3460] [Business code] Improved performance when receiving multiple accepted collection states from mobile devices.

  • [RD-3486] [cadisCONTROL] Resolved an issue with the search field above the tables finding too many results.

  • [RD-3488] [Reports] The weight and freight weight were both corrected on the delivery list with signature.

  • [RD-3459] [Business code] Shipments (mainly collections) with only cancelled packages will now be treated as if they had no packages initially. The main difference is that they remain planned/dispatched at their current trip.

  • [RD-3482] [cadisAPP] Resolved an issue in manual data synchronization if more than two tour topics are subscribed.

  • [RD-3495] [Business code] Added database changeset to restore lost freight carrier data (tour/departure table) to prior migration.

  • [RD-3536] [Reports] Delivery list with signature: Total weight (rounding discrepancy corrected).

  • [RD-3537] [Reports] Delivery list: “Prio” is displayed; the amount is empty if there is actually nothing there.

Released on February 25th 2025

Improvements

General

  • [RD-3126] [cadisCONTROL] After the action “Send collection to driver again” the mask is updated and the menu is no longer active.

  • [RD-3156] [cadisCONTROL] The context menu options “Load Item,” “Item Not Loaded,” and “Re-dispatch” are disabled for shipments that have been revoked from the dispatch. The dispatch validation check “There is already another shipment with the same stop ID on departure XY” will no longer be displayed.

  • [RD-3157] [Reports] Loading list: undispatched shipments does not appear in loading list.

  • [RD-3165] [Business code] Fixed an issue when using info scan under specific circumstances.

Temperature Control (TCL)

  • [RD-3147] [Reports] Temperature report: Fixed a CadisRuntimeException – inserted a check for division by zero.

  • [RD-3151] [Reports] There is no temperature report available for the package barcodes that were scanned using the MDE.

cadis 2025.1.2

Released on February 5th 2025

Improvements

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-2902] [Business code] If configuration is set to generate unloading lists for target depot after line haul loading, close in origin depot cadis will n0t send partial shipments anymore. Instead, every package will be sent within the main shipment. Furthermore, only loaded packages will be included as packages to be unloaded for the generated list. A new column showing the total package count of the shipment has been added to unloading detail’s shipment table (to distinguish between packages to be unloaded and pure shipment data).

Temperature Control (TCL)

  • [RD-2711] [cadisSTUDIO] The Tab “Terminal compartments” in the “Temperature control” has been improved. The agency list is sorted by agency name instead of ID by default. This enhancement makes navigation easier and improves usability by displaying agencies in a more intuitive order.

Bug fixes

General

  • [RD-2692] [Improvement] Improved the handling of missing properties in Variation JSON. The default value is obtained directly from the JSON schema generated from the Config classes used to create the dynamic user interface.

  • [RD-1741] [Performance] The library “leaflet” for displaying overlays on the map was broken. We are using the latest version – 1.9.4 now.

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-1862] [cadisSTUDIO] Vehicles in master data could be created with a name with more than 20 chars. This could result in exceptions because other places have restrictions of 20 chars for the vehicle name. Now the maximum length of vehicle names in master data is also 20 chars.

  • [RD-2160] [cadisSTUDIO] Tour stop: Fixed a possible runtime issue.

  • [RD-2721] [cadisSTUDIO] The actual storing position must not be longer than 20 characters. This is now enforced in cadisSTUDIO.

  • [RD-2959] [cadisSTUDIO] cadisSTUDIO showed all cadisSERVER errors as CADIS-1000 instead of displaying the correct error code.

  • [RD-2964] [cadisSTUDIO] Rules and templates -> Collection templates -> Create mask: for Shipper/Consignee, click the Geocoding icon – the mask is displayed in all languages.

  • [RD-2918] [cadisCONTROL] Undo dispatching in the sequence optimisation dialogue is possible for delivery stops, even if the dialogue is started from the tour monitor.

  • [RD-2970] [cadisCONTROL] Fixed an issue preventing the user from opening trip details after creating a partial shipment for a partially loaded shipment on the affected loading list.

  • [RD-2698] [cadisAPP] Enhanced the cadisAPP to validate telematics providers scanned from barcodes. If an unknown or unsupported provider is detected, an error message will be triggered, ensuring that only valid providers are processed. This update helps improve scanning accuracy and system reliability.

  • [RD-2963] [cadisAPP] Fixed an issue where the status could not be set correctly via the context menu while creating a master pallet in the cadisAPP. This update ensures that users can now reliably select and apply the desired status during the pallet creation process.

  • [RD-2977] [cadisAPP] Improved the unloading process by ensuring that the “Edit collections” option in the context menu in the cadisAPP is only displayed when configured. If the feature is not configured, the option will no longer appear, streamlining the user experience and preventing unnecessary options from being shown.

  • [RD-2955] [Reports] Dangerous goods attachment: For limited quantities (isLimitedQuantitys = true) the marking “ADR certificate and identification” and “The vehicle must be marked” are not set.

  • [RD-2931] [cadisCONTROL] Fixed an issue where the first inbound time of a dynamically scanned package was increased by scanning any following package (only occurred at unloading lists of handling type “STANDARD”).

  • [RD-2997] [cadisCONTROL] Fixed a rare issue when trying to save shipment data changes.

Temperature Control (TCL)

  • [RD-2699] [Improvement] During loading, the temperature sensor (assembled from PROVIDER – DATALOGGER – SENSOR) is set as the “actualStoringPosition” for the scanned package. This storing position is limited to 50 characters. If this assembled name becomes too long, then storing position is truncated.

  • [RD-2700] [cadisSTUDIO] Introduce an improvement to the sorting behaviour of the temperature in the sensor overview in cadisSTUDIO by handling empty values correctly. Now, when comparing values, empty values are treated as smaller, ensuring more consistent and reliable results when sorting data.
