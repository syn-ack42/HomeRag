Pick Up and Delivery (PUD) and Terminal Handling (TH)

New features

  • None

Improvements

  • [RD-2145] [cadisSTUDIO] Improved logic for displaying "Top Product" and priority information at stop level.

  • [RD-3712] [cadisCONTROL] Automatic dispatching of LocalTraffic shipments is now possible in the shipment overview. Shipments can be automatically assigned to departures based on their execution, delivery, or pickup date, even if the transport departs the night before.

  • [RD-3765] [cadisSTUDIO] Drivers and carriers are now treated equally across both transport types. This means you can choose whether to link them or not. If they are not linked, you can either enter the information freely or select it from the master data. This functionality is currently available for tour templates, tour creation, and when changing a vehicle on a tour in "Ready for trip start" status via the context menu. Tours generated from tour templates are also created correctly.
    Carrier: The carrier associated with the vehicle is always displayed, regardless of the transport type. It cannot be edited.
    Driver (Local transport): Only selection from the master data is supported. A driver entered as free text will not be displayed; instead, the default driver will appear.
    Driver (Long-distance transport): Only free-text entry is supported. Manual selection from the master data or linking to a vehicle is not available.

  • [RD-4358] [cadisSTUDIO] Create Shipment: Added a new “Save” button that redirects the user back to the Shipment Overview page.

Bug fixes

  • [RD-3901] [cadisSTUDIO] The duplicate display of the stop object's "priority" property has been resolved.

  • [RD-4115] [cadisSTUDIO] Status details mask created (usable in multiple places - e.g. shipments, unloading, inventory).

  • [RD-4137] [cadisSTUDIO] When a user clicks on a GTP Notification filter "Pending notes" or "To be routed", this filter persists even after a browser refresh: clicking a GTP Notification ensures that the corresponding filtered view remains active when the page is reloaded, providing a seamless and consistent user experience.

  • [RD-4215] [Business code] Free tour optimization with PTV2 now supports tours with only direct traffic. Enhanced the free tour optimization process using PTV2 to also work for tours that consist exclusively of direct (non-stop) traffic. The optimization engine now processes such tours correctly, improving flexibility for various transport scenarios.

  • [RD-4240] [Business code] Corrected master pallet handling when loading is recorded before unloading.

  • [RD-4331] [cadisSTUDIO] Create Collection: Delivery date and time are no longer required when selecting a product.

  • [RD-4380] [cadisSTUDIO] When trying to change the vehicle of a tour in My trips, the "Edit trip" dialog behaved incorrectly if the action was accessed via the "more tools" (three dots) menu (e.g. in smaller browser windows). The dialog would close immediately after selecting or removing a vehicle, without saving the change. This is now resolved: the dialog remains open and changes can be saved or cancelled as expected, regardless of how the action is accessed.

  • [RD-4418, RD-4419] [cadisSTUDIO] The "Geocode shipment" function is also available for shipments that have already been geocoded.

  • [RD-4496] [cadisSTUDIO] Resolved an inconsistent calculation of dangerousGoodsEquipment in all views and for all generated trips.

  • [RD-4526] [cadisSTUDIO] Increased the maximum map zoom level to see the house number.

  • [RD-4519] [cadisSTUDIO] Shipments table: The “Not Temperature-Controlled” icon is no longer displayed in the "Shipment Properties" column.

 Alpha features 

Order details

  • [RD-3534] [cadisSTUDIO] Order details can now be accessed via the context menu in the Orders table. The Order Details screen provides a read-only, comprehensive view of specific shipping operations. Contains information:

    • Order Information: Order type, Order ID, Order Number, Customer Numbers, Delivery List Number, Vehicle, Tour, Status, Shipper and Consignee information, Shipping Details (Product category, pickup/delivery dates and times, weight, loading meters, volume, cash on delivery, postage), Attributes (Tail lift, hazardous goods, empties, direct delivery, neutral shipper, load carrier exchange)

    • Tabs:

      • Items: Package details including quantity, packaging, content, weight, drawing number, remarks, and dimensions (length, width, height)

      • Instructions/Comments: Driver and internal comments displayed in a structured table format

      • Collection/Delivery Details: Order number, reference number, commission number, packages per order, collected packages, damaged packages, not collected packages, scanned packages, photos, goods receipt number, waiting time, status time, signatory, signature type, GPS position

      • Map: Visual representation of the order location on map

      • History: Complete audit trail including: what changed, change details, who made the change, vehicle, status time, entry timestamp, import/export file

      • Packages: Package number, status, status time, scan mode

      • Load Carriers: Load carrier type, target quantity, actual quantity, to customer, from customer, not exchanged, reason not exchanged, remarks

      • Photos: Image documentation of the order 9. Additional Services: Service ID, completed status, start time, end time, quantity

    • Actions:

      • Set Status- Opens "PUD Status" dialog for setting order status (already implemented)

      • Print Actions: Print order, print delivery note, print photo documentation

      • Receipt Signatory: Edit signatory information for sender block (name and address) with an editable table showing signatory name and last usage.

    • Includes Save and Delete buttons.

Master pallet details

  • [RD-3535] [cadisSTUDIO] Master pallet details accessible via context menu in shipment overview. Master pallet details can now be accessed via the "Sheet" option in the context menu from the shipment overview. The detail view uses the same format and structure as the shipment details, and includes a dedicated tab displaying specific information for both master pallets and their associated shipments. Additional table listing all shipments on the master pallet with context menu options for "Status details" and "Shipment details". Photo section included. Statistics block shows details for shipments and packages fully or partially contained on the master pallet.

  • [RD-3559] [cadisSTUDIO] Tourmonitor: The tour table in Tourmonitor has been created and its columns adjusted for better clarity and usability. Columns now present relevant tour data.

  • [RD-3574] [cadisSTUDIO] Tourmonitor:The stop list in Tourmonitor has been created and adapted.

    • Dispatch cancellation: cancel dispatch for shipments that were not collected.

    • Dispatch again: If a driver has declined a collection, this action enables the shipment to be reassigned to the same tour.

    • Stop Details: Opens the Stop Details screen, where all relevant information for the stop can be viewed. It is also possible to manually record or change the pickup or delivery status.

Additionally, manage local transport tours and line tours, with the ability to create tours for specific date ranges in advance.

  • [RD-3579] [cadisSTUDIO] Tourmonitor: Stop Details Dialog. The non-editable Stop Details dialog has been implemented with four tabs:

    • Stop Details: Shows stop status, type, tour details, date, driver, carrier, stop details, stop address, event details (with icon/MDE values), and statistics (number of shipments, total weight, hazardous goods points).

    • Delivery: Table view for deliveries.

    • Collection: Table for collections — same fields as in the Delivery tab.

    • Geoposition: Displays the stop's geolocation.

Loading

  • [RD-4057] [cadisSTUDIO] Added “Loading” tile with new main sections "Last mile" and "Line haul".

    Within the loading area, two main operational sections are now available:

    • Last mile: For managing and monitoring local transport and final delivery processes.

    • Line haul: For controlling and tracking long-distance transport operations.

    The overview provides a compact yet detailed summary of all loadings for a selected day, supporting both monitoring of loading progress and tracking the current state of local transport trips.

    • Loading table displays all relevant data with detailed columns, tooltips, and column footers with summaries.

    • Additional explanations and progress indicators for shipments, packages, and weight.

    • Line haul: Standard filters such as execution date, trip status, cancelled values, and outbound route are now available in the "Loading" section for Line haul operations.

  • [RD-4060] [cadisSTUDIO] Context menu created for "Loading"

Unloading

  • [RD-4376] [cadisSTUDIO] Added “Unloading”: The unloading overview can be accessed directly via the corresponding icon on the cadisSTUDIO tile. After opening, you receive a compact yet detailed overview of all unloadings for a selected day. The unloading overview displays unloadings in a summary format with the ability to monitor unloading progress.

  • [RD-4379] [cadisSTUDIO] Filters implemented for "Unoading". A standard filter similar to other tables is used. The available filter options include:

    • Date Selection: Selection of the desired day for which unloadings should be displayed.

    • Input Relation: Selection of the specific tour or route to which the display should refer.

    • Traffic Type: All, Local Traffic, Long-Distance Traffic

    • Status: Filtering by the current status of the unloading. The status values differ as follows:
        - All
        - Created
        - Released (for unloading)
        - In Progress (unloading has started)
        - Completed (warehouse employee has reported the end of unloading)
        - Finalized (dispatcher has finalized the unloading)

  • [RD-4381] [cadisSTUDIO] Context menu created for "Unoading". A right-click on a row opens the context menu with the following options:

    • Details - Direct access to the detail view of an unloading, can alternatively be performed with a double-click on the unloading.

    • Release unloading - This allows you to finalize the unloading after a warehouse employee has reported the end of unloading.

    • Close unloading - This allows you to finalize the unloading after a warehouse employee has reported the end of unloading.

    • Print unloading report - The unloading report contains a summary of the unloading performed.

    • Print start label - With the start label, the warehouse employee can register for the unloading.

    • Print dangerous goods attachment - The dangerous goods attachment contains information about dangerous materials included in the unloading.
