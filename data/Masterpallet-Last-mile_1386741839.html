CADIS Knowledge Base : Masterpallet Last mile

Created by Mohamed.Shebl, last modified by Julian Beyer on Sep 08, 2025

General

The masterpalletlast mile allows the creation of a masterpallet, in which any number of consignees can be integrated. The traffic type "last mile" is the only criteria that has to be fullfilled, so that a package generally can be added to this type of masterpallet.

With this type a simplified loading process is possible, for example in the case of a temperature controlled masterpallet. Creation can take place before the actual loading and the loading itself is documented with a single scan.

Required licenses and permissions

License: Manage master pallet for last mile

Permissions:

  • entryMDEMasterPalletLastMile

  • newMDEMasterPalletLastMile

Creation of a masterpallet last mile

To create a masterpallet last mile the user needs the license and the permissions mentioned above and additionally the permission "useMasterPalletGUI".

When creating a masterpallet on the MDE the user has to select the type last mile.

After selecting the type of last mile the field for the target depot is deactivated.

The storing position does not necessarily have to be filled.

After entering and saving all required data (barcode, type) the masterpallet is created.

A MP last mile can only be created for last mile processes.

Adding a package to a masterpallet last mile

So that a package is allowed to be added to the masterpallet, it has to be a package of a last mile shipment. The masterpallet itself and also the shipment can be dispatched as well as they can be undispatched. So it must be ensured that the route of the masterpallet and the trip of the package match.

Case 1: MP is dispatched to a trip

  • A package is allowed to be added to the MP, if the associated shipment is already dispatched on the trip or if the associated shipment is not yet dispatched (than the shipment gets dispatched on the trip automatically).

  • A package is not allowed to be added to the MP, if the associated shipment is already dispatched to another trip. (Error message: “The shipment is already dispatched to another trip.”)

Case 2: MP is not dispatched

  • A package is allowed to be added to the MP, if the associated shipment is not yet dispatched.

  • As soon as the first “dispatched package” is added to the MP, the masterpallet gets dispatched to this trip automatically and the rules of case 1 apply.

Expanded search

The MP search is expanded by the type of last mile.

Display of the masterpallet last mile

The icon for the MP last mile is displayed in all relevant places in cadisAPP, cadisCONTROL and cadisSTUDIO.

Loading / Dissolving / Delivery of a masterpallet last mile

The masterpallet last mile loaded by one scan. As soon as the loading is closed, the masterpallet is considered dissolved and the packages have to be processed individually in delivery.
