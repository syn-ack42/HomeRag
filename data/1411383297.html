General

The map can be accessed via the "First & Last Mile" tab "Map" on the cadisSTUDIO tile.

The local transport vehicles of the depot are responsible for conducting local transport tours. To support the drivers, a mobile device is used as the central tool for digital order processing. Through this device, drivers receive real-time access to all relevant tour data, including individual delivery and pickup orders assigned to them for the respective tour. The processing and status of orders are documented directly on the device and reported back to the central system.

During the entire period between driver login and logout on the mobile device, GPS position data is transmitted to the cadisSERVER at regular intervals. This transmission enables current tracking of vehicle positions. The most recently transmitted position of each vehicle is visualized in the digital map view of the system, allowing dispatchers to track the current location of each vehicle at any time.

The individual orders assigned to a tour have geo coded address data. These geo positions are also displayed in the map view.

The map shows the current position of all depot vehicles for the current day. (https://cadissoftware.atlassian.net/wiki/spaces/kb/pages/1403224102/Vehicles?atlOrigin=eyJpIjoiOWNkZjY0OTJlMGM2NDRmYWJjMTlhMzRhNThkNzY3MzAiLCJwIjoiYyJ9 )

The system allows targeted selection of individual vehicles to view detailed information about that vehicle or perform specific actions.

Additionally, there is the option to optionally display the positions of vehicle stops - i.e., the addresses of pickup and delivery orders for any given day. See (https://cadissoftware.atlassian.net/wiki/spaces/kb/pages/1395523585/Orders?atlOrigin=eyJpIjoiZTdmZjFlOTU3Y2RlNGE0MWE2N2MzYTFmYjI2YjM3ZjciLCJwIjoiYyJ9)

The map optionally provides traffic information. Any address can be marked on the map by placing a pin.

Filters

Data Query Filters

In the left, basic filter area, there are various filter options to search for specific information.

Available filter options:

  • Vehicle Filters – With this filter, specific vehicles can be targeted, for example by date, vehicle type, vehicle name, or status. This maintains an overview of the entire vehicle fleet and relevant vehicles can be quickly found and analyzed.

    The vehicle filter interface displays the current number of vehicles and provides multiple filter options:

    • Date Filter – Allows selection of a specific date to filter vehicles active on that date

    • Vehicle Filter – Enables selection of individual vehicles from a drop down list

    • Type of Vehicle Filter – Filters vehicles by their vehicle type category

    • Vehicle Group Filter – Allows filtering by predefined vehicle groups

    • Radio Status Filter – Filters vehicles based on their current radio/communication status

    Each filter field includes intuitive icons: calendar icon for date selection, drop down arrow for list selection, blue funnel icon indicating active filters, clear (X) icon to reset individual filter values, trash can icon to completely remove a filter.

    The interface shows a "Select filter to add" option when no filters are active, allowing users to build complex filter combinations.

  • Order Filters – This allows existing orders to be filtered by various criteria, such as order type (pickup, delivery), order status, or attributes (priority, dangerous goods, empty containers, time-critical, etc.). This enables quick identification of relevant orders.

    The order filter interface displays the current number of orders and provides multiple filter options:

    • Orders Filter – Allows selection of specific orders from a drop down list

    • Status Filter – Filters orders by their current status (e.g., pending, in progress, completed)

    • Attribute Filter – Enables filtering by order attributes such as priority, dangerous goods, empty containers, or time-critical requirements

Multiple filters can be activated simultaneously to increase hit accuracy. Filter settings can be reset at any time to return to the complete data view. When filters are active, a visual indicator is displayed so you can always see which filters are currently effective.

Map Area

Map

The map shows all vehicles and associated orders according to the currently selected filter. The color coding of vehicle symbols on the map indicates the currency status of the last transmitted GPS position:

  • Yellow Icons – Indicate vehicles with current position data. These vehicles have successfully transmitted position data to the cadisSERVER within the configured time window (default 20 minutes).

  • Gray Icons – Represent vehicles with outdated position data. The last position transmission is outside the defined currency time window (default: >20 minutes). This may indicate connection problems with the mobile device or system failures.

  • Black Icons – Mark the currently selected vehicle in the user interface, regardless of the age of the last position transmission.

The map display is limited exclusively to position data of the current business day, whereby historical positions are automatically hidden and a current daily overview is ensured.

Orders can be hidden in the map view by activating the "Hide Orders" checkbox. In this mode, only vehicles are displayed on the map.

After selecting a specific vehicle, a quick view with the most important vehicle details appears on the right side.

To display the orders for the corresponding vehicle, the filter must be adjusted. To do this, the desired vehicle must be selected and the checkbox for "Hide Orders" must be removed. The number of corresponding orders is displayed in the overview. This order quantity can be further restricted by applying filters based on specific order properties.

The map automatically groups orders when they are too close together at the current zoom level. A group is displayed as a circle with a number - the number shows how many orders are in the group.

When clicking on a group, the individual orders are displayed separately. The map section and zoom level may change slightly.

Orders at exactly the same address are arranged in a star shape around the actual position when clicked. This allows all orders to be selected individually, even if they are at the same location.

This function ensures that the map remains clear when many orders are present. Individual orders can still be viewed precisely at any time.

A click on an order opens the quick view in the right area, which displays the relevant order properties and available actions for the selected order.

The selected symbol of an order is highlighted in the map view by black coloring.

Map Options

In the upper right of the map view is the settings menu for map display.

  • Show Details – Extended map display with additional information elements: detailed business and address designations.

  • Display track – Visualizes the planned routes of local transport vehicles on the map. By activating this function, the calculated routes between individual order positions are displayed as line courses.

  • Traffic incidents – Various traffic incidents symbols. The positioning of the symbols corresponds to the exact geographic coordinates of the reported incidents. The variety of symbols enables differentiated representation of various incidents types and severity levels. Traffic incidents are updated in real-time and influence the route planning of local transport vehicles.

The traffic incidents symbols on the map offer extended information options by directly clicking on the corresponding warning symbols. When a incident symbol is activated, a detail window opens with specific information about the respective traffic impairment. The detail information for traffic incidents is provided exclusively in English.

Available detail information:

  • Type of traffic incident (traffic jam, construction site, accident, closure)

  • Affected road sections or intersections

  • Estimated duration of the impairment

  • Recommended alternative routes

  • Currency status of the disruption report

  • Hide Orders – Enables selective deactivation of order positions in the map display. This option temporarily removes the geocoded delivery and pickup orders from the map visualization.

  • Auto Zoom for All – Dynamically adjusts the map view to display all active vehicles and orders simultaneously in the visible map area.

  • Search Address - Opens a specialized search mask for geographic localization of addresses.

Address Search Function

The address search is activated via the dialog in the upper right map area. After opening the address input mask, complete addresses or address fragments can be entered in the search fields.

During input, the system automatically generates a hit list based on the geo database. The found addresses are displayed in tabular form with complete address information.

After selecting an address from the hit list and adopting the search result, the corresponding geographic position is visualized by a pin marking on the map. The map automatically centers on the chosen address position.

The pin marking can be removed by directly clicking on it, whereby the map returns to the original view.
