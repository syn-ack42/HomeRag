It is possible to create a partial shipment via the cadisAPP.

Preparation

Note: You may not have the necessary rights to edit the permissions and configuration.

Permissions

The cadisAPP User needs the following permission to create a partial shipment:

  • createPartialShipment

Steps to set the permission:

  1. Open cadisSTUDIO

  2. Go to “User & devices”

  3. Select “User roles”

  4. Select the appropriate role (e.g. Driver)

  5. Select the permission “createPartialShipment”

  6. Click “Save” button

Configuration

The functionality also requires a proper configuration.

Steps to set the configuration:

  1. Open cadisSTUDIO

  2. Go to “Administration”

  3. Select “Configuration Manager”

  4. Select “Terminal Handling / Item general”

  5. Choose

    • “Manual creation of partial shipment for line haul”
      or

    • “Manual creation of partial shipment for local traffic”

  6. Click “Save” button and “publish” the configuration

Flow

While loading

Note: Conditions for the creation of a partial shipment:

  1. Package is “onTerminal” and not loaded

  2. The loading list on which the original shipment is dispatched is completed

  3. The selected loading list is “dynamic”

The usual procedure for creating a partial shipment while loading via the cadisAPP is as follows:

  1. Select a loading list

  2. Scan the package that has been left behind

  3. The user is asked (if the conditions for creating a partial shipment are met) whether a partial shipment should be created

  4. Answer the question with “Yes”

  5. The partial shipment is created and the package is loaded

While building a master pallet

Note: Conditions for the creation of a partial shipment:

  1. Package is “onTerminal” and not loaded

  2. The loading list on which the original shipment is dispatched is completed

The usual procedure for creating a partial shipment while creating a master pallet via the cadisAPP is as follows:

  1. Select or create a master pallet

  2. Scan the package that has been left behind

  3. The user is asked (if the conditions for creating a partial shipment are met) whether a partial shipment should be created (see below screenshot)

  4. Answer the question with “Yes”

  5. The partial shipment is created and the package is added to the master pallet

While loading a master pallet

Note: Conditions for the creation of a partial shipment:

  1. Package is “onTerminal” and not loaded

  2. The loading list on which the original shipment is dispatched is completed

  3. The selected loading list is “dynamic”

  4. The package has been grouped in a Master pallet

The usual procedure for creating a partial shipment while creating a master pallet via the cadisAPP is as follows:

  1. Select or create a master pallet called for ex “MPA” in which you scan some of a shipment’s packages. Close the Master Pallet “MPA”

  2. Select or create a master pallet called for example “MPB” in which you scan the remaining shipment’s packages. Close the Master Pallet “MPB”

  3. Load on a departure-1 the MPA, close the loading

  4. Load on a departure-2 the MPB

  5. You will be asked (if the conditions for creating a partial shipment are met) whether a partial shipment should be created (see below screenshot)

  6. Answer the question with “Yes”

  7. The partial shipment is created and the MPB is loaded on departure-2

FAQ

Why can’t I load dynamically or dispatch a MP despite the fact that needed partial shipment were created?

Most often, the dispatching fails because of complex ties between different MPs, linked by the shipments spread in them. & some of those shipments are not dispatchable, because no partial shipments were created.
