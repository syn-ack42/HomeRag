Regeldefinition Übersicht

Die Definition von Regeln erreichen sie über das Icon :regeldefinition: in der cadis-Bedienleiste und dort über den Eintrag Regel-Definitionen (cadisSTUDIO).

cadisSTUDIO bedeutet, dass dieser Programmteil im Web-Browser bearbeitet wird. Es öffnet sich also der Web-Browser und Sie müssen sich erneut anmelden.

Nach der Anmeldung können Sie die Regeln für die verschiedenen Regelwerke bearbeiten:

Mit jeder dieser hier aufgeführten Regelwerke, können Sie die Bearbeitung von Abholungen und Sendungen beeinflussen. Dabei wird ein Regelwerk immer auf jede Abholung / Sendung einzeln angewendet, ohne Berücksichtigung anderer Sendungen.

Ausgeführt wird ein Regelwerk immer dann, wenn eine Abholung/Sendung neu angelegt oder verändert wird. Das geschieht entweder durch Import von einem anderen System oder über die cadis Bedienoberfläche.

Das Definieren von Regeln ist für alle Regelwerke gleich. Es steht eine große Auswahl von Sendungseigenschaften zur Verfügung, die sie zur Formulierung ihrer Regeln verwenden können.

Ergebnis ist jeweils ein Wert, der die weitere Abarbeitung der Abholung / Sendung in cadis steuern kann. Jedes Regelwerk hat einen anderen Typ Ergebniswert und eine andere Bedeutung für Behandlung von Sendungen in cadis.

Regelwerk

Ergebnis

Priorität

Das Ergebnis ist eine ganze Zahl für die Priorität. Welche Bedeutung dieser Wert hat und ob kleinere oder größer Zahlen höhere Priorität bedeuten, ist in cadis nicht festgelegt.   Sie können in der Sendungsübersicht nach der Priorität sortieren. Sie können aber die Priorität auch verwenden, um in anderen Regeln z.B. die Sendungsgruppe oder den Stellplatz zu bestimmen oder die Disposition zu steuern.

Sendungsgruppe

Das Ergebnis ist die Bezeichnung einer Sendungsgruppe (Text). Sie können die Sendungsgruppen frei benennen. Welche Bedeutung Sendungsgruppen haben, legen Sie fest.
Sie können in der Sendungsübersicht nach Sendungsgruppen filtern und die Sendungen einer Sendungsgruppe gemeinsam bearbeiten, z.B. alle Sendungen einer Gruppe per Tourenoptimierung automatisch disponieren.

Sperrigkeit

Hiermit definieren Sie, wann eine Sendung als Sperrig markiert werden soll. Das Ergebnis ist entweder wahr (true) = sperrig oder falsch (false) = nicht sperrig.

Umschlaghallen-Stellplatz

Das Ergebnis ist ein Hallenstellplatz, den Sie aus den Stellplatzstammdaten auswählen oder ein Sonderstellplatz, den Sie frei benennen können. Wenn Sie einen Sonderstellplatz definiert haben, dann wird der reguläre Stellplatz ignoriert.

Disponieren

Hier wählen Sie als Ergebnis eine Regelabfahrt aus der Liste der definierten Regelabfahrten aus. Zusätzlich geben Sie an, ob automatisch auf diese Regelabfahrt disponiert werden soll, wenn die Sendung importiert wird (Autodispatch=wahr) oder ob die Disposition nur durch den Bediener über die Funktion „Autodispo“ erfolgen soll (Autodispatch=falsch).

Hinweistext

Als Ergebnis wählen Sie einen Hinweistext aus den Stammdaten + einen zusätzlichen freien Text.

Abholstatus-Optionen

Als Ergebnis wählen Sie eine der vorgegeben Abholstatuskonfigurationen, in denen mögliche Statuswerte, Verpflichtung zur Fotodokumentation usw. festgelegt sind.

Zustellstatus-Optionen

Als Ergebnis wählen Sie eine der vorgegeben Zustellstatuskonfigurationen, in denen mögliche Statuswerte, Verpflichtung zur Fotodokumentation usw. festgelegt sind.

Unterschriften-Optionen

Als Ergebnis wählen Sie eine der vorgegebenen Unterschriftsoptionen, z.B. Unterschrift auf Display, auf Original-Lieferschein, Ausweiskontrolle.

Anlegen der Regeln

Neue Regel anlegen

Durch Klick auf ein Regelwerk kommen Sie zum Definitionsfenster:

Beginnen Sie mit dem Hinzufügen einer Regel:

Es ist nun die Wenn-Bedingung für eine Regel eingetragen. Dies dient nur als Beispiel. Sie müssen die Regel nun verändern und an ihre Erfordernisse anpassen und außerdem einen Effektwert definieren. Das ist der Wert, der als Ergebnis benutzt wird, wenn die Bedingung zutrifft.

Im linken Bereich sehen Sie die kompletten Regeln, pro Zeile immer eine Bedingung und den Effektwert.

Im rechten Bereich definieren Sie die Regelzeile, bestehend aus Bedingung und Effektwert.

Sie wählen zunächst die gewünschte Sendungseigenschaft aus der Liste:

In dieser Liste können Sie suchen, indem sie einen Teil der gewünschten Sendungseigenschaft eingeben.

Wählen Sie dann den Vergleichsoperator. Der ist unterschiedlich, je nachdem, ob die Sendungseigenschaft eine Zahl, ein Text, ein Datum oder ein Wahrheitswert ist.

Anschließend geben Sie noch den Vergleichswert ein, ebenfalls unterschiedlich je nach Datentyp der Sendungseigenschaft. Optional können Sie noch einen Kommentar hinzufügen

Regel ändern

Wenn Sie im linken Bereich eine Zeile anklicken, so wird diese blau unterlegt. Im rechten Bereich können Sie die Definition dann ändern.

Mit dem Anfassersymbol

können Sie bei gedrückter Maustaste eine Regelzeile verschieben, die Reihenfolge verändern, eine Zeile einrücken oder ausrücken.

Exportieren / Importieren

Klicken Sie auf Exportieren, wenn Sie die komplette Regeldefinition in eine Datei sichern wollen, z.B. weil sie sich einen funktionierenden Zustand sichern möchten, bevor Sie Änderungen vornehmen. Mit Importieren können Sie die Regeldefinition aus einer Datei wieder einlesen.

Abarbeitung der Regel

Die Regeln werden in der Reihenfolge von oben nach unten abgearbeitet. Der Effektwert der ersten erfüllten Bedingung ist das Ergebnis der Berechnung. Die weiteren Regelzeilen werden dann nicht mehr ausgewertet, es sei denn, zu der Regel gibt es Unterregeln.

Gibt es bei erfüllter Bedingung Unterregelzeilen (eingerückte Zeilen), dann werden diese ausgewertet, um das Ergebnis zu ermitteln, auch hier wieder in der Reihenfolge von oben nach unten. Ist hier eine Bedingung erfüllt, dann ist der Effektwert das Ergebnis. Ist bei den Unterregeln keine Bedingung erfüllt, dann ist der Effektwert der übergeordneten Regel das Ergebnis. Ist dort keine Effektwert definiert, dann wird mit der nächsten Regel weitergeprüft.

Testen der Regeldefinition

Sie können die Regel testen, indem Sie mit einer Sendung simulieren, welches Ergebnis die Berechnung liefern würde. Das Ergebnis wird nur angezeigt aber nicht abgespeichert!

Dazu klicken Sie auf „Simulieren“. Das Fenster zur Eingabe einer Sendungsnummer öffnet sich.

Die Berechnung für die angegebene Sendung wird ausgeführt und die Zeile, die das Ergebnis liefert wird durch einen grünen Haken markiert: :gruenerHaken:

Andere Zeilen, bei denen die Bedingung ebenfalls erfüllt war, die aber nicht ausgewählt wurden, weil weiter oben im Regelwerk bereits einen Treffer gab, werden ebenfalls markiert: :graueAbzweigung:

Erklärung gibt es jeweils per Tooltip, wenn Sie mit dem Maus kurz auf dem Symbol verweilen.

Beispiele

Beispiel 1

Gefahrgut: ja
Abfall: ja
Empfänger-Ort: München
Empfänger-Land: DE
Empfänger-PLZ: 80133

Hier ist die erste Bedingung erfüllt und liefert damit das Ergebnis „DG“. Die anderen mit :graueAbzweigung: gekennzeichneten Bedingungen wären ebenfalls erfüllt, kommen aber nicht zu, tragen, weil nach der ersten erfüllten Bedingung die Auswertung abgebrochen wird.

Example 2

Gefahrgut: nein
Abfall: ja
Empfänger-Ort: München
Empfänger-Land: DE
Empfänger-PLZ: 80133

Hier ist „Abfall ist wahr“ erfüllt. Da es Unterbedingungen gibt, werden diese ausgewertet und da die Bedingung „Empfänger Ort ist München“ ebenfalls erfüllt ist, liefert die das Ergebnis „WASTE-MUC“.

Example 3

Gefahrgut: nein
Abfall: nein
Empfänger-Ort: Stuttgart
Empfänger-Land: DE
Empfänger-PLZ: 70045

Hier ist die Bedingung „Abfall ist wahr“ erfüllt und die Unterbedingung ist nicht erfüllt. Somit ist das Ergebnis „WASTE“.
