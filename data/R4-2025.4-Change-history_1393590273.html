cadis 2025.4.2

Released on September 8th 2025

Bug fixes

Master data

  • [RD-4097] [Business code] Resolved an issue where an org unit without its working days set in master data could cause CADIS-1000 errors.

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-4182] [PTV2] Added configuration for applied driving/working time regulation for PTV2 calculation to "cadis.ptv2". (cadis.PTV2.regulationType)

Temperature Control (TCL)

  • [RD-4100] [Reports] Electronic Proof of Temperature: No incorrect sensor information or sensors without temperature data.

cadis 2025.4.1

Released on July 28th 2025

Bug fixes

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-4077] [cadisAPP] Resolved an issue where the system time reverted to an incorrect past date.

  • [RD-4039] [cadisCONTROL] List of shipments: limit in the display of results is correct now.

  • [RD-4041] [Business code] Improved behaviour when cancelling a shipment from a master pallet.

cadis 2025.4.0

Released on July 17th 2025

General

New features

  • None

Improvements

  • [RD-2087] [cadisSTUDIO] Improve display of roles for external users (e.g., users from Azure).

  • [RD-2377] [cadisAPP] The languages in cadisAPP have been restricted to the default languages German, English and French. Additional languages may be available depending on server configuration.

  • [RD-3853] [Rules engine] Negative priority values are not allowed anymore in the rules editor.

Pick Up and Delivery (PUD) and Terminal Handling (TH)

New features

  • None

Improvements

  • [RD-2021] [Configuration, cadisAPP] Extend configuration to enable auto creation of partial shipment when creating a master pallet.

  • [RD-2967] [Business code] Saving a collection template now requires that all referenced stop locations and customers exist. Any invalid or missing references must be corrected or removed before the template can be saved.

  • [RD-3160] [cadisAPP, cadisCONTROL, cadisSTUDIO] The icons for Driver Avis and Top Product have been standardised across cadisCONTROL, cadisSTUDIO, and cadisAPP.

  • [RD-3724] [cadisCONTROL, Configuration] Added Departure Name in the Delivery note report (Requires configuration via cadisSTUDIO)

  • [RD-3878] [Business code] Priority will now be calculated for shipments with an execution date in the future.

  • [RD-3862] [cadisCONTROL] Updated shipment status terminology:

    • Shipments previously marked as "disqualified" will now show "OK" instead of "Loaded".

    • Shipments previously marked as "dispatchable" will now show "OK" instead of "Unloaded".

Bug fixes

  • [RD-2002] [cadisSTUDIO] My Trips: For routes with the status 'Loading released', a vehicle can be entered.

  • [RD-3884] [Business code] It is now forbidden to add a package to a master pallet if another package of the shipment is assigned to a master pallet that is dispatched to another trip/departure.

  • [RD-3817] [cadisAPP] Photos of loading and unloading are now reliably preserved even if transmission errors to the server occur. This ensures that no images are lost during upload failures.

  • [RD-3902] [cadisSTUDIO] My Trips: It is now possible to change the trip and vehicle in cadisSTUDIO until the loading is closed. Changes are allowed in the following additional states: LOADING_RELEASED, LOADING_ACTIVE, LOADING_FINISHED, LOADING_CLOSINGREQ.

Temperature Control (TCL)

New features

  • None

Improvements

  • [RD-2257] [Business code] Introduced automated data clean-up for unused temperature sensors. A new timer task has been implemented to periodically remove data from unused temperature sensors.

  • [RD-2710] [Business code] Added informative messaging when a terminal compartment cannot be edited. Users will now see an info text explaining the reason for the restriction.

  • [RD-2482] [Reports] Electronic Proof of Temperature: Print day labels even if graph contains more than 11 days.

  • [RD-3787] [cadisAPP] Temperature compartment APIs are now accessed only when the required permissions are granted.

  • [RD-3812] [Reports] Electronic Proof of Temperature: the layout has been modified to improve clarity and usability. The following field details have been added:

    • Depot Name: Now displayed after the depot code.

    • Data Logger Name: Included for better traceability.

    • Unloading (Terminal Compartment): Data Logger information is now displayed using the same Data Logger–Sensor formatting as used for Loading.

    • Monitoring Details: Date, time, and duration of the monitoring process are now included.
