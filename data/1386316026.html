CADIS Knowledge Base : How to optimize sequence – cadisSTUDIO

Created by Mohamed.Shebl, last modified by Julian Beyer on Aug 19, 2025

Available as of release R4.21.1

General

Sequence optimization optimizes the route of a first&last mile trip based on optimization parameters while taking time restrictions into account.

You can optimize before loading to calculate the best loading sequence.
You can optimize the stops that have not yet been completed on the trip, for example after you have added additional collections.

Change sequence

Change Sequence opens a dialogue that allows you to select the individual stops using check boxes and change their position using the arrow keys. The modified sequence is applied to the trip when you press the “Apply” button.

Auto-optimize

The automatic optimization of first & last mile trips is available for trips for which the loading is not yet closed.

In First & Last Mile -> Planning, you can trigger the calculation of the optimal sequence via Auto-Optimize. The trip is automatically updated with the new sequence and planned arrival/departure times (also on the map).

Optimize (custom)

The manual sequence optimization function gives the opportunity to customize the optimization of the trip.

You reach the functions via First & Last mile -> Planning. Select a single trip with all stops geocoded.

A dialogue opens with the following information:

  • general information about the trip

  • stop table with planning relevant data about each stop

  • geo map showing stop sequence

  • planning results for the trip

    • Trip: effective optimization data

    • Optimization: optimized data

    • If available: Initial optimization: Resulting trip from function “Dispatch by trips optimization" Route optimization, if executed.

Optimization parameters on trip level

The header provides information about the trip (name, vehicle, vehicle profile) and the departure time/date.

  • Time and date can be configured and fixed

  • Vehicle profile comes from vehicle type and implies other optimization parameters

The following parameters are not configurable in the dialogue anymore since they are stored in the vehicle profile:

  • deprecated: Optimization objective (fastest/shortest route)

  • deprecated: Allow Road charges

Stop table

Overview

The stop table shows all planning relevant information about the stops.

Default optimization parameters are shown in the columns “Fixed”, “Time”, “PT”. They can be adjusted for the optimization service, as decribed below.

The colored columns respectively refer to the same-colored result block at the bottom. Arrival times (PTA) that do not meet the time window (i.e. are time critial) are highlighted.

Unplanned stops (after optimization) are arranged at the end.

Optimization parameters on stop level

Configurable for all stops except depot and break stops are:

Fixed position

The position for this stop must be fixed for optimization. Default: false.
If a stop position is marked as fixed, the optimization result is calculated by two separate optimization parts:

  • the sequence of all stops before the fixed ones will optimized

  • the fixed stop then follows in the sequence

  • thereafter, the sequence of all stops after the fixed one will be optimized

Time

Time window of the stop must be observed for optimization. Default: true.

Stop sequence

The desired sequence of stops can be changed by drag&drop. Hold the draggable area in the cell of the column “No.” and drag it to the desired position. Dragging multiple rows is possible via checkbox selection. The new sequence is not yet persisted but considered for trip calculation or optimization (if additionally position fixed is used).

The resulting stop sequence must ensure that

  • depot stops are at first/last position

  • collection of a direct traffic comes (somewhere) before its corresponding delivery stop

  • already processed stops stay unchanged at the beginning of the tour

If any of the conditions is violated, the stop sequence will not be applied.

Processing time per stop

Available as of Release 2024.1

The column “Processing time” displays the value for the service time at the stop. This time is calculated from the values contained in a shipment / collection, otherwise predefined values are used.
The value can be overwritten by editing values in the table.

Result blocks

The result blocks show all planning information on trip level for comparison.

Trip

Effective optimization data which will be considered in the end. Trip calculates the trip planning data with PTV based on the given sequence.

Optimization

Results of the last route optimization by PTV. Optimization optimizes the stop sequence via PTV, based on the given parameters.

Initital optimization

Only displayed if the trip was calculated via dispatch by trips optimization, route Optimization, and shows the calculated result.

No action is possible.

Apply

Apply will apply all calculations and configurations in the trip.

How to optimize your trip

First check the optimization parameters and correct them if necessary.

Run the sequence optimization by clicking Optimize. The stops are now put into an optimal sequence according to your requirements, whereby all collection stops are generally placed after the delivery stops (except for direct deliveries).

If not all stops or time slots could be met you have the following options:

  • Adjust time of departure

  • Ignore time slots settings for individual stops by editing the Time column of the default parameters

  • Put individual stops in preferred position/order

  • Adjust processing times for individual stops

  • The calculated values (PTA) of the subsequent stops are no longer valid and are deleted.

  • Now calculate the values for your manually altered trip (Trip button).

  • The km, PTA, Departure, WZT values are recalculated for each stop.

  • You can then compare the overall result in the Trip block (Time of Arrival, Driving time, Distance) with the Optimization block. Make further adjustments if necessary or apply the result.
