cadis 2024.1.7

New and changed functions

Logistic Process Platform (LPP) – General functionality

  • [RD-2423] [cadisAPP] cadisAPP now supports the scanner models Honeywell CK62 and CK67.

Bug fixes

Terminal Handling (TH)

  • [RD-2386] [cadisSTUDIO] Fixed a crash in cadisSTUDIO MyTrips if a trip was selected after a trip was created.

  • [RD-2419] [cadisAPP] Load the resources bundles for cadisAPP only if changed.

  • [RD-2428] [Business code] Fixed some trip related statistic values (some undispatched shipments were included mistakenly).

  • [RD-2467] [Business code] Fixed an issue regarding loading close when updating to 2024.1.6 or higher.

  • [RD-2418] [Reports]  Dangerous Goods document: ADR certificate and identity verification and vehicle labelling requirement only if more than 1000 points or isHighConsequence = true.

cadis 2024.1.6

New and changed functions

Logistic Process Platform (LPP) – General functionality

  • [RD-2316] [Improvement] Stable distribution of cron tasks across cluster node.

Bug fixes

Logistic Process Platform (LPP) – General functionality

  • [RD-2293] [cadisAPP] Fixed a occasional error when scanning a barcode multiple times.

  • [RD-2178] [Logging] When mobile devices use outdated loading ids (older than data retention days), the monitoring will now classify this error as 1730 (INCORRECT_KEY) instead of 1000 (INTERNAL_ERROR).

  • [RD-2354] [Performance] Set user id to lowercase if id and name are case-insensitive identical. Prevent that external system ignore case of login name and return same as id.

  • [RD-2355] [Performance] Added possibility to log LDAP password exception for debugging.

Terminal Handling (TH)

  • [RD-1976] [Business code] Collection based package events (event table within shipment details) will now be ignored for unloading, since collections are not unloaded directly. Instead, collections share the package events of their referenced shipment.

  • [RD-2230] [Business code] Not loaded master pallets will now be undispatched properly on loading close. Furthermore, instead of keeping and flagging discrepant master pallets (contained shipments dispatched on different trips), cadis will now perform undispatch actions for every affected shipment to avoid conflicts as soon as possible.

  • [RD-2247] [Business code] Fixed a bug when displaying the timeline of a shipment (details view within cadisSTUDIO).

  • [RD-2275] [Business code] Master pallets will now be undispatched from their current trip after they have been dissolved. This does not affect prior grouped shipments since the master pallet is dissolved before the undispatch action.

  • [RD-2340] [Business code] Avoid Null pointer Exception while scanning a package (no stop match code) into a master pallet.

  • [RD-2347] [Business code] Fixed an issue where undispatching seemed not to work (only visually). Generally improved dispatching state visually within trip details. Furthermore, corrected number of expected shipments that sometimes led to failure on loading close.

  • [RD-2409] [Business code] Prevent empty packing type in loading equipments.

  • [RD-2331] [Rules engine] Rule Editor: “Required pallet spaces”: one entered, one displayed.

  • [RD-2104] [cadisSTUDIO] Several bugs in the disposition of shipments (MyTrips and First & Last Mile) fixed.
    – Info scan features (e.g. adding unknown packages) disabled during disposition.
    – Display of messages why a shipment cannot be dispatched.

  • [RD-1977] [cadisCONTROL] Shipment details, photos or order details, images can be enlarged.

  • [RD-2020] [cadisCONTROL] Unprocessed collections (neither collected nor collection hindrance) will now be undispatched when trip is finished.

  • [RD-2168] [cadisCONTROL] Load capacity and correct calculation are correctly displayed.

  • [RD-2169] [cadisCONTROL] Unloading details: Shipment is displayed during unloading.

  • [RD-2219] [cadisCONTROL] Collections without packages dispatched at unloading lists will no longer display wrong dispatching state ‘Cancelled’.

  • [RD-2283] [cadisAPP] Set package status to ‘handled’ if Set package status to delivered if it was scanned and the status was previously set to ‘missing’ in cadisAPP.

  • [RD-2294] [cadisAPP] Fixed a possible crash if Location Tracking is active and the cadisAPP is in Background.

  • [RD-2296] [cadisAPP] Fixed a multiple click on action button when finishing loading for a departure.

  • [RD-2297] [cadisAPP] Fixed an error when scanning storing position barcode while pushing the app to background.

  • [RD-2365] [cadisAPP] Bugfix: The documentation of free or remaining loading meters allows to enter 0, if the “is full” checkbox is hidden through configuration.

  • [RD-2109] [Reports] Removed the unreadable text in the history of printed orders.

  • [RD-2312] [Reports] Delivery list with signatures: Fixed an incorrect format of “Package scanned, %”.

Pick Up and Delivery (PUD)

  • [RD-2314] [cadisCONNECT] API /{branch}/loadings/details: Reverted unintended api change where the property “status” was renamed “terminalStatus” and a new property “status” (with different content) was added.
    The content of property “status” is now restored to it’s behaviour in R4.21, the erroneous “terminalStatus” has been removed.
    A new property “packageStatus” has been added.

Yard Manager (YM)

  • [RD-2204] [cadisCONTROL] New shunting order – LG-ID is a required field and is marked accordingly.
