CADIS Knowledge Base : How to handle shipments which are not planned to be dispatched

Created by Mohamed.Shebl, last modified by Julian Beyer on Sep 08, 2025

It is possible that shipment data appear in the system, although these shipments shall not or cannot be dispatched at the moment. In this case a dispatcher possibly wants to "park" the shipments to care about them later.

Several options are conceivable.

Option 1: Shipment is planned for today, but it shall be delivered in a second trip.

  • Open the context menu of the shipment in the shipment overview and select “Dispatching/Loading Lock”

  • Trips: Open the filter in the tab “to be dispatched”

  • Set “Loading Lock” on “red” (Move slider down)

  • The table only shows shipments without a loading lock

  • If you explicitly want to dispatch shipments with loading lock, the filter just has to be set on “green” (Move slider up)

  • If shipments with loading locks get dispatched, a warning appears which can be overruled.

Option 2: Shipment is planned for today, but it shall be delivered in the future.

If the shipment shall be delivered in the future and because of this it should not appear in the number of shipments to be dispatched, it is possible to change the execution date to a date in the future. The shipment automatically appears on the corresponding day, because the default period is until "today".

Option 3: Shipment is planned for today but shall not be dispatched at the moment

  • There´s the possibility to use the rules engine Dispatch to “park” such shipments on a trip.

  • Create a rule that makes sure, that shipments with a certain attribute are immediately dispatched to a trip by using the autodispo button.

Example: I want shipments that have to be notified get out of my dispatching focus.

  • Rules engine Dispatch: Shipments with attribute “Notification” shall be auto-dispatched to the trip named “AVIS-1”.

  • The dispatcher can care about these shipments later!
