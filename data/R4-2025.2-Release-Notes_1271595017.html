Introduction

This release note is based on cadis R4 2025.2.0

Please take a look at the R4 2025.2 Change history, which contains the full list of changes.

Many of the features listed here are available across all basic service cut levels. However, some may require an additional licence, while others may have varying accessibility or utility depending on service cut limitations and your specific use of cadis. In some cases, the availability of information and/or functionality within a feature may also depend on the service cut level and other licenced options. 

General

New Features

cadisAPP Compatibility

We’re excited to announce that cadisAPP now supports Android 15! This update enhances compatibility, delivering a smooth and seamless experience on the latest Android devices.

cadisSTUDIO Data Exports

Say goodbye to CSV exports—cadisSTUDIO now offers Excel exports for your tables, making data handling easier and more intuitive than ever.

“Closed Alpha” & “Closed Beta” Features

We’re thrilled to unveil our newest testing innovations designed exclusively for advanced users: Closed Alpha and Closed Beta features. Here’s what’s new:

Enhanced Testing Options

Users can be easily admitted to “Closed Alpha” and/or “Closed Beta” at any time by a CADIS representative. You can see the alpha/beta use status of people in your organization in the cadis user management.

Seamless Integration & Security

Managed entirely within CADIS, these features take effect immediately upon login without extra steps. Access is controlled through specific permissions: only system users can enable these testing features (only the cadis team).

Improvements

cadisSTUDIO

Enhanced Table Summaries

All tables now feature an automatic summary row that calculates numeric totals and counts logical (true) entries. Plus, the status bar displays the total number of rows and the last update time. Grouped rows dynamically show their own totals, and some tables (e.g. the order table) now include extra details for key statistics like selected versus total stop locations and weight.

Consistent Edited Perspective Indicator

The asterisk marking an edited perspective is now consistently aligned, and view names are italicised during modifications.

Enhanced Search Functionality

Find what you need faster: Use CTRL + SHIFT + F for our advanced CADIS search (CTRL + F for the standard browser search still works as normal).

STRG-Shift-F.mp4
Dynamic Browser Tab Updates

Stay oriented with our smart interface: The browser tab now automatically updates to reflect the active module and function, so you always know exactly where you are within CADIS.

Fixes in Table View Management

We’ve updated table views to offer a smoother experience:

  • Restored Default View: Fixed columns no longer linger when reverting to the default view.

  • Custom View Deletion: Removing a custom view now reliably resets the table to its default state.

  • Accurate Default Selection: The system accurately reflects default selections without mistakenly marking them as custom.

Seamless Navigation and State Preservation when navigation away from a table

When you navigate away from a view with a table and the return back to that table, your last applied filters, sorting, and column settings are fully restored, ensuring continuity in your workflow.

Streamlined Toolbar

Now tucked behind a ‘…’ button, the toolbar expands to reveal additional functions as needed, with a scroll bar ensuring easy access when many functions are available.

Improved Activity Monitoring

New “Last Login” and “Last Failed Login” columns have been integrated in the user management section, to help you better monitor user activity.

cadisCONTROL

Smart PDF Reader Detection

If a PDF reader isn’t installed on your system, our application now provides clear, informative feedback instead of an error message, guiding you to the solution you need.

Bug Fixes

cadisSTUDIO

Persistent Language Preferences

When you change your browser language, your selected language now remains active—even after browser updates. This enhancement ensures that your preferred language is always in place, delivering a consistent and user-friendly experience every time you log in.

Fix “Create Collection” Feature in cadisSTUDIO in Closed Alpha

The “Create Collection” feature in cadisSTUDIO was mistakenly available to some users before it was ready. It is now correctly assigned to “Closed Alpha” phase.

Pick Up and Delivery (PUD) and Terminal Handling (TH)

New Features

—None—

Improvements

cadisSTUDIO

Full-Detail Stop Documentation Display

The stop documentation screen now shows complete status and sub-status texts, giving you all the details you need at a glance.

Distinct Logos for Visual Clarity

We’ve introduced unique logos for “Arrival/Driver Pre-advice” and “Pre-advice,” ensuring each function is clearly differentiated.

Revamped Header Layout for Shipment & Package Details

The updated header layout now utilises the full width in shipment and package details, offering a cleaner, more structured display of information.

URLs in “Create Trip from Template” Response

When generating trips form templates cadis may respond with a list of trips that could not be generated because they already exist. Each entry is now a regular URL that allows you to open the URL in a new tab via right-click menu or the middle mouse button. This is useful for reviewing the already created trip 
(Tip: “Open link in new tab” or “middle click” is very useful standard browser feature that works on most hyperlinks on web pages – including cadisSTUDIO.)

Complete Timestamp Display in Mobile Overview

The full timestamp now appears when hovering over “Last Communication” in the mobile access overview, giving you precise timing details at your fingertips.

cadisCONTROL

Improved Outbound-Route Filter

Filter combo boxes for “Outbound Routes” now display only routes from the master data that were valid within the past 90 days (or more precisely: the configuration setting  cadis.data.retentionDays –  the number of day most non-master data is kept in cadis). This removes clutter from expired master data records while still allowing to conveniently filter for rouses used in the recent past.

cadisAPP

Immediate Feedback for Unknown Barcodes

When scanning an unknown barcode during the delivery process, a sharp sound alerts you instantly, ensuring quick detection and correction.

Interfaces

  • A new configuration parameter has been added to avoid imports of ctOutboundTransport failing certain dispatching checks 

  • A new configuration parameter has been added to add the full cTDepotItem “References” substructure in the OrderState Export (N.B. this is not change in the xsd as this structure was already present in the cTOrderState.xsd but only a subset of the references was previously exported)

Bug Fixes

cadisSTUDIO

Date Input Validation for MyTrips and Orders

We’ve resolved an issue where entering an invalid date (a five-character year) in STUDIO on Chrome would cause departures to vanish from the local transport loading screen and tour monitor in CONTROL. Now, date inputs in MyTrips and Orders are restricted to a proper four-character year, ensuring accurate and error-free displays.

Info Scan Storing Position Limit

For improved data consistency, the storing position in Info scan is now limited to 15 characters, preventing some downstream errors.

cadisCONTROL

“Truck Attributes” no longer displayed as “Traffic Incidents” with ptv2 maps

We’ve fixed with ptv2 maps where “Truck Attributes” (i.e. high restrictions) were also displayed when “Traffic Incidents” was activated. This lead to confusion as a “normal” height restriction looked very similar to a serious traffic incident.

Report “Control List Deliveries

Several values in the “Control List Deliveries” report have been corrected.

cadisAPP

Correct Signature Display in Delivery Stops

An issue where a previously taken photo was mistakenly shown as a signature when switching between apps during a delivery stop has been resolved. Now, cadisAPP correctly refreshes the signature, ensuring that you always see the intended image.

cadisCONNECT

Fix in API for Processed Orders

The public API for processed orders now uses documented stop coordinates (sent by mobile devices) instead of planned coordinates. 

Business Code

Error during loading when scanning shipments assigned to “hand over points”

We’ve fixed a “CADIS-1000” exception that happened when a package belonging to an item is scanned in loading while still assigned to a handover point.

Mobile Session Continuity when switching devices

Fixed an issue when switching mobile sessions between devices (i.e. moving trips from one cadisAPP to another to continue work).

Dynamic Reset of Expected Storing Position

After a shipment update, if no matching rule is found, the expected storing position now resets—either adopting the host-provided value or remaining empty. The cadisCONTROL set position will still take precedence.

Fixed issue with incorrect vehicle in unloading

Fixed an issue where a wrong vehicle could be assigned to an unloading list generated by cadis. 

Optimised Shipment Data Clean-up

We’ve improved the shipment data clean-up process, especially when master pallets are involved. This ensures that your records remain accurate.

Enhanced Loading/Unloading List Imports

Our list import process is now more robust: CADIS automatically creates dummy shipments for non-existing shipment host IDs, and dummy packages for new or unknown package host IDs added to existing shipments. This enhancement guarantees that your data imports are complete and consistent.

Temperature Control (TCL)

New Features

—None—

Improvements

cadisCONTROL

Enhanced Unloading Details with Thermometer Data

For cadisCONTROL users who need to verify that drivers have properly processed their collections, we’ve added the thermometer columns in the “Assigned Collections” tab.

Granular Temperature Monitoring History Logging

When documenting a collection during the unloading process, an history entry has been added at the package level. This enhancement delivers detailed logs to help you analyse the temperature tracking process with greater accuracy.

Reports

Refined Graph Legends for Electronic Proof of Temperature

Our electronic proof of temperature now comes with an updated graph legend that accurately reflects cases where a package is stored in two different compartments with separate sensors. This improvement ensures your data visualisation is both clear and reliable.

Technical Topics

New Features

—None—

Improvements

Server

Optimised Shipment Import Performance

Experience faster data processing: importing multiple shipments with numerous unknown packages is now significantly more performant, ensuring the system handles heavy loads with ease.

Advanced Asynchronous Task Distribution

Boost your system’s efficiency with enhanced asynchronous task distribution in clusters. This improvement increases scalability and ensures faster, more reliable processing across your network.

Streamlined Temperature Data Handling

To optimise operations, we have removed the TCL NAOCOM temperature data retrieval. This change reduces unnecessary overhead.

Real-Time Configuration Validation for Order Status

Prevent system corruption with our new real-time configuration checks. Order-level attribute config is now validated every time, ensuring that temporary misconfiguration don’t persist and impact your data.

Exception Handling Improvement for Unknown Message Exports

Our ESB now bypasses the CADIS-1729 Exception when exporting unknown, disabled, or removed message types.

Multi-Message Download and Re-send Option

Our latest update adds an option to download or re-send multiple messages directly from ESB/Search.

Monitoring

Enhanced Server Monitoring with Version Details

The new addition of server version details in monitoring entries and filters improves traceability and simplifies system diagnostics.

PTV

Weekly Pre-Calculated HPRs for Enhanced Route Optimisation

Optimise your routes with our new weekly pre-calculated HPRs for PTV2. These enhancements, paired with a renamed configuration parameter for easier management, deliver superior performance in route optimisation.

Configurable Traffic Incidents & Streamlined Map View

Gain greater control with a new configuration switch for PTV2 that makes traffic incidents configurable. Additionally, truck attributes have been streamlined out of the cadisSTUDIO map, resulting in a cleaner, more focused interface.

Default Traffic Information Enabled

ptv2.trafficInfoAvailable now defaults to “true,” ensuring that critical traffic information is automatically available for your optimisation needs.

Installer

Optional Sudo Usage for Non-Root Installations

Installation flexibility is here—our installer now supports optional sudo usage when running without root privileges, making the setup process more adaptable and secure.
