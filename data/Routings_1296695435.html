Principles & Access

You can reach the screen for creating Routings by using the master data overview (:regeldefinition: symbol in the cadis control panel) respectively via the master data area in cadisSTUDIO.

The Routing serves for detection of the last mile route / line haul route and automatic predispatching when a shipment is imported or updated.

A data set includes always a route id, a zip code range (from zip code, to zip code), a scope (Delivery/Pickup) and the country code.

The fields marked with an asterisk are mandatory.

A complementary role with your rules engine

The master data Routings help in setting an Outbound route for your shipments. They are complementary with the rules engine where you can select a departure according to your outbound route.

❶Your hosts system sends to cadis a shipment’s data with a postal code “1234” for example

❷The master data “Routings” have been defined to say postal codes between 1000 & 1500 get the Outbound route ID “North”.

❸The Rules Engine can be built to say “If Outbound route ID contains “North”, dispatch shipment to trip “34-01”

So, a shipment imported to cadis first gets it’s outbound route define by the master data routings before the rules engine decides of its dispatching to a departure.
