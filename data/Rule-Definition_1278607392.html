Rule Definition Overview

The Rule definition window is accessed by using the :regeldefinition: symbol in the cadis control panel or directly by using the Rule Definitions entry in cadisSTUDIO.

With each rule listed here (see screenshot above), you can affect the processing of collections and shipments. A set of rules is always applied individually to each collection/shipment, without taking other shipments into account.

A set of rules is always executed when a collection/shipment is newly created or changed. This is done either by importing from another system or via the cadis user interface.

The definition of rules is the same for all rule sets. There is a large selection of shipment properties available that you can use to formulate your rules.

Rule

Result

Priority

The result is an integer for the priority. The significance of this value and if smaller or larger numbers mean a higher priority is not defined in cadis. You can sort by priority in the shipment overview. However, you can also use the priority to determine the shipment group or the storing position in other rules, for example, or to control the dispatching.

Shipment Group

The result is the name of a shipment group (text). You can name the shipment groups as you wish. You can determine the significance of the shipment groups.
You can filter by shipment groups in the shipment overview and process the shipments of a shipment group together, e.g. automatically dispatch all shipments of a group using the route optimization tool.

Bulkiness

You define when a shipment is marked as bulky. The result is either true = bulky or false = not bulky.

Terminal Storing Position

The result is a terminal storing position that you can selected from the storing positions master data or a specific position that you can name freely.   If you have set a specific storing position on the terminal, the regular parking space is not considered.

Dispatch

This assigns a regular departure from the list of predefined regular departures. In addition, you can specify whether dispatching is carried out automatically for this regular departure when the shipment is imported (Autodispatch=true) or whether dispatching is carried out only by the operator using the “Autodispatch” function (Autodispatch=false).

Handling Instruction

As a result, you select text from the master data + adds additional free text

Collection Status Options

As a result, you select one of the predefined collection status configurations, in which possible status values, requirements for photo documentation, etc. are defined.

Delivery Status Options

As a result, you select one of the predefined delivery status configurations, in which possible status values, requirements for photo documentation, etc. are defined.

Signature Options

As a result, you select one of the predefined signature options, e.g. Signature on display, on original delivery note, ID check.

Create rules

Create new rule

Clicking on a set of rules takes you to the definition window

Start by adding a rule:

The “If Condition” for a rule is now selected. This only serves as an example. You must now change the rule and adapt it to your requirements and define an effect value. This is the value that is used as the result if the condition applies.

In the left-hand area you see the complete rules, there is always one condition and the effect value on each line.

In the right-hand area you define the rule line, consisting of a condition and the effect value.

You first select the desired shipment property from the list:

In the list you can search by entering part of the desired item property.

You then select the comparison operator. This differs depending on whether the shipment property is a number, text, date or a true/false value.

Afterwards, you enter the comparison value, which also differs depending on the data type of the shipment property. You can also add a comment.

Export / Import

Click on Export if you want to save the complete rule definition to a file, e.g. because you want to save a functioning status before applying changes. By clicking on Import you can read the rule definition from a file.

Processing the rules

The rules are processed from top to bottom. The effect value of the first applicable condition is the result of the calculation. The further rule lines are then no longer considered, unless there are sub-rules to the rule.

If there are sub-rule lines (indented lines) when a condition is fulfilled, then these are evaluated to determine the result, in sequence from top to bottom. If a condition is fulfilled here, then the effect value is the result. If no condition is fulfilled by the sub-rules the effect value of the superordinate rule is the result. If no effect value is defined there, then the test continues with the next rule.

Test the rules

You can test the rule by simulating with a shipment and see what the result of the calculation would be. The result is only displayed but not saved!

By clicking on "Simulate" you access the screen for entering a shipment number.

The calculation for the specified program is carried out and the line that displays the result is marked with a green tick: :gruenerHaken:

Other lines for which the condition was also met but which were not selected because there was already a match further up in the set of rules are also marked:

Conditions that are fulfilled but did not come into play because the higher-level condition were not fulfilled are marked with :graueAbzweigung:.

Explanations are provided in a tooltip if you briefly hover the mouse over the symbol.

Examples

Example 1

Dangerous Goods: yes (true)
Waste: yes (true)
Consignee City: München
Consignee Country: DE
Consignee PC: 80133

In this case, the first condition is fulfilled and so returns the "DG" result. The other conditions marked with :graueAbzweigung: would also be fulfilled, but do not apply because the evaluation is stopped after the first fulfilled condition.

Example 2

Dangerous Goods: false (no)
Waste: true (yes)
Consignee City: München
Consignee Country: DE
Consignee Postal Code: 80133

The "Contains waste is true" condition is met. Since there are sub conditions, that are evaluated and as the condition "Consignee city equals München" is also met, the result is "WASTE-MUC".

Example 3

Dangerous Goods: false (no)
Waste: true (yes)
Consignee City: Stuttgart
Consignee Country: DE
Consignee Postal Code: 70045

In this case, the condition "Contains waste is true" is fulfilled and the sub condition is not fulfilled. Thus the result is "WASTE”.
