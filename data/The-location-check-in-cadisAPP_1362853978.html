CADIS Knowledge Base : The location check in cadisAPP

You can enable & configure the GPS tracking of drivers on tour & follow up on it in cadis back-office.

Prerequisite: the ConfigManager

At opening of a stop

Case 1: GPS position is unknown

The mobile device does not send GPS data. The driver may be in a non covered zone, mobile data is interrupted… No configuration switch can solve this issue.

Case 2: GPS position is obsolete

cadisAPP checks the time stamp of last available GPS position vs. current time. The ConfigManager allows to extend the time stamp so that the alert does not pop-up too often. Check for “Geo location invalidation seconds” in the ConfigManager. By default it is set to 300 seconds.

Case 3: GPS position does not match

cadisAPP checks the radius that separates driver’s position vs stop location. The ConfigManager allows to extend that radius so that the alert does not pop-up too often. Check for “Geo fence enter radius meters”. By default, it is set to 150 meters.

At merging of stops

A driver can merge stops while on tour by opening them both for processing. In such case, the location check also applies.
