CADIS Knowledge Base : How do cadisSTUDIO Table Filters Work?

Created by Mohamed.Shebl, last modified by Julian Beyer on Sep 08, 2025

Tables in cadisSTUDIO have two types of filters: Data query filters and Quick column filters.

Data query filters

Data query filters apply to the data when it is retrieved from the database by the cadisSERVER. When a Data query filter is changed the data is always reloaded from the server. On many tables in cadis setting a Data query filter is mandatory to limit the amount of rows transferred to the cadisSTUDIO page.

Which columns can be filtered, and if there is a Data query filter at all, depends on the individual table and the use cases it serves.

The table footer in cadisSTUDIO always displays a row count in the Form of “Rows: XX of YY”. The “YY” number shows how many rows were found in the database with the Data query filter applied.

Data query filters are set via the tab on the left hand side of a table. You can add from the filters options by clicking on Add a filter and set the filters by choosing an option from the dropdown list. The settings for the filters will appear at the top of the tab. One filter may have multiple settings.
Once a Data query filter is applied it will show as a larger orange bubble above the table.

For each Data query filter you can click the “Struck out funnel” icon to empty the filter or click the “Recycling bin” icon to remove it completely. Some types of filters also provide a “Filled out funnel” symbol. If this is the case the filter has a pre-defined default (e.g. “today”). Re-set that default by clicking the “Filled out filter” button.

You can empty /remove/re-set the default of all filters at a time by clicking the icons above the filters.

 

If a Data query filter is mandatory for a table, but none is set, the filters that can serve as a mandatory filter are marked with a red star “*”. Select and configure one of these to load data from the server. While the table is missing a mandatory filter the data area will remain empty and display an error message.

In some cases there may still be too many rows to retrieve from the server. If this happens a message is displayed prompting you to set more filters. Either change the criteria on the existing Data query filters or add another Data query rilter.

Quick column filters

Quick column filters apply to the data after it has been retrieved from the cadisSERVER. When a Quick column filter is changed the data is not reloaded from the server.

All columns that are displayed in a table can be used in a Quick column filter. Check out his article on cadisSTUDIO table features to see how: https://cadissoftware.atlassian.net/wiki/x/E4CMUg

In the table footer in cadisSTUDIO always displays a row count in the form of “Rows: XX of YY”. The “XX” number shows how many rows are actually shown with the Quick column filter applied. The “YY” shows how many rows would eb shown if no quick filter is applied based on the result returned due to the Data query filter.
