Starting the optimization

Open the Route optimization screen

  • if you want to schedule all shipments and collections using the Route optimization using the :Tourenoptimierung: symbol in the cadis control panel

  • if you want to plan a selection of shipments and collections using route optimization by selecting the shipments to be planned in the shipment overview and calling the Start free route optimization with selection function from the context menu.

The route optimization plans shipments and collections on routes.

Set the parameters for the optimization in the Route optimization screen.

At maximum capacity it is recommended to specify less than 100% depending on the accuracy of the shipment data and the shipment structure.

For fleet use, you can choose whether as few routes as possible should be created or whether all routes should be created for all vehicles

In the Vehicle fleet area, enter the vehicles you want to use for this optimization. You have 2 options:

  1. Select from the list of vehicles and enter the individual vehicles. In this case, a vehicle is immediately assigned to the routes created by the optimization.

  1. Enter the number of vehicles for each vehicle type. In this case, you then assign the vehicles to the routes manually.

In the table, the :nbo: column shows the total number of vehicles with the same characteristics used for the optimization: payload, pallet storing locations, dangerous goods equipment and tail lift.

The number of available vehicles is displayed in the :verfügbareFahrzeuge: column (marked as available in the master data).

You can edit the :verwendenFahrezeug: column and enter the number of vehicles you want to use for this optimization.

After the optimization, the Used column shows you how many vehicles were actually used.

Start the optimization process by clicking the Start the optimization button. The optimization takes a few minutes depending on the number of shipments and runs in the background. You can start another optimization run during this time.

The optimization result is initially only displayed. Only when you are satisfied with the result you apply it and only then the concrete routes are created.

In this way you can, for example, have various scenarios calculated and then adopt the best result.

Overview of optimization runs

You can see the current and completed optimizations in the Route optimizations screen.

For each optimization, you see the start time, the duration of the optimization in minutes, the status (Calculation, Finished, Applied) and the result (number of routes, total driving time, and total distance).

You can open or delete the optimization using the context menu.

The Finished status means that the optimization run is done. By double clicking on a line you can view the result in detail.

Result of route optimization

Displays the result of the optimization in the Route optimization screen

Tour table (top left)

This shows a summary of the data for the routes created with a display of the capacity utilization for collections and deliveries.

Result Tab

The optimization parameters used are displayed here along with the vehicles actually used in the vehicle table.

The Orders section shows the scheduled and unscheduled collections and deliveries

The Routes section shows statistics on the routes created with minimum, maximum, average and total values.

Stops Tab

Shows the stops for the selected routes

Map

The itinerary of the selected routes are shown as a line on the map,

Apply optimization route

This modelling is not applied to your data until you click on the Apply optimization results button. At which point, the routes will be created and the shipments and collections assigned to them. You can then see the tours in the Loading of secondary distribution screen and in the List of routes screen.
