Introduction

This release note is based on cadis R4 2025.4.0

Please take a look at the R4 2025.4 Change history, which contains the full list of changes.

Many of the features listed here are available across all basic service cut levels. However, some may require an additional licence, while others may have varying accessibility or utility depending on service cut limitations and your specific use of cadis. In some cases, the availability of information and/or functionality within a feature may also depend on the service cut level and other licenced options. 

Highlighted Enhancements

Streamlined Loading Workflows in cadisAPP (RD-2021)

You can now configure cadisAPP to automatically create partial shipments without prompting the user during loading. It speeds up terminal operations, especially in high-volume hubs, by eliminating unnecessary confirmation dialogues. The setting is configurable per traffic type (Line Haul or Local Traffic).

Enhancements to ePOT Reports (RD-3812)

The Electronic Proof of Temperature (ePOT) report has been updated for clarity and usability. It now includes depot names, data logger names, monitoring details (with timestamps and durations), and improved layout elements. These updates ensure more readable, traceable, and audit-proof reports, even when spanning long monitoring periods.

General

Icons standardisation across platforms (RD-3160)

Unifies icons for key attributes like “Driver Arrival Notification” and “Top Product” across cadisCONTROL, cadisSTUDIO, and cadisAPP.

cadisAPP: Restrict default languages to DE/EN/FR (RD-2377)

cadisAPP now only includes German, English, and French by default. Other languages are available via server-side configuration, ensuring consistency and reducing confusion due to incomplete translations.

Rules Engine: Disallow negative priority values (RD-3853)

The priority rule editor now prevents negative values, aligning with internal semantics and external optimisation constraints (e.g. xTour integration).

cadisSTUDIO: fix display of roles for external users (RD-2087)

External users' roles are now displayed correctly in cadisSTUDIO, without confusing angle brackets suggesting a missing reference.

Pick Up and Delivery (PUD) & Terminal Handling (TH)

Partial shipment confirmation dialogue removed (configurable) (RD-2021)

See highlights. Adds a configuration option to suppress the confirmation prompt when cadisAPP needs to create a partial shipment. This improves scanning speed during loading or master pallet creation.

cadisSTUDIO: "Edit Trip", Changing vehicles (RD-3902, RD-2002)

In cadisSTUDIO, vehicle assignment was previously blocked for tours once they reached the "Loading released" status. This has now been corrected: users can assign or modify the vehicle for trips that are still in loading phases (e.g., LOADING_RELEASED, LOADING_ACTIVE, etc.). This ensures dispatchers can complete necessary configurations even after initial release.

Improved error handling during shipment import (RD-4041)

Improved error handling during shipment import from FTP. Files with unresolved shipment references no longer fail silently and are now processed correctly or rejected with meaningful feedback.

cadisCONTROL Delivery Note Report: Include Departure Name (Tour) (RD-3724)

Optionally adds the departure name to printed delivery notes. This can be toggled via cadisSTUDIO Config Manager.

Write “OK” instead of “Loaded” or “Unloaded” for disqualified/redispatched shipments (RD-3862)

The status text shown in shipment events has been corrected to "OK" for disqualified or redispatched shipments, reducing confusion for users analysing shipment histories.

Priority calculation for future shipments (RD-3878)

The priority recalculation process has been adjusted to include future-dated shipments and avoid incorrect exclusions based on outdated filtering logic.

Collection template / Stop Location ID validation (RD-2967)

When saving a collection template, the system now checks whether referenced stop locations and customers actually exist. Invalid or dangling references (e.g., deleted stops) will trigger a clear validation message, and saving is prevented until corrections are made. This prevents silent data corruption and improves robustness of planning templates.

Corrected missing master pallet package assignment validation (RD-3884)

Fixed an issue that failed to prevent the assigning of packages from the same shipment to master pallets that are dispatched to different departures which caused incidents and inconsistent states.

Photo in loading missing (RD-3817)

Fixed an issue where photos taken during loading or unloading could be lost if server communication failed. The app now retries the upload and provides appropriate error messages.

Fix delivery status with wrong date & time (RD-4077)

Resolved an issue in cadisAPP where device time resets caused shipment events to be recorded with incorrect timestamps. This led to delivery statuses appearing in the past and affected host system exports.

cadisCONTROL list of shipments: corrected confusing result limit display (RD-4039)

Fixed incorrect result limit display in the cadisCONTROL shipment list view when filtering by transport type.

Temperature Control (TCL)

TCL ePOT improvements: (RD-3812)

See highlights. TCL ePOT: layout clarified, depot name and data logger name added, unified formatting for unloading sections, and detailed monitoring information (start/end time, duration). This significantly improves usability, transparency, and regulatory compliance for temperature-sensitive shipments.

TCL ePOT: bad x-axis labelling when period too long (RD-2482)

Improved x-axis rendering in long-duration ePOT reports to avoid overlapping timestamps. Date labels are now placed more clearly and legibly across extended time frames.

TCL ePOT: prevent report issues if data logger name is missing (RD-3987)

Introduced a fallback mechanism that ensures reports still generate correctly even if the expected data logger name is missing.

Temperature sensor clean-up (RD-2257)

Introduced an automated clean-up job to remove inactive temperature sensors after a configurable retention period. This reduces database bloat and improves performance.

Terminal compartments: add info message when not editable (RD-2710)

Users now see a helpful message explaining why terminal compartments can’t be edited when they belong to another depot.
