CADIS Knowledge Base : Distribution of cadisCONTROL

Created by Julian Beyer on Oct 14, 2025

The installation and distribution of cadisCONTROL via Software Distribution Tools such as SCCM or Software Center is supported and allowed by CADIS GmbH. The installer is fully compatible with unattended (“silent”) installation modes, specifically for enterprise rollout scenarios.

The recommended approach is to:

  • Use the .exe installer with the -q flag (unattended mode).

  • Set the installation directory and other parameters via -V variables.

  • Optionally, use a response.varfile for standardized configurations.

Where to get the .exe installer?

The downloads are available at @PROTOCOL@://@SERVER@:@PORT@/@OPTIONAL@/cadis/control, for example: http://http://192.168.0.0:8080/cadis/control/

Click the Download button and save the file exactly as suggested by the server. Do not rename the file, as its name contains important information about the server URL.

Updates and Version Management

Proper version management and communication between CADIS and the software distribution administration is essential.

cadisCONTROL automatically checks for version compatibility with the connected server each time it is launched. If a mismatch is detected, the client prompts for an update and performs a reinstallation to align with the server version.

In locked-down customer environments, where end users do not have installation rights, it is the responsibility of the administrator of the internal software distribution system to manage and deploy updated cadisCONTROL versions.

When updates are applied to the server without a corresponding client rollout, affected users may be unable to start cadisCONTROL until the new client version is distributed.

To avoid service interruptions, IT administrators must ensure that client updates are rolled out in coordination with any server-side updates or hotfixes.

Variables and varfile

You can specify single variables with the syntax: -V<variablename>=<parameter>. After installation, you can find a sample of the response.varfile in the installation directory under: .install4j\response.varfile.

To change the default installation directory use: -VforceInstallDir=D:\cadisCONTROL.

The server URL is already encoded in the installer filename. If you need to override it manually, use: -VcadisProjectURL=<URL>.
