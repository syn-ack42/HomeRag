CADIS Knowledge Base : What do the different dispatching status mean?

Created by Mohamed.Shebl, last modified by Julian Beyer on Sep 08, 2025

Shipment / delivery

To be dispatched:

  • Initial status after shipment import (as item or as part of a loading / unloading list).

  • All packages of a shipment are dispatchable.

  • Changes to “dispatched” after dispatching on a trip.

Dispatched:

  • It is possible to cancel or change dispatch to another trip.

Not dispatchable:

  • Loading of the trip is closed.

Partly dispatchable:

  • A dispatchable subsequent delivery exists.

  • If the shipment is dispatched, actually the subsequent delivery is dispatched.

Collection

Dispatchable:

  • Initial state after creating a collection (via import or manually).

  • All packages of the collection are dispatchable, or no packages exist.

  • Changes to “dispatched” after dispatching on a trip.

Dispatched:

  • It is possible to cancel or change dispatch to another trip (PUD-status S “Dispatched”, A “Accepted”, X “Rejected”).

Not dispatchable:

  • Collection has already been processed on a trip.

  • Collections in the state AH “collection hindrance” and “collected” get this status.

Partly dispatchable: not relevant for collections
