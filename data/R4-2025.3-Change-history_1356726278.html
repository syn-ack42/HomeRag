cadis 2025.3.3

Released on July 25th 2025

Bug fixes

General

  • [RD-4077] [cadisAPP] Resolved an issue where the system time reverted to an incorrect past date.

cadis 2025.3.2

Released on July 9th 2025

Improvements

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-3747] [cadisSTUDIO] In cadisSTUDIO, users can now create not only collection shipments but also courier and delivery shipments using the same input form with shipment-type-specific required fields. Additionally, barcodes and shipping labels can be automatically generated for all shipment types and exported as PDFs to support smooth logistics operations.

  • [RD-3748] [cadisSTUDIO] The Shipment Upload feature enables users to import shipment data directly into cadisSTUDIO via a simplified Excel template, streamlining the upload workflow

  • [RD-3885] [Reports] A new report, “Shipment Label”, has been implemented. The label format follows industry standards (e.g., DHL, Hermes) and includes the handling unit barcode (if available). The label is printable as PDF.

Bug fixes

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-3849] [PTV] Implemented caching of PTV2 Dima data to reduce redundant listDima service calls and improve performance.

  • [RD-3946] [Export] The selected driver in the Trip Details section will now be included in the LoadingStatus export. 

cadis 2025.3.1

Released on July 2nd 2025

Improvements

General

  • [RD-3913] [Configuration] Configuration switch ‘includeSequences’ moved from cadis.exports to cadis.th.loading.exports.

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-3721] [cadisSTUDIO] “Package Types” Interface in cadisSTUDIO: A new “Package Types” interface has been added to cadisSTUDIO for managing package-type master data. This feature offers a complete create, read, update, and delete (CRUD) workflow with real-time validation and support for bulk import/export. Accessible directly from the Master Data sidebar, the interface integrates seamlessly with the existing Master Data architecture. It also includes full localisation support and is fully responsive across desktop and mobile devices.

  • [RD-3723] [cadisSTUDIO] A new “Package Deviations” interface has been added to cadisSTUDIO for managing package deviation master data. This feature supports full Create, Read, Update, and Delete (CRUD) operations and introduces advanced process management with dynamic workflow configuration. Accessible from the Master Data sidebar, users can set up multi-step workflows with intelligent form adaptation and full audit trails, perform bulk operations, and rely on advanced validation rules and role-based read-only modes.

  • [RD-3744] [cadisSTUDIO] The new “OrgTree” interface enables efficient management of organisational structures in cadisSTUDIO. It provides a hierarchical view of corporate, country, and depot units with full create, read, update, and delete (CRUD) functionality. Branch management features include interactive map views, bulk import/export, and detailed address and contact handling. Built-in validation rules ensure data integrity across the entire hierarchy.

Bug fixes

Pick Up and Delivery (PUD) and Terminal Handling (TH)

  • [RD-3563] [cadisAPP] A new monitoring event has been added to enhance proactive support for tour import errors. This update allows for quicker identification and resolution of issues, ensuring smoother operation and reducing downtime.

  • [RD-3795] [cadisAPP] Fixed an issue where confirming a delivery with partial loading would result in an error.

  • [RD-3796] [cadisAPP] Updated the Honeywell scanner API to improve compatibility with modern devices.

  • [RD-3801] [cadisCONTROL] Improved driver assignment: When the loading is closed, changing a vehicle with an assigned driver will now automatically update the driver assignment to match the new vehicle.

  • [RD-3850] [cadisCONTROL] Route optimisation: The “Apply” button was previously obstructed by the tooltip. The tooltip has been removed to improve usability.

  • [RD-3861] [Business code] Host statistics calculation: Triggered the host statistics calculation when importing shipments, even if the shipment is made dispatchable again by the import.

  •  

  • [RD-3924] [Business code] Rejected collections from mobile devices will now be ignored if the corresponding collection shipment has been cancelled in the meantime.

Yard Manager (YM)

  • [RD-3892] [cadisSTUDIO] Resolved an issue where data from the previously selected orgUnit was incorrectly retained after logging into a different one, leading to errors.

cadis 2025.3.0

Released on June 24th 2025

General

New features

  • None

Improvements

  • [RD-3546] [cadisSTUDIO] Logfile Request Handling: Users can now initiate a new logfile request even if an existing request is still open. A confirmation dialogue is shown before creating the new request.

  • [RD-3640] [Business code] Monitoring Exception Handling: A suppressed or silent inner CadisRuntimeException will no longer prevent an outer CadisRuntimeException from being reported in monitoring. 

Bug fixes

  • [RD-3438] [cadisAPP] Resolved an issue where the progress bar was incorrectly displayed on completed service pages.

  • [RD-3525] [Business code] System user can update Alpha/Beta Flags in details mask even for external user.

  • [RD-3598] [Business code] Locale from the Servlet request is now provided in the call-scoped request context.

  • [RD-3777] [cadisAPP] Resolved internal time synchronisation issues in CADIS that occurred after device reboot.

Pick Up and Delivery (PUD) and Terminal Handling (TH)

New features

  • [RD-3330] [Improvement] cadisSTUDIO now includes a driver selection drop-down in trip template dialogues, allowing trips to be assigned to specific drivers from master data. This feature enables overriding default vehicle driver assignments for local traffic operations.

  • [RD-3654] [Improvement] cadisSTUDIO: The tour template now includes a new “Driver” column showing the assigned driver’s name. This name appears in the trip templates overview, regardless of any default driver set in the vehicle master data.

  • [RD-1980] [Mobile access] Driver assignments can now be selected and modified directly from route details in Mobile Access.

Improvements

  • [RD-1797] [Mobile access] When drivers log into Mobile Access, their driver ID is automatically applied to trip details.

  • [RD-2595] [Mobile access] Mobile Access users with Driver type now see the correct tours in MyTrips.

  • [RD-2639] [cadisSTUDIO] MyTrips: stops continue to be displayed even after status assignment in cadisCONTROL.

  • [RD-2702] [Business code] When merging unknown packages after shipment data import, loading equipment documentation from loading, unloading, and inventory operations is now properly transferred to the dispatched shipments.

  • [RD-2749] [Rules engine] Rule definitions now include Name2 for shippers and consignees.

  • [RD-2832] [Export, Configuration] The cTLoadingStatus now includes Planned loading sequence, Actual loading sequence (in the order of the first positively loaded package) and Planned stop sequence (where multiple shipments may have the same sequence number if they are in the same stop). These features must be enabled via configuration (cadis.exports.loadingState.includeSequences).

  • [RD-3199] [Performance] Item Table Service: Introduced LIMIT ROWS EXAMINED (MariaDB) with Default of 50,000: To improve performance, the Item Table service now leverages MariaDB’s LIMIT ROWS EXAMINED feature, defaulting to a maximum of 50,000 rows examined per query. If this behaviour is not suitable for your use case, you can configure the limit to 0 to revert to the previous behaviour with no restriction.

  • [RD-3531] [cadisCONTROL] Improved the icons for geocoding status in shipment and order overviews.

  • [RD-3587] [Business code] Removed unreachable loading shipment states ‘Cancelled’ and ‘Surplus’ from trip details.

  • [RD-3638] [cadisSTUDIO] Resolved a crash that occurred when the visibility of menu items changed based on selection.

  • [RD-3647] [cadisCONTROL] The “Geocode all not geocoded shipments” dialogue now has a “Skip” button to skip the current shipment and proceed with the next shipment.

  • [RD-3648] [cadisAPP] The unit of measurement for packaging dimensions in the shipping item details has been changed from meters to centimetres.

  • [RD-3655] [cadisCONTROL] In Local Traffic “Tour details”, the “Driver” field no longer pre-fills with “Default Driver” when no default driver exists.

  • [RD-3678] [Configuration] Added JSON Schema description for pud-transport configuration to improve validation and documentation support.

  • [RD-3722] [Configuration] Added a new configuration option to control whether auto-generated unloading lists, based on line haul loadings, are released automatically upon creation.

  • [RD-3725] [Business code] Improved error handling for stopLocation geocoding when PTV service is unavailable.

  • [RD-3803] [cadisCONTROL] Removed ‘Generate unloading list’ function from line haul trip details (action never had any effect here).

  • [RD-3816] [cadisSTUDIO] Mandatory Volume Entry in GTP:

    • Validation on Save/Route: Action is blocked if both calculated and manual volume are missing. Error: “Volume entry required: Please enter here or in the shipment items.”

    • Calculated OR Manual Volume > 0: Save/Route is allowed.

    • Both present: No restriction.

Bug fixes

  • [RD-2925] [cadisSTUDIO] Resolved an issue preventing users from viewing trips assigned to them when using myTrips with driver-based pairing tokens.

  • [RD-3305] [Business code] Resolved an issue where collections would disappear from unloading lists after being toggled from “collected” to “not collected” and back to”collected” via cadisCONTROL, provided unqualified packages were scanned on site.

  • [RD-3380] [cadisCONTROL] The departure details mask of an imported departure can now be closed.

  • [RD-3388] [cadisCONTROL] The departure details mask of a departure created via Template & Rules Engine can now be closed.

  • [RD-3399] [cadisAPP] Resolved a sporadic crash in cadisAPP when accessing order and stop details.

  • [RD-3421] [cadisSTUDIO] Vehicle table now properly uses organisation’s local date for vehicle overview requests when no date filter is set, resolving issues with orders showing incorrect dates.

  • [RD-3422] [Business code] The server now performs a license check when creating mobile accesses, ensuring compliance and preventing unauthorised access.

  • [RD-3477] [cadisAPP] Resolved an issue with transport unit data entry in protocol when switching the app from background to foreground.

  • [RD-3479] [cadisAPP]  The application now correctly handles confirmation of order updates.

  • [RD-3480] [cadisAPP] Resolved a (de)serialisation exception occurring when confirming tours while putting the app into background.

  • [RD-3483] [cadisAPP] The application now correctly processes tour rejections.

  • [RD-3503] [cadisSTUDIO] Vehicles table: Resolved display of values in the “Departures” column.

  • [RD-3623] [Business code] Unknown loading container types in cTOutboundTransport imports are now treated as no type present, falling back to default behaviour (derived by vehicle references or swap body).

  • [RD-3642] [cadisCONTROL] Resolved an issue allowing trips to be cancelled even when shipments remained in “To be unloaded” state. The bug only affected trip details. Both tables in trip monitor and loading overview did not allow the user action.

  • [RD-3665] [cadisSTUDIO] Resolved an issue when trying to create collection or collection template using a stop location that is closed on every working day.

  • [RD-3680] [Import] Unloading import: Multiple partial shipments of a single main shipment are now properly merged when sent within unloading lists.

  • [RD-3683] [cadisSTUDIO] Trip templates: fixed sorting of columns Vehicle, Created and Last changed.

  • [RD-3789] [Business code] Resolved an edge case where the stop sequence of a trip could become corrupted, leading to multiple downstream errors.

Temperature Control (TCL)

New features

  • [RD-3427] [Public API] Implemented new TCL interface supporting NIC-Place integration. 

  • [RD-3662] [cadisSTUDIO] Menu Temperature Control: the Sensors tab no longer displays sum calculations for the temperature column.

Improvements

  • [RD-3784] [Improvement] Koovea: Added Configurable Caching for Client Request Responses. Client request responses can now be cached for a configurable duration. This enhancement improves performance and reduces redundant backend calls by storing recent responses for reuse within the specified cache window.

Yard Manager (YM)

Bug fixes
