Master pallet Last mile.

License Requirement: PUD-MPL

The master pallet (MP) "Last mile" allows the creation of a MP, in which shipments to any consignee can be consolidated. The only criteria that has to be fulfilled for a shipment to be added to the MP is that the "Traffic type" must be "Last mile".

The main utility of this MP is to speed up loading of delivery vehicles. It is differentiated from the already existing "Consignee" MP by not being meant for delivery at a single consignee as a whole MP. Instead, packages/shipments will be physically taken off the MP and delivered as individual to different consignees.

A mobile user must have the following permissions to be able to create and edit master Pallets of the "Last mile": entryMDEMasterPalletLastMile, newMDEMasterPalletLastMile

  • Creating and editing “Last mile” MPs on the cadisAPP is done in the same way as other master pallets. There is a corresponding new option available for searching and creating “Last mile” MPs.

  • A new icon for the MP Last mile is displayed in all relevant places in cadisAPP, cadisCONTROL and cadisSTUDIO.

  • Like all MPs, the MP Last mile can be loaded by one scan of the MP-Barcode. As soon as the loading is closed, however, the MP is considered dissolved, and the packages have to be processed individually in delivery.

New permission for setting expected storing location

License requirement: TH-PCS or TH-TH-CSC

The user has the possibility to allocate a planned storing location to a shipment. The permission "editItemPlannedStoringPosition" is now required. The permission is assigned to the roles "Administrator", "LT-Dispatcher", "Area Freight Forwarder" and "TerminalManager" by default. You can remove it if you do not want these users to edit planned storing locations for shipments.

Line Hauls in myLoadings

The “My loadings” section now also shows line haul loadings.

The user has a filter option to display all loading lists, only local traffic loadings or line haul loadings.

User interface and functionality are largely the same as for local traffic loadings.

Available loading lists with their status are listed on the left side. They can be filtered by date or status and they can be sorted by status. Not yet released loading lists can be released for loading. Loading lists that are at least "released for loading" can be closed.

When selecting a loading list, assigned shipments and their details are shown.

When selecting a shipment, the packages and their details are shown.

For loading lists as well as for shipments and packages setting a status is possible.

Both shipments and packages have their own history.

Multi-select is possible for loading lists, shipments and packages. When multiple items are selected, the only options that are possible for all selected items are offered. All the other options are disabled or not visible.

Line haul loadings are also integrated into the “Dashboard”.

Configurable mandatory seals on loading “close” in myLoadings

A new configuration is available that controls how many (or none) seals have to be entered when closing a loading list in cadisSTUDIO "myLoadings". If fewer seals than required have been recorded, the user will be get a corresponding error message indicating how to correct the error.

The required number of seals can be configures independently for traffic type (line haul, local traffic) and user type (driver, terminal worker)

cadisCONNECT

cadisCONNECT provides an API layer for programmatic access to cadis functionality.

Depending on the use case, APIs may be designed to

  • integration use cases: retrieve detailed information about specific business objects (e.g., the ETA for the delivery of a certain shipment) or perform actions in cadis (e.g., post a note to a shipment)

  • Mass data load: perform cyclic data retrieval for use in business intelligence solutions.

The API documentation for the current product major release is available via https://demo.cadis.cloud/cadis-rest-public-doc/ with a “Swagger UI” fronted. The URL is protected by a login. If you are interested in cadisCONNECT please contact your project-, service- or account-manager for login data and a brief introduction by one of our consultants.

The API documentation for your current cadis release version is also directly available on you own cadis domain under the same path (https://<your.cadis.domain>/cadis-rest-public-doc/). Please sign into cadisSTUDIO first to authenticate before accessing the documentation URL. Please note that all API specs can be read on the public documentation, but to actually use the API requires the appropriate license to be activated and for the user/token to have the relevant permissions assigned.

Please note that we are currently in BETA with active development ongoing.

  • available APIs don’t yet cover the full range of cadis

  • APIs are subject to change

  • we appreciate feedback!

Length of SealNumber in “TypeSealStatus” (cTBaseComplexTypes)

The maximum length of the "SealNumber" Element in TypeSealStatus was increased to 50. The change to this base type affects the messages cTLoadingStatus and cTUnloadingStatus.

This is a bugfix as the allowed length for imported seal numbers and some seal number entries was always 50 leading to a situation where status messages could not be sent for completely valid input data.
