The dialog with the shipment details can be opened via the context menu "Sheet" (will later be renamed to "Details") or by double-clicking on a shipment. Access is possible from the shipment overview as well as from the shipment list in the loading, unloading, and inventory processes.

Menu

The menu is located at the top of the dialog under (⋮) and offers the same functions as the menu in the Shipments table. (Shipments )

Tabs

The tabs on the left side represent various sections or categories within the application. Each tab is labeled with a name and sometimes an icon. In parentheses to the right of the tab name, the number of entries or items present in this section is displayed.

Summary Tab

In the "Summary," the most important information from all sections is displayed compactly on a single page. This gives you a direct overview of all relevant data and the current status of the shipment without having to open individual sections.

Shipment Data

The "Shipment Data" section contains all essential information about the current shipment. This includes the shipment number, product details, weight, volume, temperature specifications, and other relevant shipment characteristics. Right next to it are the consignor, sender, and recipient with addresses as well as optional contacts including contact details.

Additionally, the planned pick-up and delivery dates with corresponding times appear here—all clearly arranged in separate boxes. Important attributes and special requirements are visible as icons and can be set individually. All data can be edited or supplemented directly in the respective section.

Positions Tab

In the "Positions" tab, the individual position entries of a shipment are displayed. In the overview table, you can see all essential details for each position, such as host ID, sequence, quantity, packaging type, gross weight, and other information such as comments or measurements (length, width, height).

Each row in the table represents a position with its specific data, and at the end of the table, the total for relevant columns is often shown, such as the total weight.

This view provides a quick overview of all position data related to a shipment.

Positions can be edited directly in the table: double-clicking on a table row opens the selected position for editing. New positions can be added via the "+" button at the top of the bar, bringing up a dialog for the necessary position data. This way, existing positions can be adjusted and others added as required.

For positions, actions can be selected by ticking the desired row. The menu or the three-dot menu (⋮) offers the options "Details" (to display and edit the selected position) and "Delete" (to remove the marked position). This allows individual or multiple positions to be conveniently edited or deleted after being marked.

Instructions Tab

Instructions are displayed in the corresponding tab. Each instruction appears as a row in a table and includes important information such as the type, the specific instruction, the related text, and the source. New note texts can be added via the "+" button.

By double-clicking a row or via the three-dot menu (⋮), the instruction can be edited. In detail mode, you can scroll through the individual instructions of a shipment using the arrow keys at the top right (forward/backward). The display "Edit Handling instruction (x/y)" indicates which instruction of the total is currently displayed. This enables fast and convenient toggling between the available note texts without leaving detail mode.

In the "Instructions" tab, individual instructions can be deleted. Mark the desired instructions entries (checkbox), then select "Delete" from the context or three-dot menu (⋮). Once confirmed, the selected instructions are permanently removed from the list.

Notes Tab

In the "Notes" tab, all notes recorded for the shipment are displayed as a list. Notes are a means of communication between dispatcher, customer service, or processor in an ordering depot.

Each row contains information on who the note is intended for (e.g., customer service, recipient), the actual note text, the creator, the creation date and time. It also shows whether and when the note was read and completed.

New notes can be added via the "+" button.

The note text is entered, and by selecting a radio button, you define which group of people the note is intended for (e.g., internal, customer service, disposition). After saving, the relevant columns (!D, !S, !N) in the shipment overview indicate the intended audience for the note.

By double-clicking on a note, it is marked as read.

And with the "Complete" button, it is marked as completed.

A note without a recipient (internal) is instantly marked as completed.

In detail mode, you can scroll through individual notes of a shipment using the arrow keys at the top right (forward/backward). The display "Note (x/y)" indicates which note out of the total is currently displayed, allowing for fast and convenient navigation through existing notes without leaving detail mode.

Money amounts Tab

In the "Money mounts" tab, all financial amounts associated with the shipment are displayed. The table shows the type (e.g., declared value, COD), the amount, the currency, the payment method, and, if applicable, the document type.

At the bottom, the total sum of all entered amounts is calculated and displayed automatically.

Packages Tab

The "Packages" tab lists all packages belonging to a shipment as a list. Each row provides information such as the sequence, standard barcode, the package status (e.g., expected, in process), as well as the current status regarding goods receipt, dispatch, and inventory. Other displayed data include the last status, the current and planned warehouse location, weight and dimensions (width, length, height, volume).

For each package, the selection menu provides access to additional functions such as "Package events," "Package history," or "Photo documentation." The status and recent changes (e.g., loading/unloading) are clearly shown in the table. By selecting a package and choosing "Package events" from the menu, you can view detailed information about the progress and current status of the selected package. This view is for information only and does not allow direct editing of package data.

Addresses Tab

The "Addresses" tab displays all parties involved in the shipment in a list form. For each role (e.g., consignor, recipient, sender), the corresponding address data is clearly presented in a table. Each row contains information such as type, customer number, group reference, company name, additional name fields, street, postal code, city, state, country, and if applicable, additional information such as geographic coordinates or address notes.

By default, the main addresses (such as consignor, recipient, sender) of the shipment including assigned responsible persons or contacts are shown.

Contacts Tab

The "Contacts" tab lists all assigned contacts for the shipment. For each contact it shows which type or role they belong to (e.g., consignor) and the name. Additionally, if available, contact details such as phone, mobile, fax, and email are displayed. This overview allows quick allocation of contacts to their roles and ensures essential communication data is always accessible.

Details Tab

The "Details" tab contains all specific data for the selected shipment. This includes key information such as source, origin system, receiving depot, external host status, target lead time, as well as relevant attributes like distribution order, unloading block, or waybill. Additionally, handling times per stop and per package in seconds are displayed for pickup and delivery. Information about the assigned regional forwarder (determined/calculated) is also visible. All shown details are for viewing purposes only.

Dates Tab

All relevant dates for a shipment are listed in the "Dates" tab. For each date, the type (e.g., delivery date, execution date) and the corresponding date are indicated. Additional time frames (from/to) are provided if required. The display is for informational purposes only; dates cannot be edited or added in this section.

References Tab

The "References" tab lists all references associated with the shipment. Each reference shows the type (e.g., delivery note, client) and the corresponding value. This section is for viewing only and entries cannot be edited.

Loading equipment Tab

The "Loading equipment" tab lists all loading equipment assigned to the shipment, including those mentioned in shipment positions and any additionally added loading aids. Each loading equipment’s quantity and type are shown, and at the bottom, the total sum is displayed. This section is view-only and not editable.

Loading locks Tab

In the "Loading Blocks" tab, all shipment blocks are displayed in a table. For each block, information such as the type (system-generated or manually set in cadis), block reason, block status time, and the responsible user is visible.
The tab is for viewing purposes only and is not editable. This allows for transparent display of whether and why a shipment is currently blocked without changing any data within the tab itself.

Weights / Volumes Tab

The "Weights/Volumes" tab presents all relevant shipment weight and volume data in a table. Each position provides information like host ID, reference type, load carrier, volume, weight, chargeable weight, length, width, height, and girth measurement. The reference type column shows whether the weights and volumes refer to shipment positions or individual packages. These values provide a clear overview of the dimensions and masses of all shipment items. This is a view-only section.

Value added services Tab

All additional/special services relevant for the shipment upon delivery are displayed in the "Value added services" tab. The table includes for each entry information such as ID, unit, service, required flag, time window, min/max values, and additional parameters. This tab is view-only.

Photos Tab

The "Photos" tab shows all photos associated with the shipment or its packages from all processes as previews. Each picture provides supplementary data like title, capture date, and related identifier. Images are for viewing only and cannot be edited or deleted from this tab.

PSts / Processing status Tab

The "BSTs" tab lists all special statuses for a shipment. Each status entry shows info such as type, current status, and a note if available.

The "+" button in the "BSTs" tab allows adding a new special status. Clicking opens a dialog to enter relevant information (e.g., type, status, note). Once saved, the new status appears in the BSTs list. The processing status indicates that processing of some kind is taking place (or has taken place) outside of cadis.

Via the context menu (⋮), details for a selected status entry can be viewed and edited. Deletion of a BST entry from the menu is also possible.

Dangerous goods Tab

The "Dangerous Goods" tab shows all key details for dangerous goods positions in a shipment. For each, the UN number, primary and secondary hazards, classification, package count, packaging type, packaging group, and gross/net quantities are shown. This overview is view-only.

Customs Tab

The "Customs" tab displays all customs-relevant data for a shipment, including, for example, info on customs documents (e.g., import/export) and a table of all customs-related items by type, name, ID, and host ID. This is a view-only area and cannot be edited from the tab.

Status Tab

The "Status" tab provides a clear overview of all status-relevant data for a shipment. The upper section shows general status information such as current status, hall assignment ("in hall"), disposition status, entry, exit and inventory status, plus indicators for damaged goods or packaging. The bottom table lists all status-related records for individual packages, with details like tracking number, type, host ID, hall and disposition attributes, and status for unloading, loading, and inventory. No editing is possible here.

History Tab

The "History" tab shows all tracked actions and status changes for the shipment in table form. Each entry shows data such as level, type, source, user, code, description, and timestamps for both status and entry, as well as any job reference.
