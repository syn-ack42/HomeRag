CADIS Knowledge Base : Add unknown packages to a shipment

Created by Mohamed.Shebl, last modified by Julian Beyer on Sep 08, 2025

Use case and constraints

There are customers, who unfortunately not correctly transfer the packages of a shipment:

  • More packages than physically available are transferred: cadis can already handle this – the packages are marked as “missing”.

  • Less packages than physically available are transferred: The driver / terminal worker can add additional packages to a shipment during the loading / unloading / inventory by cadisAPP.

Data like dangerous goods points or weight of the shipment has to be correct on shipment level. This data cannot be adapted via an unknown package.

Configuration

It can be set by configuration (depot wise, separated for local traffic and line haul) if it is allowed to add an unknown shipment during loading / unloading / inventory.

  1. Open cadisSTUDIO

  2. Go to “Administration”

  3. Select “Configuration Manager” and then “Terminal Handling”

  4. Select “Loading” (or “Unloading” or “Inventory”)

  5. Click the “Plus-Button”

  6. Select “Add package to shipment allowed” under “Package processing”

Scan in the package editing mask

An unknown package is scanned in the package editing mask (the package has not yet been created as unknown package in the unloading or in the inventory – it is completely unknown in the system).

Loading

In the loading the user gets a message, that an unknown package was scanned.

Discard of the scan:

The scan is discarded. You are still in the package editing mask. Further actions (for example loading equipment sacn, …) are assigned to the current package.

Unloading and Inventory

The behavior is different in the unloading and the inventory, because the recording of surplus (unknown) packages is also possible. Because of that, the user is offered the decision to register the package as surplus or to add it to the shipment.

Adding the package to the shipment via context menu

In the shipment overview of the events the context menu of the shipments is expanded by following entry: “Add package”

A window for scanning or entering the unknown package is opened by selecting this menu entry. This can be canceled at any time.

Scanning the package checks if it is actually unknown.

  • Package is not unknown: Error message with reference to the shipment of the package (also NoEDI-packages are refused).

  • Package is unknown: Shift to the package editing mask and further processing (package is added to the shipment) analogous to the scan in the package editing mask.

Adding the package

The package is added to the shipment.

An entry is added to the history that the package has been added to the shipment as unknown package.

The package is now an inherent part of the shipment and is treated as package of the shipment in every export. The package counters of the shipment are increased in the “Should” as well as in the “Is”.

Adding an already known (unknown) package

In the package editing mask or via context menu entry of the shipment an already in the system known “unknown” package is scanned (it is already created as unknown package by unloading or inventory).

Now this should be resolved as unknown package in the system and added to the given shipment.

An entry is added to the package history that the package is added to the shipment as unknown package. The package is marked as ‘added’ so that it can be recognized as added package.

The package must not be a NoEDI-package. If they are packages, which were unloaded in a NoEDI-unloading, this will be handled in accordance with the previous procedure. This package gets its shipment data with the NoEDI-data coming in. (These may be loaded in local traffic, but the trip is not allowed to be closed, before the shipment data is available.)

Removing an added unknown package

An added package can also be removed (for example because of scanning it wrongly and it was not supposed to be added).

The context menu of the added packages offers to remove it from the shipment again. A corresponding entry in the history of the package is made. The package is removed and marked as canceled.

If the loading is already closed, no more changes are made. In this case the package cannot be removed.

Marking the added packages

The added packages are marked with the symbol

in cadisAPP, cadisCONTROL (shipment details and status details) and cadisSTUDIO.

Host import of a shipment, to which packages are added in cadis

The shipment is updated as usual. That means: When added packages are missing, than they are not removed from the shipment, because packages that are created in cadis cannot be removed by host import.

Adding a package to a shipment during building a master pallet

During the building of a master pallet the user can also add a completely unknown package to a shipment.

Precondition:

  • Adding is allowed by configuration.

  • A known package has already been scanned to identify the shipment to which the unknown package should be added.

The user is offered to add the package to the current shipment when scanning the unknown package.

The user can also add the package on his/her own by selecting the corresponding context menu entry of the shipment. Afterwards the barcode of the unknown shipment is scanned or entered manually.

A corresponding icon in the master pallet details marks, when a package was added by an user.

The adding was noted in the package history.
