Maintenance of R4.21

R4.21-LTS-5

Bugs

Logistic Process Platform (LPP) – General functionality

  • [RD-1878] [Business code] Fixed a rare issue where ‘(‘ or ‘)’ within a query search pattern could cause CADIS-1000. [R4.21 LTS-5, 2024.1.0]

  • [RD-1933] [Logging] The following message “Unknown password policy error value” no longer appears in the server log file.

Terminal Handling (TH)

  • [RD-1815] [Business code] Fixed “CADIS-1260″ error in GIS during sequence optimization in the future.

  • [RD-1873] [Business code] Adapted response of loading scan in case of undispatching check failure (already loaded) to behave like “planned on other tour”.

  • [RD-1932] [Export] Loading status export / ActualFreightCarrier: If an ID is present, the address data will not be added. Only if the ID is missing, the full address will be added.

R4.21-LTS-3

New and changed functions

Logistic Process Platform (LPP) – General functionality

  • [RD-1485] [Performance] Info-servlet: display last statistic update for MariaDB tables.

Terminal Handling (TH)

  • [RD-1002] [Configuration] The loading equipment actual values can be configured in advance (“one to one”, “0” or “none”) for processing in the cadisAPP. This configuration is available for each process (loading, unloading, inventory) in Terminal Handling.

  • [RD-1623] [cadisAPP] Barcodes with leading or trailing whitespace are trimmed now before using them. This makes operation more robust because some scanners append a line feed to the barcode. Consequently, barcodes may not contain leading or trailing blanks, tabulators, or line feeds. If they do, cadisAPP will not find the corresponding package.

Terminal Handling (TH)

  • [RD-1169] [cadisSTUDIO] Reworked tour properties handling (long load, dangerous goods, etc.) for trip templates (cadisSTUDIO) and trips (cadisCONTROL). For both traffic types – local traffic and line haul – all of them can be overridden all the time (even if a vehicle combination has been determined). Furthermore, fixed some bugs related to those properties, like that they are derived from vehicle types as well. Especially worth mentioning is a bugfix for imported loading lists (cTOutboundTransport) concerning the max. allowed gross weight. Now, the allowed weight will be derived from vehicle master data while importing. Previously, it only had an effect as soon as the trip was changed in cadisCONTROL. Hence, it is recommended to check vehicle type master data because this fix may affect trip planning and associated checks.

  • [RD-1451] [Business code] Fixed an issue leading to DepotItemState ‘Unknown’ when exported to GTP. Collections will now stop at state ‘PICKUP_OK’ when they have been collected and hence added to an unloading list.
