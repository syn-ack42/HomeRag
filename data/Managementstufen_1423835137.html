Im Folgenden verwenden wir „Benutzer“ als Synonym für „Benutzeraccount“, so wie es auch in der cadis GUI verwendet wird.

Jedem Benutzer kann genau eine Managementstufe zugewiesen werden.

Es gibt folgende Managementstufen, sortiert nach ansteigenden Berechtigungen:

  • Keine
    heißt keine Managementstufe.
    Die meisten cadis Benutzer haben diese Stufe. Sie haben keinen Zugang zu Benutzerverwaltung oder Benutzerrollen.
    Sollten Sie diese Managementstufe haben, möchten Sie vielleicht hier bereits aufhören weiterzulesen.

  • Benutzer, lokal
    um Benutzer lokal zu verwalten, das heißt fokussiert auf (in der Regel einige wenige) Depots.
    Die lokale Benutzerverwaltung ist beschränkt auf die Benutzer ohne Managementstufe. Der Hauptanwendungsfall ist das Verwalten von Benutzeränderungen in den eigenen Depots. Benutzerrollen können gelesen, aber nicht geändert werden.

  • Benutzer, global
    um Benutzer (und nur Benutzer, nicht das ganze System) global (für alle Depots der laufenden Installation) zu verwalten UND die Benutzerrollen zu pflegen.

  • System

Um das gesamte cadis System zu verwalten: Sie können alles anpassen, ohne Einschränkungen und ohne zugewiesene Rolle.

Wenn Sie eine Managementstufe höher als Keine haben, können Sie Änderungen in der Benutzerverwaltung unter „Benutzer & Geräte“

im Menüpunkt „Benutzer“ und „Benutzer-Rollen“ vornehmen:

Der Benutzer Administrator ist immer verfügbar und sichtbar. Dieser ist der einzige automatisch angelegte Benutzer und wird ausschließlich von CADIS Mitarbeitern verwendet, um das cadis Management zu unterstützen.

In Ihrem cadis System hat der Administrator mindestens einen Benutzer für Ihre Organisation angelegt. Dieser hat mindestens die Managementstufe Benutzer, global und ist daher befähigt Benutzer und Benutzerrollen zu verwalten.

Zusätzlich zum Administrator Benutzer listet cadis alle Benutzer auf, die importiert oder in cadis angelegt wurden.

Außerdem listet cadis auch externe Benutzer auf (Benutzer, welche sich über ein externes Authentifikationssystem anmelden). Diese werden als „Externe Benutzer“ in der entsprechenden Spalte gekennzeichnet.

Der Benutzer Administrator und externe Benutzer können in cadis nicht geändert werden.

Besonderheiten der Benutzer pro Managementstufe

System

Benutzer mit der Managementstufe System, kurz Systemmanager, haben keine Rollen zugewiesen: Sie haben die Berechtigung alles anzupassen, ohne eine Rolle zugewiesen zu haben.

Die Benutzerdetails sehen möglicherweise so aus:

Benutzer, global

Benutzer mit der Managementstufe Benutzer, global, kurz globale Benutzerverwalter, verwalten die Benutzer global, also installationsweit.

Diese haben meist weitere Rollen in ihrer Organisation. In cadis haben Sie deshalb eine oder mehrere Benutzer-Rollen zugewiesen, entweder installationsweit oder für einige Depots.

Die Benutzerdetails könnten so aussehen:

Benutzer, lokal

Benutzer mit Managementstufe Benutzer, lokal, kurz lokale Benutzerverwalter, unterstützen ihre globalen Benutzerverwalter Kollegen durch depotspezifisches Verwalten der Benutzer: Sie haben Kenntnis über die täglichen Personaländerungen, zum Beispiel welche neuen Mitarbeiter anfangen, oder welche nicht mehr für das Depot oder Unternehmen arbeiten.

Lokale Benutzerverwalter sind fest eingebunden in das tägliche Transportgeschäft haben deshalb in cadis Benutzer-Rollen zugewiesen. Eine Benutzer-Rolle kann installationsweit oder für eines / mehrere spezielle Depots zugewiesen sein.

Die Benutzerdetails sehen möglicherweise wie folgt aus, wenn wir annehmen, der lokale Benutzerverwalter verwaltet die Benutzer in den Depots Frankfurt und Nürnberg.

Anwendungsfälle je Managementstufe

Im Abschnitt Benutzerverwaltung können Sie Ihre eigene Managementstufe herausfinden. Suchen Sie das entsprechende Kapitel, um die passenden Verwaltungsmöglichkeiten und Anleitungen zu finden.

Managementstufe System

Als Benutzer mit der Managementstufe System verwalten Sie das gesamte cadis System.
Das enthält die Verwaltung von allen nicht externen Benutzern sowie Benutzer-Rollen. Der Fokus dieses Kapitels liegt auf Benutzer-Rollen.

Die Verwaltung der Benutzer und der Benutzer-Rollen können Sie auch an Benutzer, global übertragen.

Erste Aktionen

  • Importieren von Benutzern, unabhängig der Managementstufe, oder Delegation des Imports an einen globalen Benutzerverwalter.

  • Anlegen eines weiteren Benutzers mit der Managementstufe System wenn Sie der Verantwortung mit einem Kollegen teilen möchten.

  • Hinzufügen von globalen Benutzerverwaltern

Wenn Ihr Fokus die Systemverwaltung ist, können Sie die Aufgabe der Benutzerverwaltung delegieren, indem Sie einen oder mehrere Benutzer mit der Managementstufe Benutzer global hinzufügen:

Geben Sie die persönlichen Daten ein sowie den Managementstufe Benutzer, global.

Fügen Sie eine Benutzer-Rolle für den Benutzer hinzu, den Sie gerade anlegen: Was ist seine Rolle? In welchen Depots? Sie können “Alle Depots der Installation” oder ein bzw. mehrere spezielle Depots auswählen.

Weitere Rollen können Sie hinzufügen.

  • Anlegen und Ändern von Benutzer-Rollen

Im Bereich Benutzerverwaltung Reiter Benutzer-Rollen können Rollen angelegt oder eingebaute editiert werden.

Generelle Anwendungsfälle

  • Anlegen, ändern und löschen von Benutzern

  • Ändern der Benutzer-Rollen

  • Import von Benutzern

  • Import von Benutzer-Rollen

Managementstufe Benutzer, global

Als Benutzer mit der Managementstufe Benutzer, global, also als globaler Benutzerverwalter, verwalten Sie die Benutzer global, das heißt installationsweit.

Sie können Benutzer-Rollen anlegen, ändern und löschen. Ebenfalls können Sie Benutzer importieren, allerdings gibt es eine Einschränkung: Sie können keine Änderungen (anlegen / ändern / löschen) von System Benutzern vornehmen.

Sie können zudem die depotspezifische Benutzerverwaltungsrolle delegieren.

Erste Aktionen

  • Import von Benutzern, wenn sie nicht Managementstufe System besitzen.

  • Hinzufügen von weiteren globalen Benutzerverwaltern

Ihr System Manager hat Ihnen möglicherweise die Aufgabe übertragen, weitere globale Benutzerverwalter anzulegen. Siehe dazu den Abschnitt “Hinzufügen von Benutzer, global“ im Bereich „Managementstufe System“

  • Hinzufügen von lokalenBenutzermanagern
    Gegebenenfalls fokussieren Sie sich auf das globale, installationsweite Verwalten der Benutzer und delegieren die Aufgabe der depot-spezifischen Benutzerverwaltung an einen Kollegen. Dazu legen Sie einen oder mehrere Benutzer mit der Managementstufe “Benutzer, lokal“ angelegt:

  • Lokaler Benutzerverwalter

    • Sollen Benutzer mit Fokus auf einzelne Depots verwalten.

    • Kennen die täglichen Personaländerungen möglicherweise besser als ein Globaler Benutzermanager

    • Zum Beispiel: Diese wissen wer als Fahrer neu startet oder welcher Fahrer seine Tätigkeit beendet.

Hierzu klicken Sie auf “Anlegen“
Befüllen Sie die persönlichen Angaben

Fügen Sie eine Rolle hinzu: Was ist seine Rolle? In welchen Depots? Wählen Sie “Alle Depots installationsweit” oder eine bzw. mehrere spezielle Depots für die er die Rolle ausübt.

Fügen Sie bei Bedarf weitere Rollen hinzu.

Wählen Sie die Managementstufe “Benutzer, lokal” und die Depots welche der lokale Manager verwalten soll. 

  • Ändern und Löschen von nicht-System Benutzern

  • Ändern der Benutzer-Rollen

  • Im Bereich Benutzerverwaltung Reiter Benutzer-Rollen können Sie Rollen anlegen oder vorhandene editieren.

Generelle Anwendungsfälle

  • Anlegen, Ändern und Löschen von Benutzern außer System Benutzern

  • Ändern von Benutzer-Rollen

  • Import von Benutzern (ausgenommen System Benutzer)

  • Import von Benutzer-Rollen

Managementstufe Benutzer, lokal

Als Benutzer mit der Managementstufe Benutzer, lokal, also als lokaler Benutzerverwalter unterstützen Sie Ihren globalen Benutzerverwalter, indem Sie Benutzer eingeschränkt auf „Ihren“ Depots verwalten.

Sie wissen genau über die täglichen Personalveränderungen Bescheid, beispielsweise wer heute als Fahrer zu arbeiten beginnt oder welcher Fahrer seine Tätigkeit beendet. Des Weiteren wissen Sie wer seine Rolle in „Ihren“ Depots wechselt bzw. weitere Rollen übertragen bekommt. cadis stellt sicher, dass all Ihre Änderungen an Benutzern nur lokale Auswirkungen in „Ihren“ Depots haben.

Diese Managementstufe beinhaltet keine Benutzer-Rollen-Verwaltung oder Benutzerimport bzw. Benutzer-Rollen-Import. Die Benutzer-Rollen können Sie öffnen, um sich damit vertraut zu machen.

Anwendungsfälle

  • Hinzufügen von Benutzern der Stufe „Keine“

Hierzu klicken Sie auf “Anlegen“
Befüllen der persönlichen Angaben

Auswahl der Benutzer-Rolle für den Benutzer, der angelegt wird: Was ist seine Rolle? In welchen Depots? Es kann “Alle Depots installationsweit” oder eine bzw. mehrere spezielle Depots gewählt werden.

Mehr Rollen können hinzugefügt werden.

Die Managementstufe ist standardmäßig Keine und kann nicht geändert werden.

  • Ändern von existierenden Benutzern
    Um sicherzustellen, dass die Änderungen nur Auswirkungen auf “Ihre” Depots haben, sind die Bearbeitungsmöglichkeiten eingeschränkt:

    • Sobald ein anderer lokaler oder globaler Benutzermanager ein anderes als “Ihre” Depots zugewiesen hat, sind die persönlichen Daten (Benutzer- und Kontaktdetails) nicht mehr editierbar.
      Sollten Sie dennoch eine Änderung benötigen, wenden Sie sich an den globalen Benutzerverwalter.

    • Benutzer-Rollen in den eigenen Depots können weiterhin verwaltet werden

      • Je vorhandene Rolle: Zuweisen und entfernen eines „Ihrer“ Depots.

      • Löschen einer Rolle, welche nur „Ihren“ Depots zugewiesen ist

      • Hinzufügen von Benutzer-Rollen für „Ihre“ Depots

    • Einschränkungen aufgrund der Rollenkategorie
      Jede Berechtigung hat eine bestimmte Kategorie aus den folgenden {BUSINESS, INTERNAL, ADMIN, GLOBALADMIN}.
      Rollen erhalten automatisch die höchste Kategorie aus allen zugewiesenen Berechtigungen (aufsteigend von BUSINESS nach GLOBALADMIN). Die Kategorien sollen dabei helfen verschiedene Ebenen der Verantwortung zu identifizieren, welche ein Benutzer durch eine Rollenzuweisung bekommt.

       

      • Benutzer mit Benutzer-Rollen der Kategorie GLOBALADMIN können nicht editiert werden.
        Sollte eine Anpassung notwendig sein, kann dies durch einen globalen Benutzerverwalter geändert werden.

  • Löschen von bestehenden Benutzern
    Es gelten die gleichen Einschränkungen, wie auch für das „Ändern von existierenden Benutzern“, siehe vorheriger Abschnitt.
