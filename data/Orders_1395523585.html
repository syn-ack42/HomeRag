General

The orders table can be accessed via the "First & Last Mile" tab "Orders" on the cadisSTUDIO tile.

The order overview provides a comprehensive view of all collection and delivery orders that have been assigned for processing in a secondary distribution route. During tour planning, concrete delivery and collection orders are created based on dispatched shipments and collections. These orders are then assigned to the corresponding tours and can be accessed and processed digitally by the driver.

A delivery order may also refer to a delivery attempt that has already been made but was unsuccessful. For example, if the initial delivery attempt fails, a new delivery order can be created for a subsequent attempt. This ensures flexible and traceable processing of shipments – both during the initial delivery and in the case of any redelivery attempts.

The order overview enables seamless tracking and coordination of all collection and delivery processes carried out in local transport.

Important: Collection and delivery orders assigned to a tour only become visible in the system once loading has been completed and the tour has been officially released.

In cases of returns or rescheduling of collections the originally created collection orders remain in the system. These are marked with the status "Cancelled".

You can filter, select, and group orders as needed.

Filters

Data Query Filters

In the left-hand, basic filter section, various filter options are available in a combo box to help you tailor the order overview and quickly find relevant information.

The following filter options are available:

  • Attribute – Complete list of all possible attributes that can be assigned to an order, e.g. "Cash on delivery", "Priority", “Empties”, "Dangerous goods", "No processing status", "Time-critical orders", "Time-critical orders (not confirmed)", "Temperature-controlled – Individual", "Temperature-controlled 15–25°C", "Temperature-controlled 2–8°C".

  • Date from – to

  • Author

  • Commissioning number

  • Consignee – Name

  • Consignee – City

  • Consignee – Postal code

  • Consignor – ID

  • Consignor – Name

  • Consignor – City

  • Consignor – Postal code

  • Departure – Filter for destination route

  • Name of trip

  • Order ID

  • Order number

  • Order type – Collection or Delivery

  • Package number

  • Purchase order number

  • Reference number

  • Shipper – Name

  • Shipper – City

  • Shipper – Postal code

  • Status – Filters by order status, e.g. "All (except cancelled)", "With problem", "Not accepted orders", "Orders in process", "Orders to be completed", “Delivered with discrepancies”, "Completed orders", "Cancelled orders".

  • Trip – Selection from route master data; filters orders by their departure relation.

  • Vehicle – A vehicle from the master data can be selected.

  • Vehicle group – A vehicle group defined in the vehicle master data.

Additionally, multiple filters can be used simultaneously to allow for targeted narrowing of the displayed orders.

The currently active filters are shown above the table, providing a clear overview of which filter settings are currently applied. The primary filters selected on the left side are highlighted in yellow.

Toolbar "Columns" and "Quick Column Filter"

To the right of the table, within the vertical tab, you will find the Columns section. This area displays all available columns.

Also on the right side of the table, in the Filter tab, you can further narrow down the list of vehicles based on entries in each column as needed. Above the table, you can see the currently active filters: the additional restrictions selected on the right are shown in white.

Table

The columns are self-explanatory. A description of each column is displayed as a tooltip by hovering the mouse over the column header. In many cases, if the field content is unclear, you can use the tooltip to obtain further details.

The table also includes a summary row that sums up all numerical values. In addition, Boolean values are counted.

The status times that are marked in yellow to indicate that the status was assigned on a different day than the one set in the filter area.

Context Menu for Orders

Using the context menu (right-click on the table), you can perform various actions on the selected orders.

  • Details – This option opens the detailed view of the order.

  • Change status – This option allows you to manually change or set the status of an order. It is particularly useful if the mobile device has transmitted an incorrect status or if no status information is available at all.
    The Change Status dialog differs for collection and delivery orders, as different information and options are displayed for each.

  • Set processing status – This option allows you to set a processing status in order to document processing steps carried out outside of CADIS.
    The defined processing status is shown in the overview in the “BSts” column, allowing the current status of the respective processing to be tracked at any time.

  • Confirm time critical – This option is used to acknowledge an order as time-critical. Once confirmed, the order is removed from the "Time-Critical (Unacknowledged)" filter and no longer appears in that view.

  • Show Order on Map – This option opens the map window, displaying the selected order. This allows you to locate the order directly on the map.
