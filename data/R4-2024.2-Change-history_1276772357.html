cadis 2024.2.2

New and changed functions

Logistic Process Platform (LPP) – General functionality

  • [RD-1992] [Improvement] Removed unused classes and resource bundles from cadis.

Bug fixes

Terminal Handling (TH)

  •  [RD-2584] [cadisSTUDIO] A tile (main menu item) is only visible if the user has permission to see at least one menu item (tab) in it. This also affects menu items marked as ALPHA features.

  • [RD-2619] [cadisCONTROL] Collections created from template will now initiate geocoding if the template/stoplocation is not geocoded. If successful, data quality is improved; if unsuccessful, the user will now be able to recognize such collections properly (the icon has been missing before).

  • [RD-2622] [Business code] It’s now possible to create temperature reports for collections that registered packages at stop site.

  • [RD-2669] [Business code]  Not loaded packages are now dispatchable after loading close + enabled “make dispatchable” more often.

  • [RD-2589] [Reports] Dangerous Goods document: “ADR certificate and identity verification” and “vehicle labelling requirement” should also be set if the “transportCategory” = 0.

  • [RD-2480] [cadisAPP] Fixed a crash of the cadisAPP on finishing the loading if the user was logged out of the loading list.
