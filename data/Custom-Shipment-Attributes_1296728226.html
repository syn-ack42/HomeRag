CADIS Knowledge Base : Custom Shipment Attributes

Created by Julian Beyer on May 22, 2025

The implementation procedure for new custom attributes is as follows.

Specification

The customer submits the desired attributes to a CADIS consultant stating the following information for each attribute:

  • AttributeID of the attribute as is will be used in the cTExpectedDepotItem interface (or corresponding cTExpectedItem stucture)

  • Data type (currently cadis supports string and long integer custom attributes up to a maximum of 6 of each type)

  • Label: How the attribute shall be labeled in the user interface (including translations if needed)

Customer Side Implementation

Customer extends shipment data interface (ctExpectedDepotItems) to contain these required attribute. If customer is using cadisDIS (Data Integration Services) mapping is adjusted accordingly; if the data is already available somewhere in the data supplied from the host system a simple addition to the existing mapping may be sufficient.

The data is to be added in the structure at this xPath:

/cTExpectedDepotItems/item:cTExpectedItems/item:cTExpectedItem/item:CustomAttributes

Please refer to the interface definition (cTDepotItem-2-0.xsd and includes)

Example:

...
</item:HandlingTimes>
<item:CustomAttributes>
    <item:CustomAttribute>
        <item:AttributeId>size</item:AttributeId>
        <item:Value>46</item:Value>
    </item:CustomAttribute>
</item:CustomAttributes>
<bct:AttachmentReferences>

...

cadis Configuration

A CADIS system integrator

  • configures the attribute names in the cadis system configuration

  • adds the translations in cadis

Operation and Usage

The additional attributes are imported with the standard shipment (item) data as per configuration.

  • additional columns are shown in the cadisCONTROL consignment overview.

    • Column heading is the (translated) attribute label

    • The filed value is the data sent in the interface evaluated as the configured data type

    • a new filter for each attribute is provided in “Master Search Dispatching”

  • additional columns are shown in cadisCONTROL “Trip details” shipment list

  • the new attributes can be used as conditions in the “Rule Engine” rule definitions
