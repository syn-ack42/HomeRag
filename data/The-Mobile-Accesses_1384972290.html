CADIS Knowledge Base : The Mobile Accesses

Created by Mohamed.Shebl, last modified by Julian Beyer on Sep 08, 2025

You can create in cadisSTUDIO QR code accesses for your CadisAPP users. These QR can be configured to match your users profile & your organization needs.

Configuration of your Mobile accesses

Three major steps for configuration of mobile accesses exist in cadis. The Mobile access type, the authentication modes & the role granted.

The mobile access type

Vehicle & Multiple Vehicles

The CadisAPP session is linked to one of your vehicles & will receive its related trips. You must have created your vehicles in the Master data menu.

Scanner

The CadisAPP session is free from any master data. Depending on your mobile licenses, you can create as many accesses as you want.

Driver

The CadisAPP session is linked to one of your drivers. You must have created your drivers in the Master data menu.

The authentication modes

❶ If you select the authentication mode “Code only”, the user only has to scan the QR code & press “log in”, no additional data or password is required.

❷ If you select the authentication mode “Password”, you must define a password at creation of the access & give it to your user. After scanning the QR code, he will have to enter the password to log in

❸ f you select the authentication mode “User account”, you must have created a user account previously (more details here. At log in, your CadisAPP user will have to enter his ID & password.

The three roles

❶ The Driver gets access to the menus & features related to pick-up & delivery (last mile tasks)

❷ The terminal worker is in charge of cross-dock activities (mainly unloading, inventories, linehaul loadings & Infoscan)

❸ The Mobile User is a “super-user”, gathering the access rights of drivers & Terminal Workers. Not often used in Production but in Test, could fit the needs of a key user or Terminal manager.

How to create a new Mobile access

Follow the guide below to create a new Mobile access. To know more about the types of mobile accesses, the authentication types & the roles, see part 2 of this article.

❶ In cadisSTUDIO, open the « Users & Devices» menu, in the tab “Mobile accesses”, click on Create.

❷ If you can create an access of type Scanner, enter it’s ID (free text).

For an access of type vehicle, select a vehicle from the list (the vehicles must have been preiviously created in the master data).

❹ Select an Authentication type

❺ Select a role & click on “Generate”

How to log in cadisAPP

Scan the generated QR code

Press “Log in”

If the role selected is Driver or Mobile user (see below), the CadisAPP user must enter a vehicle at log in. Indeed, those roles both have the trip processing through a vehicle as a essential prerogative

Identification of a Mobile Device / User

When a mobile device (or more precisely: the mobile access tied to that device) is used to perform a significant action, e.g. scan a package, set a status etc., the actor is logged and can be found in a history, a status message and/or an API call.

How that “actor” is represented depends on the wat the mobile access was granted.

With a mobile access where the Authentication is “User account” (i.e. username + password or “Single Sign On”) the username used for the login is provided. This is the same as when cadisCONTROL or cadisSTUDIO is used.

In case of the anonymous Authentications (“Code only” or “Password”) there is no actual username. In this case the system identification from the mobile access is used in its stead. This identification is:

  • With a „Vehicle" access the VehicleID

  • With a „Driver" access the DriverID

  • With a „Scanner" access the chosen „Scanner “-name
