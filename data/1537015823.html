The dialog with the Order details can be opened via the “Details” option in the context menu or by double-clicking on an order in the order overview.

The Order Details screen provides a read-only, comprehensive view of specific shipping operations with a visual design aligned to the Shipment Details screen for consistency and improved user experience.

Header Area

At the top of the Order Details screen, you will find summarized information about the selected order, including:

  • Order ID: The unique identifier for the order.

  • Order Number: The sequential number assigned to the order.

  • Customer Number: The reference number for the customer (if available).

  • Trip: The identifier or name of the vehicle trip for the order.

  • Vehicle: The specific vehicle assigned to this order.

  • Source: Source or origin code related to the order.

  • Status: The current status of the order, highlighted for visibility.

  • Recipient / Shipper Information: Detailed names and addresses for both the shipper (sender) and consignee (recipient), shown in separate sections.

  • Deadline / Shipment: Product, Delivery Date, Delivery Times, Cash On Delivery, Delivery Terms (Specific conditions for the delivery), Total shipment weight, Loading Meters, Chargeable Weight, Volume.

Actions

At the top of the mask, the Actions section provides buttons such as “Recipient”, “Change Status”, ”Show Order on Map”, and “Print”.

  • Recipient – opens a dialog where you can view and manage the recipient (consignee) information for the selected order.

  • Change status – This option allows you to manually change or set the status of an order. It is particularly useful if the mobile device has transmitted an incorrect status or if no status information is available at all. See (Orders)

  • Show Order on Map – This option opens the map window, displaying the selected order. This allows you to locate the order directly on the map. (https://cadissoftware.atlassian.net/wiki/spaces/kb/pages/1411383297/Map+First+Last+mile?atlOrigin=eyJpIjoiY2Q1YzgyMmRmNzdmNGFhY2E3Y2YxZWU2ZDI0ZTE1MTYiLCJwIjoiYyJ9)

  • Print Delivery Order (for deliveries) – Generates and prints the delivery documentation that accompanies the shipment to the recipient. This function is used to provide proof of delivery and detailed information about the shipment contents.

  • Print Collection Order (for collections) – Generates and prints the collection documentation that guides the driver during pickup operations. This function is used to provide detailed instructions for collecting the shipment from the sender.

  • Print Proof of Delivery – Generates and prints a formal document confirming the delivery of the shipment. The Proof of Delivery (POD) includes all relevant details such as recipient information, delivery date and time, and signatory details.

  • Print Photo Documentation – Generates and prints a formal document with all photo records associated with the delivery/order. The photo documentation may include images of the goods, signatures, or the delivery location, providing additional visual evidence for the delivery process. If the option is greyed out, it means no photos are available for documentation.

Tabs

The tabs on the left side represent various sections or categories within the application. Each tab is labeled with a name and sometimes an icon. In parentheses to the right of the tab name, the number of entries or items present in this section is displayed.

Goods Items Tab

The “Goods Items” tab provides an overview of all packages or items included in the order. It allows users to see, at a glance, which goods are part of the current shipment, including their quantity, packaging types, and relevant details. 

Instructions Tab

Instructions are displayed in the corresponding tab. Each instruction appears as a row in a table and includes important information such as the type, the specific instruction, the related text, and the source.

Details Tab

The “Details” tab displays a summary of all relevant order information and package statuses. You can quickly see the number of total, delivered, damaged, not delivered, missing, and scanned packages, as well as additional shipment details such as reference numbers, waiting time, signatory, status time, and GPS position. This section provides a clear overview for tracking the progress and documentation of the order.

History Tab

The “History” tab provides a chronological overview of all status changes and key events for the order. Each entry shows the event status, user, vehicle, times of status and creation, and change details. Colored highlights indicate important status transitions. 

Packages Tab

The “Packages” tab displays a list of all packages associated with the order. For each package, you can view details such as the standard barcode, current status, status time, and scan mode.

Loading Equipment Tab

The “Loading equipment” tab shows all load carriers associated with the order, such as pallets or containers. For each type, you can see planned and actual quantities, quantities delivered to and returned from the customer, as well as any not exchanged and reasons for non-exchange. Additional remarks can also be recorded. 

Photos Tab

The “Photos” tab shows all images that have been captured and linked to the order. 

Value-added Services Tab

The “Value-added services” tab displays all extra services performed as part of the order, such as special handling, additional deliveries, or other supplementary tasks. For each service, details like ID, type of service, quantity, completion status, timing, and comments are shown. This section helps users keep track of all additional services linked to the shipment.

Map Tab

The "Map" tab displays the geographical location relevant to the order or delivery. It provides an interactive map to visually represent key addresses, such as shipping or delivery locations. This helps users quickly identify the exact location of the order on the map.
