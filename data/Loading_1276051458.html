Loading Overview

You reach the Loading overview using the :BeladungNah: symbol (Last mile loading)or :BeladungFern: (Line haul loading) in the cadis control panel. In both cases the screens and their operation are almost the same.

The loading overviews show the loading of a day in overview form with the option to monitor the loading progress.

Filter area

You filter the loading processes using date, route (secondary distribution) or Outbound relation (Primary distribution) and status.
Vehicle transfer failed: During the import of the loading list, the assigned vehicle should be modified. This was rejected by cadis because the loading had already been completed and the route has been released.

Table

In the column headers, you will receive an explanation of the column as a tooltip. In columns for which totals are displayed this is shown in the footer.

Here are additional explanations for some columns:

  • Status

    • Possible status values for secondary distribution are:

      • Created

      • Released for loading

      • Loading in progress

      • Loading end (Terminal staff have closed the loading)

      • Waiting for completion (The driver has requested the route release)

      • Ready for tour start (Dispatcher has completed the loading)

      • On tour (The first stop of the route has been processed)

      • Refused (Driver has refused the takeover of the route)

    • Possible status values for primary distribution are:

      • Created

      • Released

      • In progress

      • Finished (Terminal staff have closed the loading)

      • Closed (Dispatcher has completed the loading)

  • Shipments / Packages / Weight: In these columns you can follow the loading progress.

  • Secondary distribution only: By using the Number of stops processed / Deliveries processed/ Collections processed columns you can follow the progress of the tour as it progresses.

  • Damaged goods, damaged packaging, issues.The :SpaltenBeschAbweich: columns show the damages and issues that are recorded during this loading process. The :SpaltenBeschAbweichVorgänge: columns show the damages and issues that are registered in other operations (loading / unloading, inventory, delivery).

  • Property column: These are represented by symbols. Tooltips show the number of shipments for the corresponding property.

Context menu

  • Loading details, Open in loading details #2 / #3.
    Opens the Details screen. You can open 3 detail screens at the same time. You can define which screens the details should be displayed in.

  • Release Loading
    Driver / terminal staff can register for the loading list after its release.

  • Deny Loading completion (for secondary distribution only)By doing so, you are rejecting the driver’s wish to finish loading. You must enter a comment to the driver:

  • Close Loading
    This finalizes the loading after a driver/ terminal employee has reported the end of the loading.

  • Change the vehicle (For secondary distribution only)
    You assign another vehicle to the route

  • Release for the vehicle (For secondary distribution only)
    If the driver has refused the route, you can dispatch it on the same vehicle again.

  • Delete Departure
    You can cancel departures/routes that you have created yourself in cadis if no more shipments are scheduled.

  • Print
    Different documents are opened in the PDF reader and can then be printed.

  • Confirmation of failed vehicle transfer
    If the assigned vehicle should be changed during the import of a loading list, but this was not possible in cadis because the loading process was already completed and the route was released, then this is shown on the route as an event requiring confirmation.
    With this menu item you confirm the event. It is then no longer displayed as unacknowledged and it is no longer displayed in the desktop notification.

Loading details

In the Loading details (Trip details screen for Last mile or Line haul) you can see the details of a loading list, in particular the shipments.
You can also schedule shipments and collections for the route / departure.

  • Last mile
    A complete last mile trip is planned. Shipments with the same address and time slot are combined into stops. A more precise trip plan (e.g. sequence optimization) is based on the stops.
    The stops can be seen in the Stops Tab.

  • Line haul
    A departure is planned that only includes shipments that all go to the same address, namely the next depot in the transport chain.
    The destination address is visible in the Destination address tab. It results from the depots that are stored for outbound depots list.

Shipments Tab

Displays the shipments with loading status.

Fields with issues vs. normal case, e.g. loading with issue (damage) are framed in orange.

Drag&Drop dispatching

In this tab, you can use the Drag & Drop functionality (drag a shipment from the shipment overview to this tab) or reschedule (move a shipment from one route/departure to another).

The following functions are made available using the context menu:

  • Load item
    Sets the status “Loaded” for all packages, regardless of what the mobile system has previously reported. Since the status set by cadisCONTROL has priority over the status set by the mobile system, the entire shipment is considered loaded.

  • Item not loaded
    Sets the status “Planned” for all packages, regardless of what the mobile system has previously reported. Since the status set by cadisCONTROL has priority over the status set by the mobile system, the entire shipment is considered as “Not planned”.

  • Set MDE status again
    Deletes the status assigned using cadisCONTROL and reverts to the status assigned by the mobile system.

  • Set processing status
    Sets a processing status for processing that takes place outside cadis (status + text)

    Note: The processing status only applies to this loading list, it is not visible in other processes or in the shipment overview!

  • Undo dispatching
    The shipment is removed from the loading list

  • Create partial shipment
    A partial shipment is created for the shipment

  • Re-dispatch
    With this option you can schedule the shipment for a different route/departure. For this, the scheduling dialog appears, in which you can select the new departure.

  • Status details
    Switches to the status details, description see below.

  • Shipment details
    Switches to the Shipment details screen.

  • Corresponding order (for secondary distribution only)
    Switches to the Order details screen to show the collection/ delivery order. Only possible when loading has been completed.

  • Set PUD status (for secondary distribution only)
    This sets the collection or delivery status

  • Send collection to driver again
    A collection that was refused by the driver is sent again

  • Edit instructions
    Edits the instructions. You can add, modify or remove instructions.

  • Print dangerous goods attachment, loading list, waybill or collection order

Stops Tab (for last mile only)

Displays stops and stop sequence. Orders with the same address and time window are grouped into stops.

Using the context menu, you can view the details of all orders or perform a dispatch cancellation for all shipments on the stop.

Destination address Tab (for line haul only)

Here you can see the recipient address. This results from the address of the depot account that is assigned to the outbound relationship in the master data.

Freight carrier Tab

Shows the freight carrier data / Selection of a freight carrier

Shows the runsheet transfer: Status (Runsheet transferred or refused) Timestamp, Name und Signature of the driver.

User activities Tab

Shows the user and terminal staff activities

Info Tab

General info about the loading process

Vehicle / Loading unit Tab

The table shows the vehicles and loading units listed.
If the loading list was imported, you will see the imported values in grey and the values actually used in black.

Loading report Tab

Enter loading data without a shipment reference. Data that was entered using cadisAPP is also visible here.

Gates lists with imported values in grey and the manually entered gates (using cadisCONTROL or cadisAPP) in black.

Seals Tab

Seals with the imported values in grey and the manually entered seals (using cadisCONTROL or cadisAPP) in black.

Auxiliary loading equipment Tab

Shows loading unit (loading unit without shipment reference) that was entered using cadisAPP. You have the option to create auxiliary loading unit entries:

Double-click an entry to see the details. You can enter additional loading equipment by clicking on the New button:

Dispatching info Tab

Enter a target storing location and handling instructions for this route/departure.

For unloading operations, the system displays this target storing location and the handling instructions for the shipments that are planned for this departure.

Photos Tab

Shows photos that have been taken during the loading process without a shipment reference.

History Tab

Displays important actions that have been performed during the loading process.

Master pallets Tab

The loaded master pallets are displayed here. You can see the status of the shipments that are on the master pallets in the Shipments tab.

Status details

Status details for the loading of a shipment.

Packages Tab

Status at package level
Shows the status that was assigned using cadisAPP (MDE status), the status that was assigned using cadisCONTROL (GUI status) and the resulting status (Status column).

Context menu:

  • Package loaded
    Sets the status “Loaded” for the package, regardless of what the mobile system has previously reported. Since the status set by cadisCONTROL has priority over the status set by the mobile system, this package is considered loaded.

  • Package not loaded
    Sets the status “Planned” for the package, regardless of what the mobile system has previously reported. Since the status set by cadisCONTROL has priority over the status set by the mobile system, this package is considered as not loaded (Planned status).

  • Set MDE status again
    Deletes the status assigned using cadisCONTROL and reverts to the status assigned by the mobile system.

  • Package details
    Shows further details of the package.

    You can see here the issues recorded by cadisAPP. With the :PosAddSub: buttons you add an additional issue (discrepancy) or you delete the selected one.

  • Photo documentation
    Displays photos of the package that have been taken

Loading equipment Tab

Shows the loading equipment recorded by cadisAPP.
If non-exchangeable items have been entered, you can see them in a separate window by double-clicking on the corresponding line.

Photos Tab

Displays photos of the shipment that have been taken.
