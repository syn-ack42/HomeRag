Introduction

This release note is based on cadis R4 2025.1.4

Please take a look at the R4 2025.1 Change history, which contains the full list of changes.

Many of the features listed here are available across all basic service cut levels. However, some may require an additional licence, while others may have varying accessibility or utility depending on service cut limitations and your specific use of cadis. In some cases, the availability of information and/or functionality within a feature may also depend on the service cut level and other licenced options.

General functionalities

New features

Multi-factor authentication (MFA)

We are introducing support for Multi-Factor Authentication (MFA) in CADIS, using Two-Factor Authentication (2FA) with Code Authenticator. This enhancement adds an extra layer of security for users accessing the system. This update is designed to enhance security by providing an extra layer of protection against unauthorized access. MFA helps ensure that only authorized users can access the system, improving overall security and aligning with industry best practices.

Key Details

  • What’s New?
    MFA through Code Authenticator is now available for users who log in via cadisSTUDIO user management. This requires an additional form of authentication alongside the usual login credentials.

  • Scope of Applicability
    This feature is only available for users managed through cadisSTUDIO. It is not applicable to users with login management via AzureAD.

  • Global Activation
    Enabling MFA is a global setting for the entire installation. Once activated, users will need to complete additional authentication when logging in.

You will find the detailed description of this function on the following page: Setting-up multi factor authentication

If you need assistance with enabling this feature, please contact your Account Manager.

Added device information in cadisSTUDIO, Mobile devices

We have added mobile device and mobile access information to the Mobile devices tab. With this, you have all the information in one place.

  • Device info: Manufacturer, OS version, device serial number, IMEI, IMSI, ICCID

  • Mobile access: Pairing name, Pairing role, Pairing type

Remark
Due to security restrictions introduced by Google in Android Version 10, certain device information (e.g., serial number, IMEI) is no longer available for display in cadis. In such cases, cadis can only display the generated device ID that is not linked to the hardware. However, for customers using cadis MDM (Mobile Device Management powered by SOTI), we can now display the full device information.

On Android devices running version 10 or higher that are not integrated with our MDM, this information remains restricted by Android security policies, and only the cadis generated device ID is available.

Added cadis information on MDM (SOTI)

Customers using cadis mobile device management (SOTI) can now see the cadisAPP associated device information (e.g. client id, vehicle, user, depot, etc.) within the MDM user interface.

New supported devices for cadisAPP

We have added the following Zebra devices to our portfolio of tested and certified devices: MC34 and TC78.

Please see cadisAPP Compatibility for background information on Android device compatibility.

Improvements

cadisAPP

We have improved GPS tracking functionality in cadisAPP. One key improvement concerns BYOD users: on these unmanaged devices, not controlled by an MDM, GPS location permissions must be granted by the user himself. The user is now guided through the permission setup process, and if access is not granted for any reason, this is clearly notified within the app.

cadisSTUDIO

  • Menu tiles are now visible to users with permission to access at least one item within them. Users without access to any items in a tile will not see the tile

  • The display in dark mode has been improved

Translation

  • General: Custom translations are now consistently applied in all cases

  • cadisCONTROL: Screens now consistently display in the selected language, even when using a slow internet connection

Performance improvements

  • Improved performance of database queries for loading close reports 

  • Improved performance for interface message delivery and extended configuration for high throughput scenario optimization

Pick Up and Delivery (PUD) and Terminal Handling (TH)

New features

Unloading and Inventory (Beta)

In cadisSTUDIO, the “Unloading” and “Inventory” functionalities have transitioned from alpha to beta.

These features have been developed to be used on tablets in responsive mode. The main objective is for a dock supervisor to have all the necessary information at hand and not have to return to their PC to, for example, close an unloading process, view damages, etc.

Improvements

Simplified re-optimization for ongoing trips

In cadisCONTROL, when opening the sequence optimization dialogue for a single ongoing trip, unprocessed delivery stops will no longer be fixed by default. Additionally, an issue has been fixed where already processed stops could incorrectly appear as unchecked for a fixed position.

After

Collection templates

Numerous changes have been made regarding regular collections in cadis and the creation of templates, as part of the migration to cadisSTUDIO

  • The “Order Management” section has been removed, and the “Collection templates” have been relocated under “Rules & Templates”

  • When creating a collection template, the “Active” checkbox is now checked by default

  • It is now possible for users to assign a tour to a collection template

  • The collections created from a template will now initiate a geocoding if needed. If unsuccessful, the user can now identify such collections (icon added)

  • When creating a collection template, users can now add shipper and consignee addresses

Other improvements

cadisAPP

  • The vehicle entry is now limited to a maximum of 20 characters, regardless of the input method (keyboard or scan)

  • The shipment details and history are now adjusted to smaller device screens without being cut off

cadisCONTROL

  • The primary hazard field in the dangerous goods attachment is now empty if it has a value of 1, 1.4, 1.5 or 1.6. (Code “CC” = classification code remains)

  • On the dangerous goods report, ADR certificate, identity verification, and vehicle labelling requirements are now only checked if the shipment exceeds 999 dangerous goods points or if isHighConsequence is set to true

  • The dangerous goods classification in the recurring trip details is now displayed in the user’s selected language

  • While attempting to generate the “Delivery List with Signatures” report, a pop-up message now appears if the report is empty

  • Removed unnecessary columns in unloading details, tab Assigned collections

  • Unloading processes with assigned collections now have unnecessary columns removed for improved clarity and usability

  • A numeric priority set in cadisCONTROL can now be cleared

cadisSTUDIO

  • The Estimated Time of Arrival from an unloading list is now correctly transferred to the Arrival Board when processed through the host system

  • The maximum length for a storing position is now limited to 20 characters

New configuration options

These options can be activated by the CADIS team via our Config Manager. Contact your consultant or account manager for assistance.

  • Added a parameter to prevent stop splitting when dispatching via drag-and-drop on the map

  • Added a parameter to prevent overriding loading locks during dispatching

Bug fixes

cadisAPP

  • Fixed a rare issue where users were not automatically logged off when their session was terminated via cadisSTUDIO, Mobile devices. Now, the termination ensures automatic logout.

  • Fixed a rare issue where a “No internet connection” error was incorrectly displayed when accessing the message tab in tour details

cadisCONTROL

  • Fixed an issue where undispatched shipments were incorrectly included in certain trip-related statistics

  • Fixed an issue where a shipment could no longer be dispatched after cancelling all delivered packages of a partial delivery

  • Fixed an issue where scanning a package not listed on the unloading list caused errors, even when other packages from the associated shipment were expected

  • Fixed an issue where the actual storing position of unknown packages was not successfully transferred to the corresponding qualified packages when shipment data was imported

  • Fixed an issue where the accumulated gross and chargeable weight was not updated after item import for unloading and inventory. Additionally, weight is now displayed with two decimal places in the unloading context

  • Fixed an issue where not all shipments were displayed when using the inventory areas filter on the inventory details screen

  • Fixed a rare issue when creating stop location grouping requests that included multiple addresses with the same non-existent stop location association

New configuration options

cadisYM

New features

The Gate Manager has been migrated to cadisSTUDIO (Beta)

The Gate Manager functionality was one of the first web-based features developed by cadis to enable usage on tablets. However, the technology used has become outdated, which is why we have migrated this functionality to cadisSTUDIO. It is now available alongside other Yard Manager features, with an interface aligned with the ergonomics of the other functionalities.

Additionally, the following features have exited beta status: Shunting service, Transport units, Transport unit history
