Multi-factor authentication (MFA)

In today’s digital landscape, securing access to critical systems is more important than ever. With the increasing prevalence of cyber threats, stolen credentials, and unauthorized access attempts, relying solely on a username and password is no longer sufficient to ensure the protection of sensitive business data.

Multi-Factor Authentication (MFA) provides an additional layer of security by requiring users to verify their identity using a second, dynamically generated authentication factor. This means that even if a user’s password is compromised, unauthorized access remains highly unlikely without the second verification step.

In CADIS, MFA is implemented through Two-Factor Authentication (2FA), utilizing a Time-based One-Time Password (TOTP). This industry-standard security method ensures that login attempts are protected using unique, time-sensitive authentication codes, which are generated by a trusted authenticator application. By enforcing this additional security measure, CADIS helps businesses mitigate the risk of cyber attacks, unauthorized access, and account takeovers, thereby safeguarding vital transportation and logistics operations.

MFA in CADIS is designed to offer both security and convenience, ensuring that users can easily authenticate while maintaining a highly secure login process. The implementation of TOTP-based authentication means that:

  • Each login session is uniquely verified, reducing the risk of stolen credentials being reused.

  • Authentication codes expire quickly, preventing malicious actors from intercepting and misusing them.

  • The system remains flexible, allowing users to use their preferred authenticator app for seamless access.

By integrating robust authentication measures, CADIS reinforces its commitment to enterprise-grade security, ensuring that only authorized personnel can access critical business tools and data. Implementing MFA is a simple yet powerful step toward a safer, more resilient digital environment for logistics professionals worldwide.

MFA Availability and Limitations

MFA is exclusively available for user logins via cadisSTUDIO, the primary desktop and web application for CADIS users. Mobile users accessing CADIS via handheld devices or mobile applications are not required to use MFA, as their authentication is already secured through an integrated pairing functionality within the system.

This pairing mechanism acts as an inherent two-factor authentication method, ensuring that only authorized mobile devices can access the system. By linking a specific user account to a registered mobile device, CADIS eliminates the need for additional MFA steps while still maintaining a high level of security. This approach provides a seamless yet robust authentication process for mobile users while preventing unauthorized access.

Enabling and Using Multi-Factor Authentication

  • MFA can be enable via a configuration page accessed by the product consultants.

  • Once MFA has been enabled for a users, the following steps will take place upon the next login attempt:

QR Code Generation

  • When the user logs in after MFA activation, a unique QR code will be displayed on the login screen.

  • This QR code is unique to the user and serves as their personal authentication key.

Authenticator App Registration

  • The user must scan the provided QR code using an authenticator app that supports TOTP (Time-based One-Time Passwords), such as:

    • Google Authenticator

    • Microsoft Authenticator

    • Authy

    • Any other TOTP-compatible authentication application

Login with MFA

  • After successfully registering the QR code in the authenticator app, the app will generate a time-sensitive six-digit authentication code that changes periodically.

  • During each subsequent login attempt to cadisSTUDIO, the user will be required to enter their username, password, and the current authentication code from the authenticator app.

Resetting MFA in Case of Lost Access

If a user loses access to their authentication device or is unable to retrieve authentication codes, their MFA setup must be reset. This can only be performed by an administrator with access to the User Management section in cadisSTUDIO.

Steps for Resetting MFA

  • The administrator navigates to User Management in cadisSTUDIO.

  • The affected user’s account is selected, and the MFA reset option is used to revoke the existing authentication setup.

  • Once the reset is completed, the user will be prompted to reconfigure MFA upon their next login attempt.

  • The user must repeat the Enabling and Using Multi-Factor Authentication steps outlined above to regain access.

Best Practices for MFA Security

  • Backup Your Recovery Codes: If your authenticator app provides recovery codes during setup, store them in a safe and secure location to regain access if your device is lost.

  • Use a Backup Authenticator App: Some authentication apps (such as Authy) allow multiple device synchronization to prevent losing access.

  • Notify IT Support Promptly: If you experience login issues due to MFA, contact your system administrator or IT support team for assistance.

For further questions or assistance, please refer to the CADIS Help Center or contact your system administrator.
