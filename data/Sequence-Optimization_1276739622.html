The Sequence optimization function optimizes the route of a local transport tour based on fastest route or the shortest route whilst taking time restrictions into account.

You can optimize before loading begins in order to calculate the best loading sequence. You can also optimize the stops that have not been completed so far on the route, for example, after you have added additional collections.

You reach the Sequence optimization screen via:

  • Route details screen, Edit menu

  • List of routes, context menu in the list of routes (Right click on the route)

  • The prerequisite is that all stops are geocoded. If the menu item is grayed out, you can find out the reason why by hovering over it with the mouse.

In principle you have the option

  • To manually sequence stops

  • To let cadis optimize the sequence (based on either the fastest or shortest route)

  • To have the sequence optimized first and then make manual changes

Optimization parameters

  • Vehicle and route are set when the route is created.

  • The planned departure time is taken as the departure time. You can change this time but it will then be used as the expected departure time.

  • Fixed means that the route should start at this time. Otherwise, the optimizer can adjust the departure time so that the first stop matches the specified time window and is not reached too early.

  • Allow toll roads is preset. The optimization uses all streets

  • Fastest / Shortest route. Choose how to optimize your shipments.

Typical operating sequence

First check the optimization parameters and correct them if necessary.

Correct the service times for the stops, if necessary.

Run the sequence optimization (Optimize button). The stops are now put into an optimal sequence according to your requirements, whereby all collection stops are generally placed after the delivery stops.

If all stops or time slots could not be met you have the following options:

  • Adjust time of departure

  • Edit service times

  • Ignore time slots settings for individual stops by setting a ðŸ—¶ in the Time column for the stop

  • Remove individual stops by selecting them in the Stop table using the context menu for the stop table and choosing the Reset Disposal function.

If you want to change the result manually, e.g. because you want to place certain collection stops between the delivery stops:

Change the order of the stops manually by

  • Selecting one or more stops in the stop table and move them with the arrow keys

  • Or dragging the stops on the map over the connecting line between 2 other stops using the mouse button.

  • Any values (PTA) calculated by route optimization for the subsequent stops are no longer valid and are deleted.

  • Now calculate the values for your manually altered route (Route button).

  • The km, PTA, Departure, WZT values are recalculated for each.

  • You can then compare the overall result (Time of Arrival, Driving time, Distance) with the result of the optimization and make further corrections if necessary or accept the result.
