General

The shipments table can be accessed via the "Shipments" tile on the cadisSTUDIO dashboard.

The shipments overview provides a quick overview of all shipments that match the selected filter criteria. It shows important information at a glance such as shipment number, tracking number, transport type, shipment type, shipment properties, priority, planned departure, loading, vehicle, NV status, unloading, and other details. The overview enables efficient monitoring and control of shipment status and planning.

Filters

Data Query Filters

In the left, basic filter area, various filter options are available in a combo box to specifically adapt the shipments overview and find relevant information faster. The following filter options are available:

  • Execution Date — From and Execution Date — To — By default, the current date is preset in the "Execution Date — From" and "Execution Date — To" filters. This means that when opening the shipments overview, all shipments whose execution date falls on the current day are automatically displayed. This gives the user an immediate overview of all shipments planned for the current day without having to manually adjust the filters.

  • Additional staff Collection — Shows only shipments where additional staff is planned for collection.

  • Additional staff Delivery — Filters all shipments where additional staff is planned for delivery.

  • All Top Products — Shows only shipments with goods marked as "Top Product".

  • Appointment type Delivery — Filters by the type of delivery, e.g., fixed date, time window, or standard delivery. The following options are available:

    • Area: Shows shipments where a delivery period (from/to) is defined.

    • To: Shows shipments that should be delivered by a specific time.

    • Fixed: Shows shipments for which a fixed delivery date (exact date and time) has been agreed.

    • No Date: Shows shipments for which no special delivery date is recorded.

    • From: Shows shipments that may be delivered from a specific time.

  • Arrival Notification — Filters all shipments for which an arrival notification (e.g., by phone or email) is planned or has already been triggered.

  • Bulky — Filters all shipments declared as bulky goods.

  • Cancelled values — Shows only shipments that have been cancelled or marked as cancelled. This filter option allows users to specifically view shipments that are no longer active in the system due to cancellation.

  • Collection with crane — Filters shipments where pickup by crane is planned.

  • Collection with tail lift — Filters shipments that require pickup with a tail lift.

  • Confirmation Status — Filters by the confirmation status, e.g., whether a shipment has already been confirmed. The following options are available:

    • Accepted: Shows all shipments that have already been confirmed and accepted.

    • Rejected: Shows all shipments that have been rejected.

    • To be confirmed: Shows all shipments where confirmation is still pending.

  • Consignee — Name — Enables targeted search for shipments from a specific consignee by name.

  • Consignee — Postal Code — Filters shipments by the consignee postal code.

  • Consignor — Name — Enables targeted search for shipments from a specific consignor by name.

  • Consignor — Postal Code — Filters shipments by the consignor postal code.

  • Customs Goods — Shows only shipments that are subject to customs duty.

  • Dangerous Goods — Filters all shipments declared as dangerous goods.

  • Delivery Type — Enables filtering by the desired delivery type. The following options are available:

    • Collection by the customer - Shipments that are picked up by the customer themselves

    • Delivery by the customer - Shipments that are delivered by the customer themselves

    • Direct collection - Shipments collected directly from the sender

    • Direct delivery - Shipments delivered directly to the recipient

    • Direct delivery - depot - Shipments delivered via depot or hub locations

    • Other collection - Shipments handled through other collection methods

    • Standard - Regular standard delivery procedure

  • Delivery with crane — Shows only shipments that are to be delivered by crane.

  • Delivery with tail lift — Shows only shipments where a tail lift is required for delivery.

  • Dispatching status — Enables filtering by the current dispatch status of the shipment. The following selection options are available:

    • To be dispatched: Shows all shipments that still require dispatch (planning/assignment).

    • Dispatched: Shows all shipments that have already been dispatched, i.e., assigned to a tour or vehicle.

    • Not dispatchable: Shows shipments that are currently not dispatchable, e.g., due to missing information or blocks.

    • Partially dispatchable: Shows shipments that can only be partially dispatched, e.g., when only part of the shipment can be assigned.

With this filter, the user can specifically monitor and control the processing status of shipment dispatch.

  • Earliest Delivery Date — Enables filtering by the earliest possible delivery date of a shipment. A specific date can be selected here.

  • Empties — Filters shipments that are empty containers.

  • Failed Shipment Import — Shows only shipments where the import into the system failed. The following options are available:

    • To be acknowledged: Shows all shipments where a failed import still needs to be confirmed (acknowledged).

    • Acknowledged: Shows all shipments where the failed import has already been acknowledged.

    • No acknowledgment required: Shows all shipments where no acknowledgment is necessary for a failed import.

  • Food — Shows only shipments that contain food.

  • Geocoding Required — Filters all shipments where geocoding (e.g., for address location) is still necessary.

    • To be geocoded: Shows all shipments where geocoding (e.g., for address location) still needs to be performed.

  • Hide Main Shipments — Hides main shipments from the overview so that only partial shipments or other relevant shipments are displayed.

  • High-value Shipment — Filters all shipments declared as high-value shipments.

  • Last mile Status — Enables filtering by the current last mile status (e.g., tracking status) of the shipment. The following options are available:

    • Collection hindrance: Shows only shipments where there is an obstacle or hindrance during the collection/pickup process.

    • Delivery hindrance: Shows only shipments where there is an obstacle or hindrance during the delivery process.

    • Refused by the driver: Shows all shipments that were rejected by the driver.

  • Latest Delivery Date — Filters shipments by the latest possible delivery date. A specific date can be selected.

  • Loading Block — Shows only shipments for which a loading block is recorded.

  • Long Good — Filters all shipments declared as long goods.

  • Notification — Filters shipments for which advice (e.g., telephone pre-notification) is recorded.

  • Notifications — Shows only shipments where notifications are activated or have occurred. The following options are available:

    • Shipments with unprocessed notes for customer service: Shows all shipments for which there are still open, unprocessed notes for customer service.

    • Shipments with unprocessed notes for dispatch: Shows all shipments for which there are still open, unprocessed notes for dispatch.

  • On Terminal — Shows only shipments that are currently on terminal. The following options are available:

    • Completely on terminal : Shows all shipments that have completely arrived on terminal.

    • Partially on terminal: Shows shipments of which only a part has already arrived.

    • Partially/Completely on terminal: Shows both completely and partially stored shipments.

    • Not on terminal: Shows all shipments that are not yet on terminal.

  • Outbound Route (last mile) — Shows only shipments that have a specific outbound relation in last mile transport. One or more relations can be selected.

  • Outbound Route (line haul) — Shows only shipments that have a specific outbound relation in line haul transport. One or more relations can be selected.

  • Photo Available — Shows only shipments for which a photo (e.g., as delivery proof) is recorded.

  • Priority — Filters shipments by their priority (e.g., particularly urgent shipments). The priority can take values from 0 to 9. Additionally, there are the following selection options:

    • No Priority (Shipments that have not been assigned a priority)

    With this filter, you can specifically search for particularly urgent or less urgent shipments to control processing accordingly.

  • Product — Enables targeted selection by specific products within shipments. The product list includes the following shipping types: STANDARD, EXPRESS 08 (delivery by 8:00 AM), EXPRESS 10 (delivery by 10:00 AM), EXPRESS 12 (delivery by 12:00 PM), NIGHT EXPRESS (night shipping with delivery in the early morning hours), and other services.

  • Receipt Available — Shows only shipments for which a receipt (e.g., delivery proof) already exists.

  • Shipment Group — Enables filtering by specific shipment groups to display similar shipments together.

  • Shipment Number — Enables targeted search for a specific shipment number.

  • Shipper — Name — Enables filtering by the shipper's name.

  • Shipper — Postal Code — Enables filtering by the shipper's postal code.

  • Stackable — Filters all shipments that are stackable.

  • Temperature Range — Enables selection of a specific temperature range, e.g., for temperature-controlled transports. The following options are available:

    • 2-8°C: Shows all shipments that must be transported in the temperature range of 2 to 8 degrees Celsius.

    • 15-25°C: Shows all shipments that must be transported in the temperature range of 15 to 25 degrees Celsius.

    • Individual: Shows shipments for which an individual, deviating temperature range has been defined.

    • Not temperature-controlled: Shows all shipments that are not subject to special temperature control.

  • Temperature Sensitive — Shows only shipments that are temperature-sensitive and require special handling.

  • Tracking Number — Enables targeted search for a specific tracking number.

  • Transport Type — Filters by transport type, e.g., direct transport, long-distance transport, or local transport.

  • Type of Shipment — Filters by the type of shipment. The following options are available:

  • Pickup: Shows all shipments that need to be picked up.

    • Master Pallet: Shows all shipments marked as master pallets (e.g., a pallet containing multiple shipments).

    • Package: Shows individual packages.

    • Shipment: Shows regular standard shipments.

    • Partial Shipment: Shows partial shipments that belong to a larger overall shipment.

  • Upright during Transport — Filters all shipments where the transport instruction "transport upright" is recorded.

  • Value-added Services — Filters shipments where additional services (e.g., special services) have been booked.

  • Waste — Shows only shipments declared as waste.

Additionally, multiple filters can be used simultaneously to enable targeted narrowing of the displayed shipments. Above the table, the currently active filters are displayed so that it is immediately apparent which filter settings are active. The basic filters selected on the left side are highlighted in yellow.

Toolbar "Columns" and "Quick Column Filter"

To the right of the table in the vertical tab is the Columns block. In this area, all columns are displayed.

To the right of the table, you can also use the vertical tab Filter to further restrict shipments by entries in each column as desired. Above the table, you can see the currently active filters: in white, the further restrictions chosen on the right.

The Table

Column Overview and Groupings

Tooltips for Column Headers

You can get a brief explanation of the individual columns by hovering over the respective column header with the mouse and pausing briefly. A tooltip will appear that explains the content and meaning of the column in more detail.

Thematic Groupings

Some columns in the table are thematically grouped to ensure better overview and structure. The groups include, among others: Loading, Unloading, Inventory, NV Status, Client, Sender, Recipient, Delivery and Pickup Times, Weights and Volume, In Warehouse, Temperature, Master Pallet, etc. Each of these categories contains specific information that supports a structured and user-friendly presentation of the data.

Automatic Filters and Sum Row

When calling this view, the corresponding filters are automatically applied so that only the relevant shipments are displayed. The table contains a sum row at the bottom where all numerical values are automatically summed. Additionally, the number of entries with "True" or "False" for Boolean fields is also counted and displayed.

Context Menu

The context menu enables quick access to various actions that you can apply to selected shipments. You open the context menu by right-clicking on one or more marked rows in the table.

Functionality

Through the context menu – which is opened by right-clicking on one or more marked shipments – you can quickly and directly perform a variety of actions. These are context-dependent functions, i.e., only relevant or permitted actions for the current selection are activated.

  • Active entries can be clicked directly and executed.

  • Gray entries are not available for the current selection. → If you hover briefly over a disabled entry, a tooltip appears explaining the reason for the deactivation.

  • You can filter actions specifically using the search field at the top of the menu.

  • Favorites: Some actions have a star symbol ⭐ on the right. This serves for favorite marking.

Actions

  • Details — Opens the detail view of the shipment. This action is only available when exactly one shipment is selected. If multiple shipments are marked, this menu item is disabled.

  • Dispatch (Last mile) — Assignment of the shipment for last mile transport. You will be guided to the Dispatch mask where you can select a last mile transport tour to which the selected shipments are dispatched.

  • Dispatch (Line haul) — Assignment of the shipment for line transport. You will be guided to the Dispatch mask where you can select a line transport departure to which the selected shipments are dispatched.

  • Arrival Notification — This function is only available for local transport shipments (NV shipments) or pickups that have a specific property or configuration. Through the mask, an arrival notification can be triggered – this means the recipient (consignee) or another contact is informed about the upcoming arrival of the shipment. The notification can be made, for example, by phone, email, or other means.

  • Assign area freight forwarder — The shipment is assigned to a forwarder responsible for the destination area.

  • Cancel Assignment — If the assignment was incorrect or the responsibility has changed, the assignment can be revoked. The shipment is then "open" again for a new assignment.

  • Auto Dispatch — Starts the automatic dispatch (Auto dispatch) for the previously selected shipments. The respective dispatch target has already been determined in advance through the postal code routing and the defined rule set of the auto dispatch.

  • Disqualify shipments — With this function, a shipment is manually set to the status "shipped out". In the loading list, the note "Ship out" is entered. The warehouse status is no longer "in warehouse", and the dispatch status is set to "not dispatchable".

  • Start Free Route Optimization with Selection — Starts the free tour optimization for the selected shipments. You will be redirected to the Tour Optimization mask where the optimization parameters can be defined.

  • Loading List / Unloading List / Inventory List — Opens the respective applicable list – provided the shipment was previously part of such a list. This enables targeted tracking of the shipment within the logistical processes such as loading, unloading, or inventory.

  • Corresponding Trip — Opens the tour monitor with direct selection of the relevant tour and the associated stop, if the shipment is part of a local transport tour.

  • Shipment Dispatchable — Sets the shipment to "dispatchable" status. This function is required when a shipment is to be delivered again or newly planned after a delivery obstacle or return. Only dispatchable shipments are considered in the list for manual dispatch or automatic dispatch (Auto dispatch).

  • Create partial Shipment — Enables the creation of a follow-up shipment (also referred to as a partial shipment) based on the existing shipment. This becomes necessary, for example, when complete delivery was not possible or for partial deliveries due to volume or weight restrictions. The follow-up shipment takes over all relevant data from the original shipment and can be planned and delivered separately.

  • Set PUD Status — Enables manual setting of the pickup or delivery status (PUD = Pickup and Delivery) of a local transport shipment. This function is used when: the status was not automatically transmitted by the driver in the mobile system or the set status needs to be corrected.

Status for Delivery Order:

  • Set Processing Status — Serves to document internal processing steps outside the cadis system – e.g., for inquiries, clarification cases, or delays due to delivery obstacles. In the associated mask, a processing status as well as an explanatory free text can be entered.

The set processing status is displayed in the shipments overview in the "Bsts Shipment" column – including a tooltip that contains the complete text as well as the username of the person who entered it.

  • Change Execution Date — This function enables changing the planned execution date of a shipment – e.g., when delivery or pickup is to take place on a later day. After the change, the shipment no longer appears in the list of dispatchable shipments for the current day and is thus no longer considered by the automatic dispatch (Auto dispatch) for the current day. This is helpful to specifically exclude shipments from daily planning without completely canceling or postponing them.

  • Edit Instructions — Opens the "Shipment Details" mask with the ability to edit existing note texts (e.g., add or correct), delete, or enter new texts. Note texts serve the communication of special information about the shipment, e.g., delivery notes, special features at the recipient, or internal notes.

  • Change Shipment Group — With this function, the group assignment of a shipment can be adjusted. The original grouping was automatically made through the rule set for shipment group determination. If regrouping is required (e.g., due to changed priorities, targets, or recipient structure), you can select a new group from a list of existing groups and assign the shipment to it.

  • Dispatching/Loading Lock — Prevents the shipment from being dispatched or loaded by setting a lock. This function is used when a shipment needs to be temporarily excluded from dispatch planning or loading processes, for example, due to pending clarifications, missing information, or temporary holds.

  • Remove Dispatching/Loading Lock — Removes the previously set dispatching or loading lock, making the shipment available again for dispatch planning and loading processes. This function is used when the reason for the lock has been resolved and the shipment should be included in normal operations again.

  • Cancel Collection — Completely cancels the shipment collection, removing it from all active planning and dispatch processes. This function is used when a shipment is no longer required or cannot be processed, effectively terminating all related operations.

  • Send Collection to Driver Again — Re-sends the collection information to the driver's mobile system. This function is used when the driver needs to receive updated collection details, for example, after changes to pickup times, addresses, or special instructions.

  • Dissolve Master Pallet — Breaks down a master pallet into its individual shipments, allowing them to be processed separately. This function is used when the consolidated master pallet needs to be split for individual handling or when partial delivery is required.

  • Close Master Pallet — Finalizes a master pallet, marking it as completed and preventing further modifications. This function is used when all shipments within the master pallet have been processed and the consolidation is complete.

  • Acknowledge Failed Shipment Import — Confirms receipt and acknowledgment of a shipment that failed to import properly into the system. This function is used to document that the import failure has been noted and is being addressed.

  • Refuse — Rejects a shipment, indicating that it cannot or will not be processed. This function is used when a shipment is not acceptable due to various reasons such as missing documentation, prohibited goods, or operational constraints.

  • Change Priority — Modifies the priority level of a shipment to influence its processing order. This function is used to adjust the urgency of a shipment, with higher priority shipments being processed before lower priority ones in dispatch and delivery planning.

  • Assign Planned Storing Location — Specifies where a shipment should be stored in the warehouse or transshipment facility. This function is used to organize warehouse operations and ensure proper placement of shipments for efficient loading and dispatch.

  • Print Dangerous Goods Attachment — Generates and prints the required documentation for shipments containing dangerous goods according to local regulations. This function is used to comply with legal requirements for transporting hazardous materials.

  • Print Dangerous Goods Attachment (International) — Generates and prints the international dangerous goods documentation required for cross-border shipments containing hazardous materials. This function is used to comply with international transport regulations.

  • Print Delivery Note — Generates and prints the delivery documentation that accompanies the shipment to the recipient. This function is used to provide proof of delivery and detailed information about the shipment contents.

  • Print Collection Order — Generates and prints the collection documentation that guides the driver during pickup operations. This function is used to provide detailed instructions for collecting the shipment from the sender.

  • Print Temperature Report — Generates and prints temperature monitoring documentation for temperature-controlled shipments. This function is used to provide proof of proper temperature maintenance during transport.

  • Print Details with Notes — Generates and prints comprehensive shipment documentation including all associated notes and special instructions. This function is used when complete documentation including all annotations is required.

  • Print Details without Notes — Generates and prints shipment documentation excluding notes and special instructions. This function is used when clean documentation without additional annotations is preferred.

  • Print Package Label — Generates and prints identification labels for individual packages within a shipment. This function is used to provide clear identification and tracking information for package-level handling.

  • Geocode Address — Automatically determines the geographical coordinates (latitude and longitude) for the shipment's collection or delivery address. This function is used to improve route planning and provide precise location data for navigation systems.

Geocoding Address

  1. Select Orders

  • In the shipment table, check the boxes for one or more orders.

  • Right-click on the selected orders to open the context menu.

  • Choose “Geocode address” from the enabled menu options.

  1. Review Original Address

  • The system displays the original address details (Street, House Number, Country, Postal Code, City) in read-only fields.

  1. View Geocoded Matches

  • A results table lists all matching geocoded addresses found by the system.

    • Select the most appropriate geocoded address.

    • The chosen row is highlighted in green.

  1. Verify on Map

  • The right-hand side displays an interactive map with the selected location marked by a blue pin.

    • Use zoom and layer controls to check surrounding streets and confirm accuracy.

  1. Apply or Navigate

  • Click “Apply and continue” to save the selected geolocation to the address record.

    • Use “Previous” and “Next” buttons to process multiple addresses in sequence.

  1. Exit Options

  • Click “Cancel” or the “X” (close) button at any time to exit without saving changes.
